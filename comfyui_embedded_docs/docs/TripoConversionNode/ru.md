Вот перевод документации на русский язык:

Узел TripoConversionNode преобразует 3D-модели между различными форматами файлов с помощью Tripo API. Он принимает идентификатор задачи от предыдущей операции Tripo (генерация модели, риггинг или ретаргетинг) и преобразует полученную модель в нужный формат с различными опциями экспорта.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `original_model_task_id` | MODEL_TASK_ID,RIG_TASK_ID,RETARGET_TASK_ID | Да | MODEL_TASK_ID<br>RIG_TASK_ID<br>RETARGET_TASK_ID | Идентификатор задачи от предыдущей операции Tripo (генерация модели, риггинг или ретаргетинг) |
| `format` | COMBO | Да | GLTF<br>USDZ<br>FBX<br>OBJ<br>STL<br>3MF | Целевой формат файла для преобразованной 3D-модели |
| `quad` | BOOLEAN | Нет | True/False | Преобразовывать треугольники в четырёхугольники (по умолчанию: False) |
| `face_limit` | INT | Нет | -1 до 2000000 | Максимальное количество полигонов в выходной модели, используйте -1 для отсутствия ограничения (по умолчанию: -1) |
| `texture_size` | INT | Нет | 128 до 4096 | Размер выходных текстур в пикселях (по умолчанию: 4096) |
| `texture_format` | COMBO | Нет | BMP<br>DPX<br>HDR<br>JPEG<br>OPEN_EXR<br>PNG<br>TARGA<br>TIFF<br>WEBP | Формат экспортируемых текстур (по умолчанию: JPEG) |
| `force_symmetry` | BOOLEAN | Нет | True/False | Принудительно применять симметрию к модели (по умолчанию: False) |
| `flatten_bottom` | BOOLEAN | Нет | True/False | Выравнивать дно модели (по умолчанию: False) |
| `flatten_bottom_threshold` | FLOAT | Нет | 0.0 до 1.0 | Порог выравнивания дна (по умолчанию: 0.0) |
| `pivot_to_center_bottom` | BOOLEAN | Нет | True/False | Перемещать точку опоры в центр дна модели (по умолчанию: False) |
| `scale_factor` | FLOAT | Нет | 0.0 и выше | Коэффициент масштабирования для модели (по умолчанию: 1.0) |
| `with_animation` | BOOLEAN | Нет | True/False | Включать ли анимационные данные в экспорт (по умолчанию: False) |
| `pack_uv` | BOOLEAN | Нет | True/False | Упаковывать ли UV-координаты (по умолчанию: False) |
| `bake` | BOOLEAN | Нет | True/False | Запекать ли текстуры (по умолчанию: False) |
| `part_names` | STRING | Нет | Список через запятую | Список названий частей для включения в экспорт, разделённых запятыми (по умолчанию: "") |
| `fbx_preset` | COMBO | Нет | blender<br>mixamo<br>3dsmax | Пресет экспорта FBX (по умолчанию: blender) |
| `export_vertex_colors` | BOOLEAN | Нет | True/False | Экспортировать ли цвета вершин (по умолчанию: False) |
| `export_orientation` | COMBO | Нет | align_image<br>default | Режим ориентации экспорта (по умолчанию: default) |
| `animate_in_place` | BOOLEAN | Нет | True/False | Анимировать ли модель на месте (по умолчанию: False) |

**Примечание:** Параметр `original_model_task_id` должен быть действительным идентификатором задачи от предыдущей операции Tripo (генерация модели, риггинг или ретаргетинг). Параметры, помеченные как "расширенные", являются необязательными и требуют настройки только для специфических требований экспорта.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| *Именованные выходы отсутствуют* | - | Этот узел обрабатывает преобразование асинхронно и возвращает результат через систему Tripo API |
