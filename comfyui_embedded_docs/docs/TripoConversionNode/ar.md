بالتأكيد، إليك الترجمة العربية للوثيقة التقنية الخاصة بـ TripoConversionNode مع الالتزام التام بقواعد الترجمة المحددة:

تقوم عقدة TripoConversionNode بتحويل النماذج ثلاثية الأبعاد بين صيغ ملفات مختلفة باستخدام واجهة برمجة تطبيقات Tripo. تأخذ هذه العقدة معرف مهمة من عملية Tripo سابقة (توليد نموذج، أو تجهيزه بالهيكل العظمي، أو إعادة توجيهه) وتحول النموذج الناتج إلى الصيغة التي تريدها مع خيارات تصدير متنوعة.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `original_model_task_id` | MODEL_TASK_ID,RIG_TASK_ID,RETARGET_TASK_ID | نعم | MODEL_TASK_ID<br>RIG_TASK_ID<br>RETARGET_TASK_ID | معرف المهمة من عملية Tripo سابقة (توليد نموذج، أو تجهيزه بالهيكل العظمي، أو إعادة توجيهه) |
| `التنسيق` | COMBO | نعم | GLTF<br>USDZ<br>FBX<br>OBJ<br>STL<br>3MF | صيغة الملف الهدف للنموذج ثلاثي الأبعاد المحول |
| `رباعي` | BOOLEAN | لا | True/False | هل سيتم تحويل المثلثات إلى أشكال رباعية (الافتراضي: False) |
| `حد الوجه` | INT | لا | -1 إلى 2000000 | الحد الأقصى لعدد الأسطح في النموذج الناتج، استخدم -1 لعدم وجود حد (الافتراضي: -1) |
| `حجم النسيج` | INT | لا | 128 إلى 4096 | حجم القوام الناتج بالبكسل (الافتراضي: 4096) |
| `تنسيق النسيج` | COMBO | لا | BMP<br>DPX<br>HDR<br>JPEG<br>OPEN_EXR<br>PNG<br>TARGA<br>TIFF<br>WEBP | صيغة القوام المصدرة (الافتراضي: JPEG) |
| `فرض التماثل` | BOOLEAN | لا | True/False | هل سيتم فرض التماثل على النموذج (الافتراضي: False) |
| `تسوية القاع` | BOOLEAN | لا | True/False | هل سيتم تسوية الجزء السفلي من النموذج (الافتراضي: False) |
| `عتبة تسوية القاع` | FLOAT | لا | 0.0 إلى 1.0 | عتبة تسوية الجزء السفلي (الافتراضي: 0.0) |
| `محور إلى مركز القاع` | BOOLEAN | لا | True/False | هل سيتم نقل نقطة الارتكاز إلى منتصف الجزء السفلي من النموذج (الافتراضي: False) |
| `عامل المقياس` | FLOAT | لا | 0.0 فما فوق | عامل القياس المطبق على النموذج (الافتراضي: 1.0) |
| `مع الحركة` | BOOLEAN | لا | True/False | هل سيتم تضمين بيانات الرسوم المتحركة في التصدير (الافتراضي: False) |
| `تجميع UV` | BOOLEAN | لا | True/False | هل سيتم حزم إحداثيات UV (الافتراضي: False) |
| `خبز` | BOOLEAN | لا | True/False | هل سيتم خبز القوام (الافتراضي: False) |
| `أسماء الأجزاء` | STRING | لا | قائمة مفصولة بفواصل | قائمة بأسماء الأجزاء المراد تضمينها في التصدير، مفصولة بفواصل (الافتراضي: "") |
| `إعداد FBX` | COMBO | لا | blender<br>mixamo<br>3dsmax | الإعداد المسبق لتصدير FBX المراد استخدامه (الافتراضي: blender) |
| `تصدير ألوان الرؤوس` | BOOLEAN | لا | True/False | هل سيتم تصدير ألوان الرؤوس (الافتراضي: False) |
| `تصدير الاتجاه` | COMBO | لا | align_image<br>default | وضع اتجاه التصدير (الافتراضي: default) |
| `تحريك في المكان` | BOOLEAN | لا | True/False | هل سيتم تحريك النموذج في مكانه (الافتراضي: False) |

**ملاحظة:** يجب أن يكون `original_model_task_id` معرف مهمة صالحًا من عملية Tripo سابقة (توليد نموذج، أو تجهيزه بالهيكل العظمي، أو إعادة توجيهه). المعاملات الموسومة بأنها "متقدمة" اختيارية ولا تحتاج إلى تكوين إلا لمتطلبات تصدير محددة.
