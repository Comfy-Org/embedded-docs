Вот перевод документации узла GITSScheduler на русский язык:

Узел GITSScheduler генерирует сигмы расписания шума для метода сэмплирования GITS (Generative Iterative Time Steps — генеративные итеративные временные шаги). Он вычисляет значения сигмы на основе коэффициента и количества шагов, с возможным фактором денойзинга, который может уменьшить общее количество используемых шагов. Узел использует предопределённые уровни шума и интерполяцию для создания итогового расписания сигм.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `coeff` | FLOAT | Да | 0.80 - 1.50 | Значение коэффициента, управляющее кривой расписания шума (по умолчанию: 1.20) |
| `steps` | INT | Да | 2 - 1000 | Общее количество шагов сэмплирования, для которых генерируются сигмы (по умолчанию: 10) |
| `denoise` | FLOAT | Да | 0.0 - 1.0 | Фактор денойзинга, уменьшающий количество используемых шагов (по умолчанию: 1.0) |

**Примечание:** Если `denoise` установлен в 0.0, узел возвращает пустой тензор. Если `denoise` меньше 1.0, фактическое количество используемых шагов вычисляется как `round(steps * denoise)`. Для шагов больше 20 узел использует логарифмически-линейную интерполяцию для расширения предопределённых уровней шума до нужного количества шагов.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `sigmas` | SIGMAS | Сгенерированные значения сигмы для расписания шума |
