### **WanHuMoImageToVideo**

گره WanHuMoImageToVideo با تولید نمایش‌های نهفته (latent) برای فریم‌های ویدیو، تصاویر را به دنباله‌های ویدیویی تبدیل می‌کند. این گره ورودی‌های شرطی‌سازی (conditioning) را پردازش می‌کند و می‌تواند از تصاویر مرجع و جاسازی‌های صوتی (audio embeddings) برای تأثیرگذاری بر تولید ویدیو استفاده کند. خروجی این گره، داده‌های شرطی‌سازی اصلاح‌شده و نمایش‌های نهفته مناسب برای سنتز ویدیو است.

## **ورودی‌ها**

| پارامتر | نوع داده | ضروری | محدوده | توضیحات |
| :--- | :--- | :--- | :--- | :--- |
| `مثبت` | CONDITIONING | بله | - | ورودی شرطی‌سازی مثبت که تولید ویدیو را به سمت محتوای دلخواه هدایت می‌کند |
| `منفی` | CONDITIONING | بله | - | ورودی شرطی‌سازی منفی که تولید ویدیو را از محتوای ناخواسته دور می‌کند |
| `vae` | VAE | بله | - | مدل VAE مورد استفاده برای رمزگذاری تصاویر مرجع به فضای نهفته |
| `عرض` | INT | بله | 16 تا MAX\_RESOLUTION | عرض فریم‌های ویدیوی خروجی بر حسب پیکسل (پیش‌فرض: 832، باید بر 16 بخش‌پذیر باشد) |
| `ارتفاع` | INT | بله | 16 تا MAX\_RESOLUTION | ارتفاع فریم‌های ویدیوی خروجی بر حسب پیکسل (پیش‌فرض: 480، باید بر 16 بخش‌پذیر باشد) |
| `طول` | INT | بله | 1 تا MAX\_RESOLUTION | تعداد فریم‌ها در دنباله ویدیوی تولید شده (پیش‌فرض: 97، باید به گونه‌ای باشد که (length - 1) بر 4 بخش‌پذیر باشد) |
| `اندازه دسته` | INT | بله | 1 تا 4096 | تعداد دنباله‌های ویدیویی که به طور همزمان تولید می‌شوند (پیش‌فرض: 1) |
| `خروجی رمزگذار صوتی` | AUDIOENCODEROUTPUT | خیر | - | داده‌های رمزگذاری شده صوتی اختیاری که می‌توانند بر اساس محتوای صوتی بر تولید ویدیو تأثیر بگذارند |
| `تصویر مرجع` | IMAGE | خیر | - | تصویر مرجع اختیاری که برای هدایت سبک و محتوای تولید ویدیو استفاده می‌شود |

**توجه:** هنگامی که یک تصویر مرجع ارائه می‌شود، رمزگذاری شده و به هر دو شرطی‌سازی مثبت و منفی اضافه می‌شود. هنگامی که خروجی رمزگذار صوتی ارائه می‌شود، پردازش شده و در داده‌های شرطی‌سازی ادغام می‌گردد. اگر هیچکدام ارائه نشود، تنسورهای placeholder پر از صفر برای هر دو نمایش نهفته مرجع و جاسازی‌های صوتی استفاده می‌شوند.
