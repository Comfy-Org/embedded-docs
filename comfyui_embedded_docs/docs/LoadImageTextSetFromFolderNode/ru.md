Загружает пакет изображений и соответствующие им текстовые подписи из указанной директории для целей обучения. Узел автоматически ищет файлы изображений и связанные с ними файлы текстовых подписей, обрабатывает изображения в соответствии с заданными настройками изменения размера и кодирует подписи с помощью предоставленной модели CLIP.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `folder` | STRING | Да | - | Папка для загрузки изображений. |
| `clip` | CLIP | Да | - | Модель CLIP, используемая для кодирования текста. |
| `resize_method` | COMBO | Нет | "None"<br>"Stretch"<br>"Crop"<br>"Pad" | Метод изменения размера изображений (по умолчанию: "None"). |
| `width` | INT | Нет | от -1 до 10000 | Ширина для изменения размера изображений. -1 означает использование исходной ширины (по умолчанию: -1). |
| `height` | INT | Нет | от -1 до 10000 | Высота для изменения размера изображений. -1 означает использование исходной высоты (по умолчанию: -1). |

**Примечание:** Входной параметр CLIP должен быть действительным и не может быть None. Если модель CLIP получена из узла загрузки контрольной точки, убедитесь, что контрольная точка содержит действительную модель CLIP или модель текстового кодировщика.

**Примечание о структуре папок:** Узел поддерживает структуру папок kohya-ss/sd-scripts. Если имя подпапки начинается с числа, за которым следует символ подчеркивания (например, `5_myclass`), это число используется как количество повторений, и изображения внутри этой подпапки будут загружены соответствующее количество раз.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `IMAGE` | IMAGE | Пакет загруженных и обработанных изображений. |
| `CONDITIONING` | CONDITIONING | Закодированные данные conditioning из текстовых подписей. |
