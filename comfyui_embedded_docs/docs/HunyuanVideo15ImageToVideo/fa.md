گره HunyuanVideo15ImageToVideo داده‌های شرطی (conditioning) و فضای نهفته (latent space) را برای تولید ویدئو بر اساس مدل HunyuanVideo 1.5 آماده می‌کند. این گره یک نمایش نهفته اولیه برای یک دنباله ویدئویی ایجاد می‌کند و می‌تواند به‌صورت اختیاری یک تصویر شروع یا خروجی CLIP vision را برای هدایت فرآیند تولید ادغام کند.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|---------|----------|--------|--------|---------|
| `positive` | CONDITIONING | بله | - | پرامپت‌های شرطی مثبت که محتوای مورد نظر ویدئو را توصیف می‌کنند. |
| `negative` | CONDITIONING | بله | - | پرامپت‌های شرطی منفی که محتوای نامطلوب ویدئو را توصیف می‌کنند. |
| `vae` | VAE | بله | - | مدل VAE (خودرمزگذار متغیر) که برای رمزگذاری تصویر شروع به فضای نهفته استفاده می‌شود. |
| `width` | INT | خیر | 16 تا MAX_RESOLUTION | عرض فریم‌های ویدئوی خروجی بر حسب پیکسل. باید بر 16 بخش‌پذیر باشد. (پیش‌فرض: 848) |
| `height` | INT | خیر | 16 تا MAX_RESOLUTION | ارتفاع فریم‌های ویدئوی خروجی بر حسب پیکسل. باید بر 16 بخش‌پذیر باشد. (پیش‌فرض: 480) |
| `length` | INT | خیر | 1 تا MAX_RESOLUTION | تعداد کل فریم‌های دنباله ویدئو. باید مضربی از 4 باشد. (پیش‌فرض: 33) |
| `batch_size` | INT | خیر | 1 تا 4096 | تعداد دنباله‌های ویدئویی که در یک دسته تولید می‌شوند. (پیش‌فرض: 1) |
| `start_image` | IMAGE | خیر | - | یک تصویر شروع اختیاری برای مقداردهی اولیه تولید ویدئو. در صورت ارائه، رمزگذاری شده و برای شرطی‌سازی فریم‌های اولیه استفاده می‌شود. تنها تعداد `length` فریم اول تصویر استفاده می‌شود. |
| `clip_vision_output` | CLIP_VISION_OUTPUT | خیر | - | بردارهای تعبیه‌شده (embeddings) اختیاری CLIP vision برای ارائه شرطی‌سازی بصری اضافی برای تولید. |

**نکته:** هنگامی که `start_image` ارائه می‌شود، به‌طور خودکار با استفاده از درون‌یابی دوخطی (bilinear interpolation) به اندازه `width` و `height` مشخص‌شده تغییر اندازه می‌دهد. تعداد `length` فریم اول از دسته تصاویر استفاده می‌شود. سپس تصویر رمزگذاری‌شده به هر دو شرط `positive` و `negative` به‌عنوان یک `concat_latent_image` با یک `concat_mask` متناظر اضافه می‌شود. ماسک برای فریم‌های تحت پوشش تصویر شروع روی 0.0 و برای فریم‌های باقی‌مانده روی 1.0 تنظیم می‌شود.

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
|-----------|----------|---------|
| `positive` | CONDITIONING | شرطی‌سازی مثبت اصلاح‌شده که اکنون ممکن است شامل تصویر شروع رمزگذاری‌شده یا خروجی CLIP vision باشد. |
| `negative` | CONDITIONING | شرطی‌سازی منفی اصلاح‌شده که اکنون ممکن است شامل تصویر شروع رمزگذاری‌شده یا خروجی CLIP vision باشد. |
| `latent` | LATENT | یک تنسور نهفته خالی با ابعاد پیکربندی‌شده برای اندازه دسته، طول ویدئو، عرض و ارتفاع مشخص‌شده. |
