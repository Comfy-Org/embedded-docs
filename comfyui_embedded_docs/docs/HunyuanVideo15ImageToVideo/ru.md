Вот перевод документации на русский язык:

Узел HunyuanVideo15ImageToVideo подготавливает данные conditioning и латентного пространства для генерации видео на основе модели HunyuanVideo 1.5. Он создает начальное латентное представление для видеопоследовательности и может опционально интегрировать начальное изображение или выходные данные CLIP vision для управления процессом генерации.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `positive` | CONDITIONING | Да | - | Положительные подсказки conditioning, описывающие, что должно содержаться в видео. |
| `negative` | CONDITIONING | Да | - | Отрицательные подсказки conditioning, описывающие, чего следует избегать в видео. |
| `vae` | VAE | Да | - | Модель VAE (вариационного автоэнкодера), используемая для кодирования начального изображения в латентное пространство. |
| `width` | INT | Нет | от 16 до MAX_RESOLUTION | Ширина выходных видеокадров в пикселях. Должна делиться на 16. (по умолчанию: 848) |
| `height` | INT | Нет | от 16 до MAX_RESOLUTION | Высота выходных видеокадров в пикселях. Должна делиться на 16. (по умолчанию: 480) |
| `length` | INT | Нет | от 1 до MAX_RESOLUTION | Общее количество кадров в видеопоследовательности. Должно быть кратно 4. (по умолчанию: 33) |
| `batch_size` | INT | Нет | от 1 до 4096 | Количество видеопоследовательностей для генерации в одном пакете. (по умолчанию: 1) |
| `start_image` | IMAGE | Нет | - | Необязательное начальное изображение для инициализации генерации видео. Если указано, оно кодируется и используется для управления первыми кадрами. Используются только первые `length` кадров изображения. |
| `clip_vision_output` | CLIP_VISION_OUTPUT | Нет | - | Необязательные эмбеддинги CLIP vision для обеспечения дополнительного визуального управления генерацией. |

**Примечание:** При указании `start_image` оно автоматически масштабируется до заданных значений `width` и `height` с помощью билинейной интерполяции. Используются первые `length` кадров пакета изображений. Затем закодированное изображение добавляется как к `positive`, так и к `negative` conditioning в виде `concat_latent_image` с соответствующим `concat_mask`. Маска устанавливается в 0.0 для кадров, охваченных начальным изображением, и в 1.0 для остальных кадров.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `positive` | CONDITIONING | Модифицированное положительное conditioning, которое теперь может включать закодированное начальное изображение или выходные данные CLIP vision. |
| `negative` | CONDITIONING | Модифицированное отрицательное conditioning, которое теперь может включать закодированное начальное изображение или выходные данные CLIP vision. |
| `latent` | LATENT | Пустой латентный тензор с размерами, настроенными под указанные размер пакета, длину видео, ширину и высоту. |
