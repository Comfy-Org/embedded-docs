Вот перевод документации на русский язык:

Этот узел был добавлен для поддержки модели Alibaba Wan Fun Control для генерации видео и был добавлен после [этого коммита](https://github.com/comfyanonymous/ComfyUI/commit/3661c833bcc41b788a7c9f0e7bc48524f8ee5f82).

- **Назначение:** Подготовка информации о кондиционировании, необходимой для генерации видео, с использованием модели Wan 2.1 Fun Control.

Узел WanFunControlToVideo является дополнением ComfyUI, предназначенным для поддержки моделей Wan Fun Control для генерации видео, направленным на использование WanFun control для создания видео.

Этот узел служит точкой подготовки важной информации о кондиционировании и инициализирует центральную точку латентного пространства, направляя последующий процесс генерации видео с помощью модели Wan 2.1 Fun. Название узла четко указывает на его функцию: он принимает различные входные данные и преобразует их в формат, подходящий для управления генерацией видео в рамках WanFun.

Положение узла в иерархии узлов ComfyUI указывает на то, что он работает на ранних этапах конвейера генерации видео, фокусируясь на манипулировании сигналами кондиционирования до фактической семплирования или декодирования видеокадров.

## Входные данные

| Имя параметра      | Обязательный | Тип данных          | Описание                                                  | Значение по умолчанию |
|:-------------------|:-------------|:--------------------|:----------------------------------------------------------|:----------------------|
| positive           | Да           | CONDITIONING        | Стандартные данные положительного кондиционирования ComfyUI, обычно из узла "CLIP Text Encode". Положительный промпт описывает контент, тематику и художественный стиль, которые пользователь представляет для генерируемого видео. | Н/Д |
| negative           | Да           | CONDITIONING        | Стандартные данные отрицательного кондиционирования ComfyUI, обычно генерируемые узлом "CLIP Text Encode". Отрицательный промпт указывает элементы, стили или артефакты, которых пользователь хочет избежать в генерируемом видео. | Н/Д |
| vae                | Да           | VAE                 | Требуется модель VAE (вариационного автоэнкодера), совместимая с семейством моделей Wan 2.1 Fun, используемая для кодирования и декодирования данных изображений/видео. | Н/Д |
| width              | Да           | INT                 | Желаемая ширина выходных видеокадров в пикселях, со значением по умолчанию 832, минимальным значением 16, максимальным значением, определяемым nodes.MAX_RESOLUTION, и шагом 16. | 832 |
| height             | Да           | INT                 | Желаемая высота выходных видеокадров в пикселях, со значением по умолчанию 480, минимальным значением 16, максимальным значением, определяемым nodes.MAX_RESOLUTION, и шагом 16. | 480 |
| length             | Да           | INT                 | Общее количество кадров в генерируемом видео, со значением по умолчанию 81, минимальным значением 1, максимальным значением, определяемым nodes.MAX_RESOLUTION, и шагом 4. | 81 |
| batch_size         | Да           | INT                 | Количество видео, генерируемых за один пакет, со значением по умолчанию 1, минимальным значением 1 и максимальным значением 4096. | 1 |
| clip_vision_output | Нет          | CLIP_VISION_OUTPUT  | (Необязательно) Визуальные признаки, извлеченные моделью CLIP vision, позволяющие управлять визуальным стилем и содержанием. | Нет |
| start_image        | Нет          | IMAGE               | (Необязательно) Начальное изображение, влияющее на начало генерируемого видео. | Нет |
| control_video      | Нет          | IMAGE               | (Необязательно) Позволяет пользователям предоставить предварительно обработанное эталонное видео ControlNet, которое будет направлять движение и потенциальную структуру генерируемого видео. | Нет |

## Выходные данные

| Имя параметра      | Тип данных          | Описание                                                  |
|:-------------------|:--------------------|:----------------------------------------------------------|
| positive           | CONDITIONING        | Предоставляет расширенные данные положительного кондиционирования, включая закодированные start_image и control_video. |
| negative           | CONDITIONING        | Предоставляет данные отрицательного кондиционирования, которые также были расширены и содержат тот же concat_latent_image. |
| latent             | LATENT              | Словарь, содержащий пустой латентный тензор с ключом "samples". |
