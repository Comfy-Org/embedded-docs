### مرور کلی

گره Meshy: Rig Model یک مدل سه‌بعدی حاصل از وظایف پیشین Meshy را دریافت کرده و به‌طور خودکار برای آن اسکلت‌بندی ایجاد می‌کند. خروجی این گره یک کاراکتر اسکلت‌بندی‌شده است که قابلیت حالت‌دهی (Posing) و انیمیشن‌سازی (Animation) را دارد. این گره مدل اسکلت‌بندی‌شده را در دو فرمت فایل GLB و FBX خروجی می‌دهد.

## ورودی‌ها

| پارامتر | نوع داده | ضروری | محدوده | توضیحات |
|---------|----------|-------|--------|---------|
| `meshy_task_id` | STRING | بله | نامشخص | شناسه یکتای وظیفه از یک عملیات قبلی Meshy (مانند تبدیل متن به سه‌بعدی یا تصویر به سه‌بعدی) که مدل مورد نظر برای اسکلت‌بندی را تولید کرده است. |
| `height_meters` | FLOAT | بله | 0.1 تا 15.0 | ارتفاع تقریبی مدل کاراکتر بر حسب متر. این مقدار به دقت مقیاس‌گذاری و اسکلت‌بندی کمک می‌کند (پیش‌فرض: 1.7). |
| `texture_image` | IMAGE | خیر | نامشخص | تصویر بافت رنگ پایه (Base Color) مدل که UV آن باز شده است. |

**توجه:** فرآیند اسکلت‌بندی خودکار در حال حاضر برای مش‌های بدون بافت (untextured meshes)، دارایی‌های غیرانسان‌نما (non-humanoid assets)، یا دارایی‌های انسان‌نمایی که ساختار اندام و بدن نامشخصی دارند، مناسب نیست.

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
|-----------|----------|---------|
| `model_file` | STRING | یک خروجی قدیمی برای سازگاری با نسخه‌های قبل، که شامل نام فایل مدل GLB است. |
| `rig_task_id` | STRING | شناسه یکتای وظیفه برای این عملیات اسکلت‌بندی که می‌توان از آن برای ارجاع به نتیجه استفاده کرد. |
| `GLB` | FILE3DGLB | مدل کاراکتر سه‌بعدی اسکلت‌بندی‌شده ذخیره‌شده در قالب فایل GLB. |
| `FBX` | FILE3DFBX | مدل کاراکتر سه‌بعدی اسکلت‌بندی‌شده ذخیره‌شده در قالب فایل FBX. |
