Вот перевод документации на русский язык:

Узел Meshy: Rig Model принимает 3D-модель из предыдущей задачи Meshy и автоматически создает для неё скелет, формируя ригованную модель персонажа, которую можно позировать и анимировать. Узел выводит ригованную модель в форматах GLB и FBX.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `meshy_task_id` | STRING | Да | Н/Д | Уникальный идентификатор задачи из предыдущей операции Meshy (например, текст-в-3D или изображение-в-3D), в результате которой была создана модель для риггинга. |
| `height_meters` | FLOAT | Да | 0,1–15,0 | Примерная высота модели персонажа в метрах. Влияет на точность масштабирования и риггинга (по умолчанию: 1,7). |
| `texture_image` | IMAGE | Нет | Н/Д | Текстура базового цвета модели с UV-развёрткой. |

**Примечание:** Процесс автоматического риггинга в настоящее время не подходит для моделей без текстур, негуманоидных объектов, а также гуманоидных объектов с нечёткой структурой конечностей и тела.

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `model_file` | STRING | Устаревший выход для обратной совместимости, содержащий имя файла GLB-модели. |
| `rig_task_id` | STRING | Уникальный идентификатор задачи для данной операции риггинга, который можно использовать для ссылки на результат. |
| `GLB` | FILE3DGLB | Ригованная 3D-модель персонажа, сохранённая в формате GLB. |
| `FBX` | FILE3DFBX | Ригованная 3D-модель персонажа, сохранённая в формате FBX. |
