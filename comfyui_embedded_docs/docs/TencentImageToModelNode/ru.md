Вот перевод документации на русский язык:

Этот узел использует API Tencent Hunyuan3D Pro для генерации 3D-модели из одного или нескольких входных изображений. Он обрабатывает изображения, отправляет их в API и возвращает сгенерированные файлы 3D-моделей в форматах GLB и OBJ, а также дополнительные карты текстур.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `модель` | COMBO | Да | `"3.0"`<br>`"3.1"` | Версия модели Hunyuan3D для использования. Опция LowPoly недоступна для модели `3.1`. |
| `изображение` | IMAGE | Да | - | Основное входное изображение, используемое для генерации 3D-модели. Должно быть не менее 128x128 пикселей. |
| `изображение слева` | IMAGE | Нет | - | Дополнительное изображение левой стороны объекта для многовидовой генерации. Должно быть не менее 128x128 пикселей. |
| `изображение справа` | IMAGE | Нет | - | Дополнительное изображение правой стороны объекта для многовидовой генерации. Должно быть не менее 128x128 пикселей. |
| `изображение сзади` | IMAGE | Нет | - | Дополнительное изображение задней стороны объекта для многовидовой генерации. Должно быть не менее 128x128 пикселей. |
| `количество граней` | INT | Да | 3000 - 1500000 | Целевое количество полигонов для сгенерированной 3D-модели (по умолчанию: 500000). |
| `тип генерации` | DYNAMICCOMBO | Да | `"Normal"`<br>`"LowPoly"`<br>`"Geometry"` | Тип генерируемой 3D-модели. Выбор опции открывает дополнительные связанные параметры. |
| `generate_type.pbr` | BOOLEAN | Нет | - | Включает генерацию материалов на основе физически корректного рендеринга (PBR). Этот параметр виден только когда `тип генерации` установлен в "Normal" или "LowPoly" (по умолчанию: False). |
| `generate_type.polygon_type` | COMBO | Нет | `"triangle"`<br>`"quadrilateral"` | Тип полигонов для построения сетки. Этот параметр виден только когда `тип генерации` установлен в "LowPoly". |
| `seed` | INT | Да | 0 - 2147483647 | Значение зерна для процесса генерации. Зерно определяет, должен ли узел перезапускаться; результаты недетерминированы независимо от зерна (по умолчанию: 0). |

**Примечание:** Все входные изображения должны иметь минимальную ширину и высоту 128 пикселей. Изображения автоматически уменьшаются, если их самая длинная сторона превышает 4900 пикселей.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `model_file` | STRING | Устаревший выход для обратной совместимости. |
| `GLB` | FILE3DGLB | Сгенерированная 3D-модель в формате файла GLB (Binary GL Transmission Format). |
| `OBJ` | FILE3DOBJ | Сгенерированная 3D-модель в формате файла OBJ (Wavefront). |
| `texture_image` | IMAGE | Текстурное изображение для сгенерированной 3D-модели. |
| `optional_metallic` | IMAGE | Карта металличности для PBR-материалов. Возвращает черное изображение, если недоступна. |
| `optional_normal` | IMAGE | Карта нормалей для PBR-материалов. Возвращает черное изображение, если недоступна. |
| `optional_roughness` | IMAGE | Карта шероховатости для PBR-материалов. Возвращает черное изображение, если недоступна. |
