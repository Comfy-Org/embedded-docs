Вот перевод документации на русский язык:

Сохраняет SVG-файлы на диск. Этот узел принимает SVG-данные на входе и сохраняет их в вашу выходную директорию с возможностью встраивания метаданных. Узел автоматически обрабатывает именование файлов с суффиксами-счетчиками и может встраивать информацию о рабочем процессе непосредственно в SVG-файл.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `svg` | SVG | Да | - | SVG-данные для сохранения на диск |
| `префикс_имени_файла` | STRING | Да | - | Префикс для сохраняемого файла. Может включать информацию форматирования, например %date:yyyy-MM-dd% или %Empty Latent Image.width%, для подстановки значений из узлов. (по умолчанию: "svg/ComfyUI") |

## Выходы

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `ui` | DICT | Возвращает информацию о файле, включая имя файла, подпапку и тип для отображения в интерфейсе ComfyUI |

**Примечание:** Этот узел автоматически встраивает метаданные рабочего процесса (prompt и дополнительную PNG-информацию) в SVG-файл при их наличии. Метаданные вставляются в виде раздела CDATA внутри элемента metadata SVG.
