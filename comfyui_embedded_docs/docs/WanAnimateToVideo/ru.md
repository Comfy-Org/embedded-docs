Вот перевод документации на русский язык:

Узел WanAnimateToVideo генерирует видеоконтент путем объединения множества входных данных conditioning, включая ссылки на позы, выражения лица и фоновые элементы. Он обрабатывает различные видеовходы для создания связных анимированных последовательностей, сохраняя временную согласованность между кадрами. Узел выполняет операции в латентном пространстве и может расширять существующие видео, продолжая шаблоны движения.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `positive` | CONDITIONING | Да | - | Положительное conditioning для направления генерации к желаемому контенту |
| `negative` | CONDITIONING | Да | - | Отрицательное conditioning для отвода генерации от нежелательного контента |
| `vae` | VAE | Да | - | Модель VAE, используемая для кодирования и декодирования данных изображений |
| `width` | INT | Да | 16 до MAX_RESOLUTION | Ширина выходного видео в пикселях (по умолчанию: 832, шаг: 16) |
| `height` | INT | Да | 16 до MAX_RESOLUTION | Высота выходного видео в пикселях (по умолчанию: 480, шаг: 16) |
| `length` | INT | Да | 1 до MAX_RESOLUTION | Количество кадров для генерации (по умолчанию: 77, шаг: 4) |
| `batch_size` | INT | Да | 1 до 4096 | Количество видео для одновременной генерации (по умолчанию: 1) |
| `clip_vision_output` | CLIP_VISION_OUTPUT | Нет | - | Необязательный выход модели CLIP vision для дополнительного conditioning |
| `reference_image` | IMAGE | Нет | - | Эталонное изображение, используемое в качестве отправной точки для генерации |
| `face_video` | IMAGE | Нет | - | Видеовход, предоставляющий направляющую информацию о выражении лица |
| `pose_video` | IMAGE | Нет | - | Видеовход, предоставляющий направляющую информацию о позе и движении |
| `continue_motion_max_frames` | INT | Да | 1 до MAX_RESOLUTION | Максимальное количество кадров для продолжения из предыдущего движения (по умолчанию: 5, шаг: 4) |
| `background_video` | IMAGE | Нет | - | Фоновое видео для компоновки с сгенерированным контентом |
| `character_mask` | MASK | Нет | - | Маска, определяющая области персонажа для выборочной обработки |
| `continue_motion` | IMAGE | Нет | - | Предыдущая последовательность движения для продолжения с целью сохранения временной согласованности |
| `video_frame_offset` | INT | Да | 0 до MAX_RESOLUTION | Количество кадров для смещения во всех входных видео. Используется для генерации более длинных видео по частям. Подключите к выходу `video_frame_offset` предыдущего узла для расширения видео. (по умолчанию: 0, шаг: 1) |

**Ограничения параметров:**

- Если указан `pose_video`, длина выходного видео будет скорректирована в соответствии с длительностью видео позы, если активна логика `trim_to_pose_video` (в настоящее время установлена в `False` в исходном коде)
- `face_video` автоматически изменяется до разрешения 512x512 и нормализуется до диапазона от -1.0 до 1.0 при обработке
- Кадры `continue_motion` ограничены параметром `continue_motion_max_frames`; используются только последние `continue_motion_max_frames` кадров из входных данных
- Входные видео (`face_video`, `pose_video`, `background_video`, `character_mask`) смещаются на `video_frame_offset` перед обработкой; если смещение превышает длину видео, входные данные игнорируются
- Если `character_mask` содержит только один кадр, он будет повторен для всех кадров
- Если указан `clip_vision_output`, он применяется как к положительному, так и к отрицательному conditioning
- Если `reference_image` не указан, в качестве эталонного изображения по умолчанию используется черное изображение (все нули)
- Если `continue_motion` не указан, начальные кадры заполняются серым шумом (интенсивность 0.5)

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `positive` | CONDITIONING | Модифицированное положительное conditioning с дополнительным видеоконтекстом, включая выход CLIP vision, латентное представление видео позы, пиксели видео лица, конкатенированное латентное изображение и конкатенированную маску |
| `negative` | CONDITIONING | Модифицированное отрицательное conditioning с дополнительным видеоконтекстом, включая выход CLIP vision, латентное представление видео позы, пиксели видео лица (инвертированные), конкатенированное латентное изображение и конкатенированную маску |
| `latent` | LATENT | Сгенерированное видеосодержимое в формате латентного пространства с формой [batch_size, 16, latent_length + trim_latent, latent_height, latent_width] |
| `trim_latent` | INT | Информация об обрезке латентного пространства, указывающая количество латентных кадров для обрезки с начала (соответствует латентным кадрам эталонного изображения) |
| `trim_image` | INT | Информация об обрезке в пространстве изображений для эталонных кадров движения, указывающая количество кадров изображения для обрезки с начала |
| `video_frame_offset` | INT | Обновленное смещение кадров для продолжения генерации видео по частям, рассчитываемое как предыдущее смещение плюс сгенерированная длина |
