بالتأكيد، إليك الترجمة العربية للوثيقة التقنية لعقدة WanAnimateToVideo مع الالتزام التام بقواعد الترجمة المحددة.

---

عقدة WanAnimateToVideo تقوم بإنشاء محتوى فيديو من خلال دمج مدخلات تكييف متعددة تشمل مراجع الوضعية، وتعابير الوجه، وعناصر الخلفية. تقوم بمعالجة مدخلات فيديو متنوعة لإنشاء تسلسلات متحركة متماسكة مع الحفاظ على الاستمرارية الزمنية عبر الإطارات. تتعامل العقدة مع عمليات الفضاء الكامن ويمكنها تمديد مقاطع الفيديو الموجودة من خلال استمرار أنماط الحركة.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|---|---|---|---|---|
| `positive` | CONDITIONING | نعم | - | تكييف إيجابي لتوجيه عملية التوليد نحو المحتوى المطلوب |
| `negative` | CONDITIONING | نعم | - | تكييف سلبي لتوجيه عملية التوليد بعيدًا عن المحتوى غير المرغوب فيه |
| `vae` | VAE | نعم | - | نموذج VAE المستخدم لتشفير وفك تشفير بيانات الصورة |
| `width` | INT | نعم | 16 إلى MAX_RESOLUTION | عرض فيديو الإخراج بالبكسل (الافتراضي: 832، الخطوة: 16) |
| `height` | INT | نعم | 16 إلى MAX_RESOLUTION | ارتفاع فيديو الإخراج بالبكسل (الافتراضي: 480، الخطوة: 16) |
| `length` | INT | نعم | 1 إلى MAX_RESOLUTION | عدد الإطارات المراد توليدها (الافتراضي: 77، الخطوة: 4) |
| `batch_size` | INT | نعم | 1 إلى 4096 | عدد مقاطع الفيديو المراد توليدها في وقت واحد (الافتراضي: 1) |
| `clip_vision_output` | CLIP_VISION_OUTPUT | لا | - | مخرجات نموذج رؤية CLIP الاختيارية لتكييف إضافي |
| `reference_image` | IMAGE | لا | - | صورة مرجعية تُستخدم كنقطة بداية للتوليد |
| `face_video` | IMAGE | لا | - | مدخل فيديو يوفر توجيهًا لتعابير الوجه |
| `pose_video` | IMAGE | لا | - | مدخل فيديو يوفر توجيهًا للوضعية والحركة |
| `continue_motion_max_frames` | INT | نعم | 1 إلى MAX_RESOLUTION | الحد الأقصى لعدد الإطارات المراد الاستمرار منها من الحركة السابقة (الافتراضي: 5، الخطوة: 4) |
| `background_video` | IMAGE | لا | - | فيديو خلفية لدمجه مع المحتوى المُنشأ |
| `character_mask` | MASK | لا | - | قناع يحدد مناطق الشخصية للمعالجة الانتقائية |
| `continue_motion` | IMAGE | لا | - | تسلسل الحركة السابق للاستمرار منه لتحقيق الاستمرارية الزمنية |
| `video_frame_offset` | INT | نعم | 0 إلى MAX_RESOLUTION | مقدار الإطارات المراد تخطيها في جميع فيديوهات الإدخال. يُستخدم لتوليد مقاطع فيديو أطول عن طريق التقسيم. قم بتوصيله بمخرج video_frame_offset من العقدة السابقة لتمديد الفيديو. (الافتراضي: 0، الخطوة: 1) |

**قيود المعاملات:**

- عند توفير `pose_video`، سيتم تعديل طول الإخراج ليتناسب مع مدة فيديو الوضعية إذا كان منطق `trim_to_pose_video` نشطًا (مضبوط حاليًا على `False` في الكود المصدري)
- يتم تغيير حجم `face_video` تلقائيًا إلى دقة 512×512 بكسل وتطبيعه إلى نطاق من -1.0 إلى 1.0 عند معالجته
- إطارات `continue_motion` محدودة بمعامل `continue_motion_max_frames`؛ يتم استخدام آخر `continue_motion_max_frames` إطارًا فقط من الإدخال
- يتم إزاحة فيديوهات الإدخال (`face_video`, `pose_video`, `background_video`, `character_mask`) بمقدار `video_frame_offset` قبل المعالجة؛ إذا تجاوز الإزاحة طول الفيديو، يتم تجاهل الإدخال
- إذا كان `character_mask` يحتوي على إطار واحد فقط، فسيتم تكراره عبر جميع الإطارات
- عند توفير `clip_vision_output`، يتم تطبيقه على كل من التكييف الإيجابي والسلبي
- إذا لم يتم توفير `reference_image`، يتم استخدام صورة سوداء (جميع القيم صفر) كمرجع افتراضي
- إذا لم يتم توفير `continue_motion`، يتم ملء الإطارات الأولية بضوضاء رمادية (كثافة 0.5)

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|---|---|---|
| `positive` | CONDITIONING | تكييف إيجابي معدّل مع سياق فيديو إضافي يتضمن مخرجات رؤية CLIP، وكامن فيديو الوضعية، وبكسلات فيديو الوجه، وصورة كامنة متسلسلة، وقناع متسلسل |
| `negative` | CONDITIONING | تكييف سلبي معدّل مع سياق فيديو إضافي يتضمن مخرجات رؤية CLIP، وكامن فيديو الوضعية، وبكسلات فيديو الوجه (مقلوبة)، وصورة كامنة متسلسلة، وقناع متسلسل |
| `latent` | LATENT | محتوى الفيديو المُنشأ بتنسيق الفضاء الكامن ذو الشكل [batch_size, 16, latent_length + trim_latent, latent_height, latent_width] |
| `trim_latent` | INT | معلومات تقليم الفضاء الكامن تشير إلى عدد الإطارات الكامنة المراد قصها من البداية (تتوافق مع الإطارات الكامنة للصورة المرجعية) |
| `trim_image` | INT | معلومات تقليم مساحة الصورة لإطارات الحركة المرجعية، تشير إلى عدد إطارات الصورة المراد قصها من البداية |
| `video_frame_offset` | INT | إزاحة الإطارات المحدثة لمواصلة توليد الفيديو على شكل أجزاء، محسوبة على أنها الإزاحة السابقة بالإضافة إلى الطول المُنشأ |
