# بهبود ویدئو Topaz نسخه ۲

گره **Topaz Video Enhance V2** به شما امکان می‌دهد ویدئو را با استفاده از مدل‌های هوش مصنوعی Topaz Labs بزرگ‌نمایی و بهبود دهید. این گره می‌تواند وضوح تصویر را افزایش دهد، نرخ فریم را از طریق درون‌یابی تنظیم کند و بهبودهای خلاقانه یا واقع‌گرایانه برای جان بخشیدن به فیلم‌های شما اعمال کند.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|---------|----------|--------|--------|---------|
| `video` | VIDEO | بله | - | ویدئوی ورودی برای پردازش. باید در قالب کانتینر MP4 باشد. |
| `upscaler_model` | COMBO | بله | `"Astra 2"`<br>`"Starlight (Astra) Fast"`<br>`"Starlight (Astra) Creative"`<br>`"Starlight Precise 2.5"`<br>`"Disabled"` | مدل هوش مصنوعی مورد استفاده برای بزرگ‌نمایی ویدئو. انتخاب "Disabled" به معنای عدم اعمال بزرگ‌نمایی است. |
| `upscaler_model.upscaler_resolution` | COMBO | مشروط | `"FullHD (1080p)"`<br>`"4K (2160p)"` | وضوح خروجی هدف برای بزرگ‌نمایی. این پارامتر زمانی که یک مدل بزرگ‌نمایی انتخاب شده (نه "Disabled") الزامی است. |
| `upscaler_model.creativity` | FLOAT / COMBO | مشروط | Astra 2: 0.0 تا 1.0 (گام 0.1)<br>Starlight Creative: `"low"`<br>`"middle"`<br>`"high"` | قدرت خلاقیت در بزرگ‌نمایی. فقط برای مدل‌های "Astra 2" و "Starlight (Astra) Creative" در دسترس است. برای Astra 2، یک لغزنده است (پیش‌فرض: 0.5). برای Starlight Creative، یک منوی ترکیبی است (پیش‌فرض: "low"). |
| `upscaler_model.prompt` | STRING | خیر | - | توضیح اختیاری صحنه (نه دستوری). فقط برای مدل "Astra 2" در دسترس است. در صورت تنظیم، به ۵۰۰ فریم ورودی (~۱۵ ثانیه در ۳۰ فریم بر ثانیه) محدود می‌شود. پیش‌فرض: خالی. |
| `upscaler_model.sharp` | FLOAT | خیر | 0.0 تا 1.0 (گام 0.01) | وضوح پیش‌بهبود: 0.0=محو گوسی، 0.5=عبور مستقیم (پیش‌فرض)، 1.0=تیزکنی USM. فقط برای مدل "Astra 2" در دسترس است. پیش‌فرض: 0.5. |
| `upscaler_model.realism` | FLOAT | خیر | 0.0 تا 1.0 (گام 0.01) | خروجی را به سمت واقع‌گرایی عکاسی سوق می‌دهد. برای پیش‌فرض مدل روی 0 بگذارید. فقط برای مدل "Astra 2" در دسترس است. پیش‌فرض: 0.0. |
| `interpolation_model` | COMBO | بله | `"Disabled"`<br>`"apo-8"` | مدل هوش مصنوعی مورد استفاده برای درون‌یابی فریم. انتخاب "Disabled" به معنای عدم اعمال درون‌یابی است. |
| `interpolation_model.interpolation_frame_rate` | INT | مشروط | 15 تا 240 | نرخ فریم خروجی. زمانی که مدل درون‌یابی "apo-8" باشد الزامی است. پیش‌فرض: 60. |
| `interpolation_model.interpolation_slowmo` | INT | خیر | 1 تا 16 | ضریب حرکت آهسته اعمال شده بر ویدئوی ورودی. به عنوان مثال، 2 باعث می‌شود خروجی دو برابر کندتر و مدت زمان آن دو برابر شود. پیش‌فرض: 1. |
| `interpolation_model.interpolation_duplicate` | BOOLEAN | خیر | True/False | ورودی را برای فریم‌های تکراری تحلیل کرده و آن‌ها را حذف می‌کند. پیش‌فرض: False. |
| `interpolation_model.interpolation_duplicate_threshold` | FLOAT | خیر | 0.001 تا 0.1 (گام 0.001) | حساسیت تشخیص فریم‌های تکراری. پیش‌فرض: 0.01. |
| `dynamic_compression_level` | COMBO | خیر | `"Low"`<br>`"Mid"`<br>`"High"` | سطح CQP برای فشرده‌سازی ویدئو. پیش‌فرض: "Low". |

**محدودیت‌های مهم:**
- حداقل یکی از `upscaler_model` یا `interpolation_model` باید فعال باشد (نه "Disabled")، در غیر این صورت خطا ایجاد می‌شود.
- ویدئوی ورودی باید در قالب کانتینر MP4 باشد.
- مدل "Astra 2" با prompt به ۵۰۰ فریم ورودی (~۱۵ ثانیه در ۳۰ فریم بر ثانیه) محدود می‌شود. بدون prompt، به تعداد فریم بیشتری محدود می‌شود.
- زمانی که `upscaler_model` "Disabled" نیست، زیرپارامتر `upscaler_resolution` الزامی است.
- زمانی که `interpolation_model` "Disabled" نیست، زیرپارامتر `interpolation_frame_rate` الزامی است.
