# Topaz Video Enhance V2

Узел **Topaz Video Enhance V2** позволяет масштабировать и улучшать видео с помощью моделей искусственного интеллекта Topaz Labs. Он может увеличивать разрешение, регулировать частоту кадров через интерполяцию, а также применять творческие или реалистичные улучшения, чтобы вдохнуть новую жизнь в ваши видеоматериалы.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `video` | VIDEO | Да | - | Входное видео для обработки. Должно быть в формате контейнера MP4. |
| `upscaler_model` | COMBO | Да | `"Astra 2"`<br>`"Starlight (Astra) Fast"`<br>`"Starlight (Astra) Creative"`<br>`"Starlight Precise 2.5"`<br>`"Disabled"` | Модель ИИ, используемая для масштабирования видео. Выбор "Disabled" означает, что масштабирование применяться не будет. |
| `upscaler_model.upscaler_resolution` | COMBO | Условно | `"FullHD (1080p)"`<br>`"4K (2160p)"` | Целевое выходное разрешение для масштабатора. Этот параметр обязателен, когда выбрана модель масштабирования (не "Disabled"). |
| `upscaler_model.creativity` | FLOAT | Условно | 0.0 до 1.0 (шаг 0.1) | Творческая сила масштабирования. Доступно только для моделей "Astra 2" и "Starlight (Astra) Creative". По умолчанию: 0.5 для Astra 2, "low" для Starlight Creative. |
| `upscaler_model.prompt` | STRING | Нет | - | Необязательный описательный (не инструктивный) текстовый запрос сцены. Доступен только для модели "Astra 2". При установке ограничен 500 входными кадрами (~15с при 30fps). По умолчанию: пусто. |
| `upscaler_model.sharp` | FLOAT | Нет | 0.0 до 1.0 (шаг 0.01) | Предварительная резкость: 0.0=размытие по Гауссу, 0.5=сквозной пропуск (по умолчанию), 1.0=повышение резкости USM. Доступно только для модели "Astra 2". По умолчанию: 0.5. |
| `upscaler_model.realism` | FLOAT | Нет | 0.0 до 1.0 (шаг 0.01) | Приближает результат к фотографическому реализму. Оставьте 0 для значения по умолчанию модели. Доступно только для модели "Astra 2". По умолчанию: 0.0. |
| `interpolation_model` | COMBO | Да | `"Disabled"`<br>`"apo-8"` | Модель ИИ, используемая для интерполяции кадров. Выбор "Disabled" означает, что интерполяция применяться не будет. |
| `interpolation_model.interpolation_frame_rate` | INT | Условно | 15 до 240 | Выходная частота кадров. Требуется, когда модель интерполяции — "apo-8". По умолчанию: 60. |
| `interpolation_model.interpolation_slowmo` | INT | Нет | 1 до 16 | Коэффициент замедления, применяемый к входному видео. Например, 2 делает выходное видео в два раза медленнее и удваивает длительность. По умолчанию: 1. |
| `interpolation_model.interpolation_duplicate` | BOOLEAN | Нет | True/False | Анализировать входное видео на наличие дублирующихся кадров и удалять их. По умолчанию: False. |
| `interpolation_model.interpolation_duplicate_threshold` | FLOAT | Нет | 0.001 до 0.1 (шаг 0.001) | Чувствительность обнаружения дублирующихся кадров. По умолчанию: 0.01. |
| `dynamic_compression_level` | COMBO | Нет | `"Low"`<br>`"Mid"`<br>`"High"` | Уровень CQP для сжатия видео. По умолчанию: "Low". |

**Важные ограничения:**
- Как минимум один из параметров `upscaler_model` или `interpolation_model` должен быть включен (не "Disabled"), иначе будет выдана ошибка.
- Входное видео должно быть в формате контейнера MP4.
- Модель "Astra 2" с текстовым запросом ограничена 500 входными кадрами (~15 секунд при 30fps). Без запроса ограничение на количество кадров выше.
- Когда `upscaler_model` не равен "Disabled", подпараметр `upscaler_resolution` обязателен.
- Когда `interpolation_model` не равен "Disabled", подпараметр `interpolation_frame_rate` обязателен.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `video` | VIDEO | Улучшенное выходное видео после применения выбранных фильтров масштабирования и/или интерполяции. |
