Вот перевод документации на русский язык:

Узел TripoTextureNode генерирует текстурированные 3D-модели с использованием Tripo API. Он принимает идентификатор задачи модели и применяет генерацию текстур с различными опциями, включая PBR-материалы, настройки качества текстур и методы выравнивания. Узел взаимодействует с Tripo API для обработки запроса на генерацию текстур и возвращает результирующий файл модели и идентификатор задачи.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `model_task_id` | MODEL_TASK_ID | Да | - | Идентификатор задачи модели, к которой необходимо применить текстуры |
| `texture` | BOOLEAN | Нет | - | Создавать ли текстуры (по умолчанию: True) |
| `pbr` | BOOLEAN | Нет | - | Создавать ли PBR-материалы (физически корректный рендеринг) (по умолчанию: True) |
| `texture_seed` | INT | Нет | - | Случайное зерно для генерации текстур (по умолчанию: 42) |
| `texture_quality` | COMBO | Нет | "standard"<br>"detailed" | Уровень качества генерации текстур (по умолчанию: "standard"). Опция "detailed" стоит $0.20 USD, а "standard" — $0.10 USD. |
| `texture_alignment` | COMBO | Нет | "original_image"<br>"geometry" | Метод выравнивания текстур (по умолчанию: "original_image"). "original_image" выравнивает текстуры по исходному входному изображению, а "geometry" — по 3D-геометрии. |

*Примечание: Для работы этого узла требуются токены аутентификации и ключи API, которые обрабатываются системой автоматически.*

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `model_file` | STRING | Сгенерированный файл модели с наложенными текстурами (только для обратной совместимости) |
| `model task_id` | MODEL_TASK_ID | Идентификатор задачи для отслеживания процесса генерации текстур |
| `GLB` | FILE3DGLB | Сгенерированная 3D-модель в формате GLB с наложенными текстурами |
