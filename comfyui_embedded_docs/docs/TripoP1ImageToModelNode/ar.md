## نظرة عامة

يحوِّل هذا العقد صورة ثنائية الأبعاد واحدة إلى نموذج ثلاثي الأبعاد باستخدام واجهة برمجة تطبيقات Tripo P1. وهو مُحسَّن لتوليد شبكات منخفضة المضلعات جاهزة للألعاب.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `image` | IMAGE | نعم | - | الصورة المدخلة المراد تحويلها إلى نموذج ثلاثي الأبعاد. |
| `output_mode` | DICT | نعم | انظر الوصف | قاموس يحدد وضع الإخراج وإعدادات الجودة. يتحكم هذا المعامل في نوع النموذج المُنشأ وجودة نسيجه. تُعرَّف الخيارات المتاحة بواسطة دالة المساعدة `_build_p1_output_mode` وتتضمن إعدادات `texture_quality` (مثل "standard" و"high" و"ultra") و`image_alignment`. |
| `enable_image_autofix` | BOOLEAN | لا | True<br>False | معالجة مسبقة للصورة المدخلة لتحسين جودة التوليد. (الافتراضي: False) |
| `face_limit` | INT | لا | - | يحدد عدد الأوجه في الشبكة المُنشأة. القيمة -1 تعني عدم وجود حد. (الافتراضي: -1) |
| `model_seed` | INT | لا | - | قيمة بذرة لتوليد نموذج قابل للتكرار. إذا لم تُقدم، تُستخدم بذرة عشوائية. (الافتراضي: None) |
| `auto_size` | BOOLEAN | لا | True<br>False | تحديد الحجم الأمثل للنموذج المُنشأ تلقائياً. (الافتراضي: False) |
| `export_uv` | BOOLEAN | لا | True<br>False | تصدير إحداثيات UV مع النموذج. (الافتراضي: True) |
| `compress_geometry` | BOOLEAN | لا | True<br>False | ضغط بيانات الهندسة لتقليل حجم الملف. (الافتراضي: False) |

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|-----------|-------------|
| `model_file` | STRING | مسار الملف للنموذج ثلاثي الأبعاد المُنشأ. يُقدم هذا المخرج للتوافق مع الإصدارات السابقة فقط. |
| `model task_id` | MODEL_TASK_ID | المعرف الفريد لمهمة طلب توليد النموذج. |
| `GLB` | FILE3DGLB | النموذج ثلاثي الأبعاد المُنشأ بصيغة GLB. |
