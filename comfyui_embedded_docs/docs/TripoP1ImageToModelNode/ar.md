## نظرة عامة

يحوِّل هذا العقد صورة ثنائية الأبعاد واحدة إلى نموذج ثلاثي الأبعاد باستخدام واجهة برمجة تطبيقات Tripo P1. وهو مُحسَّن لتوليد شبكات منخفضة المضلعات جاهزة للألعاب.

## المدخلات

| المعامل | الوصف | نوع البيانات | إلزامي | النطاق |
| --- | --- | --- | --- | --- |
| `image` | الصورة المدخلة المراد تحويلها إلى نموذج ثلاثي الأبعاد. | IMAGE | نعم | - |
| `output_mode` | قاموس يحدد وضع الإخراج وإعدادات الجودة. يتحكم هذا المعامل في نوع النموذج المُنشأ وجودة نسيجه. تُعرَّف الخيارات المتاحة بواسطة دالة المساعدة `_build_p1_output_mode` وتتضمن إعدادات `texture_quality` (مثل "standard" و"high" و"ultra") و`image_alignment`. | DICT | نعم | انظر الوصف |
| `enable_image_autofix` | معالجة مسبقة للصورة المدخلة لتحسين جودة التوليد. (الافتراضي: False) | BOOLEAN | لا | True<br>False |
| `face_limit` | يحدد عدد الأوجه في الشبكة المُنشأة. القيمة -1 تعني عدم وجود حد. (الافتراضي: -1) | INT | لا | - |
| `model_seed` | قيمة بذرة لتوليد نموذج قابل للتكرار. إذا لم تُقدم، تُستخدم بذرة عشوائية. (الافتراضي: None) | INT | لا | - |
| `auto_size` | تحديد الحجم الأمثل للنموذج المُنشأ تلقائياً. (الافتراضي: False) | BOOLEAN | لا | True<br>False |
| `export_uv` | تصدير إحداثيات UV مع النموذج. (الافتراضي: True) | BOOLEAN | لا | True<br>False |
| `compress_geometry` | ضغط بيانات الهندسة لتقليل حجم الملف. (الافتراضي: False) | BOOLEAN | لا | True<br>False |

## المخرجات

| اسم المخرج | الوصف | نوع البيانات |
| --- | --- | --- |
| `model_file` | مسار الملف للنموذج ثلاثي الأبعاد المُنشأ. يُقدم هذا المخرج للتوافق مع الإصدارات السابقة فقط. | STRING |
| `model task_id` | المعرف الفريد لمهمة طلب توليد النموذج. | MODEL_TASK_ID |
| `GLB` | النموذج ثلاثي الأبعاد المُنشأ بصيغة GLB. | FILE3DGLB |
