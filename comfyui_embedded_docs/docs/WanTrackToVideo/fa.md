### مرور کلی

گره WanTrackToVideo داده‌های ردیابی حرکت را با پردازش نقاط ردیابی و تولید فریم‌های ویدئویی متناظر، به دنباله‌های ویدئویی تبدیل می‌کند. این گره مختصات ردیابی را به عنوان ورودی دریافت کرده و خروجی‌های شرطی‌سازی ویدئو و بازنمایی‌های نهفته (latent) را تولید می‌کند که می‌توان از آنها برای تولید ویدئو استفاده کرد. زمانی که هیچ ردیابی ارائه نشود، این گره به حالت استاندارد تبدیل تصویر به ویدئو بازمی‌گردد.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|-----------|-----------|----------|-------|-------------|
| `مثبت` | CONDITIONING | بله | - | شرطی‌سازی مثبت برای تولید ویدئو |
| `منفی` | CONDITIONING | بله | - | شرطی‌سازی منفی برای تولید ویدئو |
| `vae` | VAE | بله | - | مدل VAE برای رمزگذاری و رمزگشایی |
| `ترک‌ها` | STRING | بله | - | داده‌های ردیابی با فرمت JSON به صورت رشته چندخطی (پیش‌فرض: "[]") |
| `عرض` | INT | بله | 16 تا MAX_RESOLUTION | عرض ویدئوی خروجی بر حسب پیکسل (پیش‌فرض: 832، گام: 16) |
| `ارتفاع` | INT | بله | 16 تا MAX_RESOLUTION | ارتفاع ویدئوی خروجی بر حسب پیکسل (پیش‌فرض: 480، گام: 16) |
| `طول` | INT | بله | 1 تا MAX_RESOLUTION | تعداد فریم‌های ویدئوی خروجی (پیش‌فرض: 81، گام: 4) |
| `اندازه بچ` | INT | بله | 1 تا 4096 | تعداد ویدئوهایی که به طور همزمان تولید می‌شوند (پیش‌فرض: 1) |
| `دمای تصادفی‌سازی` | FLOAT | بله | 1.0 تا 1000.0 | پارامتر دما برای وصله‌گذاری حرکت (پیش‌فرض: 220.0، گام: 0.1) |
| `topk` | INT | بله | 1 تا 10 | مقدار top-k برای وصله‌گذاری حرکت (پیش‌فرض: 2) |
| `تصویر شروع` | IMAGE | خیر | - | تصویر شروع برای تولید ویدئو |
| `خروجی clip vision` | CLIPVISIONOUTPUT | خیر | - | خروجی vision از CLIP برای شرطی‌سازی اضافی |

**نکته:** زمانی که `tracks` حاوی داده‌های ردیابی معتبر است، گره ردیابی‌های حرکت را پردازش کرده و ویدئو تولید می‌کند. زمانی که `tracks` خالی است، گره به حالت استاندارد تصویر به ویدئو تغییر وضعیت می‌دهد. اگر `start_image` ارائه شود، اولین فریم دنباله ویدئویی مقداردهی اولیه می‌شود.
