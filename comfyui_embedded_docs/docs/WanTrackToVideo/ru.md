Вот перевод документации на русский язык:

Узел WanTrackToVideo преобразует данные отслеживания движения в видео последовательности путем обработки точек трека и генерации соответствующих видеокадров. Он принимает координаты отслеживания на входе и создает видеокондиционирование и латентные представления, которые можно использовать для генерации видео. Если треки не предоставлены, узел переключается на стандартное преобразование изображения в видео.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `positive` | CONDITIONING | Да | - | Положительное кондиционирование для генерации видео |
| `negative` | CONDITIONING | Да | - | Отрицательное кондиционирование для генерации видео |
| `vae` | VAE | Да | - | Модель VAE для кодирования и декодирования |
| `tracks` | STRING | Да | - | Данные отслеживания в формате JSON в виде многострочной строки (по умолчанию: "[]") |
| `width` | INT | Да | 16 до MAX_RESOLUTION | Ширина выходного видео в пикселях (по умолчанию: 832, шаг: 16) |
| `height` | INT | Да | 16 до MAX_RESOLUTION | Высота выходного видео в пикселях (по умолчанию: 480, шаг: 16) |
| `length` | INT | Да | 1 до MAX_RESOLUTION | Количество кадров в выходном видео (по умолчанию: 81, шаг: 4) |
| `batch_size` | INT | Да | 1 до 4096 | Количество видео для одновременной генерации (по умолчанию: 1) |
| `temperature` | FLOAT | Да | 1.0 до 1000.0 | Параметр температуры для подбора движения (по умолчанию: 220.0, шаг: 0.1) |
| `topk` | INT | Да | 1 до 10 | Значение top-k для подбора движения (по умолчанию: 2) |
| `start_image` | IMAGE | Нет | - | Начальное изображение для генерации видео |
| `clip_vision_output` | CLIPVISIONOUTPUT | Нет | - | Выходные данные CLIP vision для дополнительного кондиционирования |

**Примечание:** Если `tracks` содержит корректные данные отслеживания, узел обрабатывает треки движения для генерации видео. Если `tracks` пуст, узел переключается в стандартный режим преобразования изображения в видео. Если предоставлен `start_image`, он инициализирует первый кадр видео последовательности.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `positive` | CONDITIONING | Положительное кондиционирование с примененной информацией о треках движения |
| `negative` | CONDITIONING | Отрицательное кондиционирование с примененной информацией о треках движения |
| `latent` | LATENT | Сгенерированное латентное представление видео |
