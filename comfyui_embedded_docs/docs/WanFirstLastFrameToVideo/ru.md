Вот перевод документации на русский язык:

Узел WanFirstLastFrameToVideo создаёт видеокондиционирование путём объединения начального и конечного кадров с текстовыми подсказками. Он генерирует латентное представление для генерации видео, кодируя первый и последний кадры, применяя маски для управления процессом генерации и используя признаки CLIP vision, когда они доступны. Этот узел подготавливает как положительное, так и отрицательное кондиционирование для видеомоделей, чтобы генерировать связные последовательности между заданными начальной и конечной точками.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `positive` | CONDITIONING | Да | - | Положительное текстовое кондиционирование для управления генерацией видео |
| `negative` | CONDITIONING | Да | - | Отрицательное текстовое кондиционирование для управления генерацией видео |
| `vae` | VAE | Да | - | Модель VAE, используемая для кодирования изображений в латентное пространство |
| `width` | INT | Да | от 16 до MAX_RESOLUTION | Ширина выходного видео (по умолчанию: 832, шаг: 16) |
| `height` | INT | Да | от 16 до MAX_RESOLUTION | Высота выходного видео (по умолчанию: 480, шаг: 16) |
| `length` | INT | Да | от 1 до MAX_RESOLUTION | Количество кадров в видеопоследовательности (по умолчанию: 81, шаг: 4) |
| `batch_size` | INT | Да | от 1 до 4096 | Количество видео для одновременной генерации (по умолчанию: 1) |
| `clip_vision_start_image` | CLIP_VISION_OUTPUT | Нет | - | Признаки CLIP vision, извлечённые из начального изображения |
| `clip_vision_end_image` | CLIP_VISION_OUTPUT | Нет | - | Признаки CLIP vision, извлечённые из конечного изображения |
| `start_image` | IMAGE | Нет | - | Изображение начального кадра видеопоследовательности |
| `end_image` | IMAGE | Нет | - | Изображение конечного кадра видеопоследовательности |

**Примечание:** Если указаны оба изображения `start_image` и `end_image`, узел создаёт видеопоследовательность, которая плавно переходит между этими двумя кадрами. Параметры `clip_vision_start_image` и `clip_vision_end_image` являются необязательными, но если они указаны, их признаки CLIP vision объединяются и применяются как к положительному, так и к отрицательному кондиционированию. Изображение `start_image` обрезается до первых `length` кадров, а изображение `end_image` обрезается до последних `length` кадров перед обработкой.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `positive` | CONDITIONING | Положительное кондиционирование с применённым кодированием видеокадров и признаками CLIP vision |
| `negative` | CONDITIONING | Отрицательное кондиционирование с применённым кодированием видеокадров и признаками CLIP vision |
| `latent` | LATENT | Пустой латентный тензор с размерами, соответствующими заданным параметрам видео |
