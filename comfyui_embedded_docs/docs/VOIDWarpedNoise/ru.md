## Обзор

Генерирует временно-коррелированный шум для второго прохода процесса уточнения видео VOID. Принимает выходное видео с первого прохода и искажает гауссов шум вдоль векторов оптического потока, создавая шум, который движется согласованно с содержимым видео. Этот искаженный шум используется в качестве начального латентного представления для второго прохода, что улучшает временную согласованность в конечном результате.

## Входные данные

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `optical_flow` | MODEL | Да | - | Модель оптического потока из OpticalFlowLoader (RAFT-large). |
| `video` | IMAGE | Да | - | Кадры выходного видео с первого прохода [T, H, W, 3]. |
| `width` | INT | Да | от 16 до MAX_RESOLUTION (шаг 8) | Ширина выходного латентного представления (по умолчанию: 672). |
| `height` | INT | Да | от 16 до MAX_RESOLUTION (шаг 8) | Высота выходного латентного представления (по умолчанию: 384). |
| `length` | INT | Да | от 1 до MAX_RESOLUTION (шаг 1) | Количество пиксельных кадров. Округляется вниз для получения четного `latent_t` (требование patch_size_t=2), например, 49 → 45 (по умолчанию: 45). |
| `batch_size` | INT | Да | от 1 до 64 | Количество идентичных последовательностей шума для генерации (по умолчанию: 1). |

**Примечание по параметру `length`:** Значение `length` автоматически округляется вниз до ближайшего допустимого значения, которое дает четное измерение `latent_t`. Это требуется ограничением `patch_size_t=2` модели CogVideoX-Fun-V1.5. При округлении в журнал записывается предупреждение.

## Выходные данные

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `warped_noise` | LATENT | 5D тензор (B, C, T, H, W), содержащий гауссов шум, искаженный оптическим потоком, готовый к использованию в качестве начального латентного представления во втором проходе VOID. |
