Узел CLIPTextEncodeHiDream обрабатывает четыре отдельных текстовых входа с использованием различных языковых моделей (CLIP-L, CLIP-G, T5-XXL и LLaMA) и объединяет их в единый выходной кондиционирующий сигнал. Он токенизирует каждый текстовый вход с соответствующей моделью и кодирует их вместе с помощью метода запланированного кодирования, что позволяет реализовать более сложное текстовое кондиционирование за счет одновременного использования нескольких языковых моделей.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `clip` | CLIP | Да | - | Модель CLIP, используемая для токенизации и кодирования |
| `clip_l` | STRING | Да | - | Текстовый вход для обработки моделью CLIP-L. Поддерживает многострочный текст и динамические подсказки. |
| `clip_g` | STRING | Да | - | Текстовый вход для обработки моделью CLIP-G. Поддерживает многострочный текст и динамические подсказки. |
| `t5xxl` | STRING | Да | - | Текстовый вход для обработки моделью T5-XXL. Поддерживает многострочный текст и динамические подсказки. |
| `llama` | STRING | Да | - | Текстовый вход для обработки моделью LLaMA. Поддерживает многострочный текст и динамические подсказки. |

**Примечание:** Все четыре текстовых входа (`clip_l`, `clip_g`, `t5xxl` и `llama`) обязательны для корректной работы, так как каждый из них вносит вклад в итоговый выходной кондиционирующий сигнал через процесс запланированного кодирования.

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | Объединенный кондиционирующий сигнал от всех обработанных текстовых входов, закодированный с использованием метода запланированного кодирования |
