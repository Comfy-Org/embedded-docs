گره CLIPTextEncodeHiDream چهار ورودی متنی مجزا را با استفاده از مدل‌های زبانی مختلف (CLIP-L، CLIP-G، T5-XXL و LLaMA) پردازش کرده و آن‌ها را در یک خروجی شرطی‌سازی واحد ترکیب می‌کند. این گره هر ورودی متنی را با مدل متناظر آن توکن‌سازی کرده و با استفاده از رویکرد رمزگذاری زمان‌بندی‌شده، آن‌ها را با یکدیگر رمزگذاری می‌کند. این کار با بهره‌گیری هم‌زمان از چندین مدل زبانی، شرطی‌سازی متنی پیچیده‌تری را امکان‌پذیر می‌سازد.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|---------|----------|--------|--------|---------|
| `clip` | CLIP | بله | - | مدل CLIP مورد استفاده برای توکن‌سازی و رمزگذاری |
| `clip_l` | STRING | بله | - | ورودی متنی برای پردازش مدل CLIP-L. از متن چندخطی و پرامپت‌های پویا پشتیبانی می‌کند. |
| `clip_g` | STRING | بله | - | ورودی متنی برای پردازش مدل CLIP-G. از متن چندخطی و پرامپت‌های پویا پشتیبانی می‌کند. |
| `t5xxl` | STRING | بله | - | ورودی متنی برای پردازش مدل T5-XXL. از متن چندخطی و پرامپت‌های پویا پشتیبانی می‌کند. |
| `llama` | STRING | بله | - | ورودی متنی برای پردازش مدل LLaMA. از متن چندخطی و پرامپت‌های پویا پشتیبانی می‌کند. |

**توجه:** هر چهار ورودی متنی (`clip_l`، `clip_g`، `t5xxl` و `llama`) برای عملکرد صحیح الزامی هستند، زیرا هر یک از طریق فرآیند رمزگذاری زمان‌بندی‌شده به خروجی نهایی شرطی‌سازی کمک می‌کنند.
