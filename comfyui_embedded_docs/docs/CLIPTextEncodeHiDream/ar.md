عقدة CLIPTextEncodeHiDream تعالج أربعة مدخلات نصية منفصلة باستخدام نماذج لغوية مختلفة (CLIP-L وCLIP-G وT5-XXL وLLaMA) وتدمجها في مخرج توجيه واحد. تقوم بتقطيع كل مدخل نصي باستخدام النموذج المقابل له وتشفيرها معًا باستخدام نهج التشفير المجدول، مما يتيح توجيهًا نصيًا أكثر تطورًا من خلال الاستفادة من نماذج لغوية متعددة في وقت واحد.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `clip` | CLIP | نعم | - | نموذج CLIP المستخدم في التقطيع والتشفير |
| `clip_l` | STRING | نعم | - | المدخل النصي لمعالجة نموذج CLIP-L. يدعم النص متعدد الأسطر والموجهات الديناميكية. |
| `clip_g` | STRING | نعم | - | المدخل النصي لمعالجة نموذج CLIP-G. يدعم النص متعدد الأسطر والموجهات الديناميكية. |
| `t5xxl` | STRING | نعم | - | المدخل النصي لمعالجة نموذج T5-XXL. يدعم النص متعدد الأسطر والموجهات الديناميكية. |
| `llama` | STRING | نعم | - | المدخل النصي لمعالجة نموذج LLaMA. يدعم النص متعدد الأسطر والموجهات الديناميكية. |

**ملاحظة:** جميع المدخلات النصية الأربعة (`clip_l` و`clip_g` و`t5xxl` و`llama`) إلزامية لضمان الأداء السليم، حيث يساهم كل منها في مخرج التوجيه النهائي من خلال عملية التشفير المجدول.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | مخرج التوجيه المدمج من جميع المدخلات النصية المعالجة، والمشفّر باستخدام طريقة التشفير المجدول |
