Вот перевод документации на русский язык:

Узел WanDancerVideo подготавливает данные conditioning и пустой латентный тензор для генерации видео с помощью модели WanDancer. Он объединяет позитивный и негативный conditioning с опциональными входными данными, такими как начальное изображение, маска, эмбеддинги CLIP vision и аудио-характеристики, для управления генерируемым видео.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `positive` | CONDITIONING | Да | | Позитивный conditioning для направления генерации видео. |
| `negative` | CONDITIONING | Да | | Негативный conditioning для направления генерации видео. |
| `vae` | VAE | Да | | VAE, используемый для кодирования начального изображения в латентное пространство. |
| `width` | INT | Да | от 16 до MAX_RESOLUTION (шаг: 16) | Ширина генерируемого видео в пикселях (по умолчанию: 480). |
| `height` | INT | Да | от 16 до MAX_RESOLUTION (шаг: 16) | Высота генерируемого видео в пикселях (по умолчанию: 832). |
| `length` | INT | Да | от 1 до MAX_RESOLUTION (шаг: 4) | Количество кадров в генерируемом видео. Должно оставаться 149 для WanDancer (по умолчанию: 149). |
| `clip_vision_output` | CLIP_VISION_OUTPUT | Нет | | Эмбеддинги CLIP vision для первого кадра. |
| `clip_vision_output_ref` | CLIP_VISION_OUTPUT | Нет | | Эмбеддинги CLIP vision для референсного изображения. |
| `start_image` | IMAGE | Нет | | Начальное изображение(я) для кодирования. Может быть любым количеством кадров, вплоть до указанного `length`. |
| `mask` | MASK | Нет | | Маска conditioning изображения для начального изображения(ий). Белые области сохраняются, черные области генерируются. Используется для локальной генерации. |
| `audio_encoder_output` | AUDIO_ENCODER_OUTPUT | Нет | | Выходные данные аудиоэнкодера, предоставляющие аудио-характеристики, fps и масштаб внедрения для аудио-условной генерации. |

**Примечание об ограничениях параметров:**
- Входные параметры `start_image` и `mask` являются опциональными, но могут использоваться вместе. Когда `start_image` предоставлен, он кодируется и объединяется с латентным представлением. Если также предоставлен `mask`, он определяет, какие части начального изображения сохраняются (белые), а какие регенерируются (черные). Если `mask` не предоставлен, вся область начального изображения используется как направляющая conditioning.
- Входные параметры `clip_vision_output` и `clip_vision_output_ref` являются опциональными и могут использоваться вместе для предоставления визуального контекста для первого кадра и референсного изображения.
- Входной параметр `audio_encoder_output` является опциональным и предоставляет аудио-характеристики для аудио-условной генерации.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `positive` | CONDITIONING | Позитивный conditioning с прикрепленными дополнительными данными (concat latent, CLIP vision, аудио). |
| `negative` | CONDITIONING | Негативный conditioning с прикрепленными дополнительными данными (concat latent, CLIP vision, аудио). |
| `latent` | LATENT | Пустой латентный тензор с размерами, соответствующими указанной длине, высоте и ширине видео. |
