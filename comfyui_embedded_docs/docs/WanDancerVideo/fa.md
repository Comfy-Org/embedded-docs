گره WanDancerVideo داده‌های شرطی‌سازی (conditioning) و یک تنسور لاتنت خالی برای تولید ویدئو با مدل WanDancer آماده می‌کند. این گره شرطی‌سازی مثبت و منفی را با ورودی‌های اختیاری مانند تصویر شروع، ماسک، بردارهای嵌入 CLIP Vision و ویژگی‌های صوتی ترکیب می‌کند تا ویدئوی تولیدشده را کنترل کند.

## ورودی‌ها

| پارامتر | نوع داده | ضروری | محدوده | توضیحات |
|---------|----------|-------|--------|---------|
| `مثبت` | CONDITIONING | بله | | شرطی‌سازی مثبت برای هدایت تولید ویدئو. |
| `منفی` | CONDITIONING | بله | | شرطی‌سازی منفی برای هدایت تولید ویدئو. |
| `vae` | VAE | بله | | VAE مورد استفاده برای کدگذاری تصویر شروع به فضای لاتنت. |
| `عرض` | INT | بله | 16 تا MAX_RESOLUTION (گام: 16) | عرض ویدئوی تولیدشده بر حسب پیکسل (پیش‌فرض: 480). |
| `ارتفاع` | INT | بله | 16 تا MAX_RESOLUTION (گام: 16) | ارتفاع ویدئوی تولیدشده بر حسب پیکسل (پیش‌فرض: 832). |
| `طول` | INT | بله | 1 تا MAX_RESOLUTION (گام: 4) | تعداد فریم‌های ویدئوی تولیدشده. برای WanDancer باید 149 باقی بماند (پیش‌فرض: 149). |
| `خروجی بینایی clip` | CLIP_VISION_OUTPUT | خیر | | بردارهای嵌入 CLIP Vision برای فریم اول. |
| `خروجی مرجع بینایی clip` | CLIP_VISION_OUTPUT | خیر | | بردارهای嵌入 CLIP Vision برای تصویر مرجع. |
| `تصویر شروع` | IMAGE | خیر | | تصویر(های) اولیه برای کدگذاری. می‌تواند هر تعداد فریم، تا حداکثر `طول` مشخص‌شده باشد. |
| `ماسک` | MASK | خیر | | ماسک شرطی‌سازی تصویر برای تصویر(های) شروع. نواحی سفید حفظ می‌شوند، نواحی سیاه تولید می‌شوند. برای تولیدات موضعی استفاده می‌شود. |
| `خروجی رمزگذار صوتی` | AUDIO_ENCODER_OUTPUT | خیر | | خروجی از یک کدگذار صوتی که ویژگی‌های صوتی، fps و مقیاس تزریق را برای تولید شرطی‌شده با صدا فراهم می‌کند. |

**نکته درباره محدودیت‌های پارامتر:**
- ورودی‌های `start_image` و `mask` اختیاری هستند اما می‌توانند با هم استفاده شوند. وقتی `start_image` ارائه می‌شود، کدگذاری شده و با لاتنت ادغام می‌شود. اگر `mask` نیز ارائه شود، مشخص می‌کند کدام بخش‌های تصویر شروع حفظ شوند (سفید) و کدام بخش‌ها بازتولید شوند (سیاه). اگر `mask` ارائه نشود، کل ناحیه تصویر شروع به‌عنوان راهنمای شرطی‌سازی استفاده می‌شود.
- ورودی‌های `clip_vision_output` و `clip_vision_output_ref` اختیاری هستند و می‌توانند با هم برای ارائه زمینه بصری برای فریم اول و یک تصویر مرجع استفاده شوند.
- ورودی `audio_encoder_output` اختیاری است و ویژگی‌های صوتی را برای تولید شرطی‌شده با صدا فراهم می‌کند.
