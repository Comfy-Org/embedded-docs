يقوم عقدة WanDancerVideo بإعداد بيانات التكييف (conditioning) وموتر كامن (latent tensor) فارغ لتوليد الفيديو باستخدام نموذج WanDancer. تجمع هذه العقدة بين التكييف الإيجابي والسلبي مع مدخلات اختيارية مثل صورة البداية، والقناع (mask)، وإخراج رؤية CLIP، وميزات الصوت للتحكم في الفيديو المُنشأ.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|---------|---------------|--------|---------|-------|
| `positive` | CONDITIONING | نعم | | التكييف الإيجابي لتوجيه توليد الفيديو. |
| `negative` | CONDITIONING | نعم | | التكييف السلبي لتوجيه توليد الفيديو. |
| `vae` | VAE | نعم | | VAE المستخدم لتشفير صورة البداية إلى الفضاء الكامن. |
| `width` | INT | نعم | 16 إلى MAX_RESOLUTION (الخطوة: 16) | عرض الفيديو المُنشأ بالبكسل (الافتراضي: 480). |
| `height` | INT | نعم | 16 إلى MAX_RESOLUTION (الخطوة: 16) | ارتفاع الفيديو المُنشأ بالبكسل (الافتراضي: 832). |
| `length` | INT | نعم | 1 إلى MAX_RESOLUTION (الخطوة: 4) | عدد الإطارات في الفيديو المُنشأ. يجب أن يبقى 149 لـ WanDancer (الافتراضي: 149). |
| `clip_vision_output` | CLIP_VISION_OUTPUT | لا | | تضمينات رؤية CLIP للإطار الأول. |
| `clip_vision_output_ref` | CLIP_VISION_OUTPUT | لا | | تضمينات رؤية CLIP للصورة المرجعية. |
| `start_image` | IMAGE | لا | | الصورة (الصور) الأولية المراد تشفيرها. يمكن أن تكون أي عدد من الإطارات، حتى الطول المحدد `length`. |
| `mask` | MASK | لا | | قناع تكييف الصورة للصورة (الصور) الأولية. يتم الاحتفاظ بالمناطق البيضاء، بينما يتم توليد المناطق السوداء. يُستخدم للتوليدات المحلية. |
| `audio_encoder_output` | AUDIO_ENCODER_OUTPUT | لا | | الإخراج من مشفر الصوت، الذي يوفر ميزات الصوت، ومعدل الإطارات (fps)، ومقياس الحقن (inject scale) للتوليد المشروط بالصوت. |

**ملاحظة حول قيود المعاملات:**
- مدخلات `start_image` و `mask` اختيارية ولكن يمكن استخدامها معًا. عند توفير `start_image`، يتم تشفيرها وربطها مع الكامن. إذا تم توفير `mask` أيضًا، فإنه يتحكم في أجزاء صورة البداية التي يتم الاحتفاظ بها (اللون الأبيض) وتلك التي يتم إعادة توليدها (اللون الأسود). إذا لم يتم توفير `mask`، يتم استخدام منطقة صورة البداية بأكملها كدليل تكييف.
- مدخلات `clip_vision_output` و `clip_vision_output_ref` اختيارية ويمكن استخدامها معًا لتوفير سياق بصري للإطار الأول وصورة مرجعية.
- مدخل `audio_encoder_output` اختياري ويوفر ميزات الصوت للتوليد المشروط بالصوت.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|---------------|--------|
| `positive` | CONDITIONING | التكييف الإيجابي مع أي بيانات إضافية مرفقة (الكامن المربوط، رؤية CLIP، الصوت). |
| `negative` | CONDITIONING | التكييف السلبي مع أي بيانات إضافية مرفقة (الكامن المربوط، رؤية CLIP، الصوت). |
| `latent` | LATENT | موتر كامن فارغ بأبعاد تطابق طول الفيديو وارتفاعه وعرضه المحددة. |
