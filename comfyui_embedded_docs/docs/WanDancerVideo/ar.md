تُعد عقدة WanDancerVideo بيانات التكييف (conditioning) وموترًا كامنًا (latent tensor) فارغًا لتوليد الفيديو باستخدام نموذج WanDancer. تجمع هذه العقدة بين التكييف الإيجابي والسلبي مع مدخلات اختيارية مثل الصورة الأولية والقناع (mask) وإخراج رؤية CLIP وميزات الصوت للتحكم في الفيديو المُنشأ.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|---------|--------------|--------|---------|-------|
| `positive` | CONDITIONING | نعم | | التكييف الإيجابي لتوجيه توليد الفيديو. |
| `negative` | CONDITIONING | نعم | | التكييف السلبي لتوجيه توليد الفيديو. |
| `vae` | VAE | نعم | | VAE المستخدم لتشفير الصورة الأولية إلى الفضاء الكامن. |
| `width` | INT | نعم | 16 إلى MAX_RESOLUTION (الخطوة: 16) | عرض الفيديو المُنشأ بالبكسل (الافتراضي: 480). |
| `height` | INT | نعم | 16 إلى MAX_RESOLUTION (الخطوة: 16) | ارتفاع الفيديو المُنشأ بالبكسل (الافتراضي: 832). |
| `length` | INT | نعم | 1 إلى MAX_RESOLUTION (الخطوة: 4) | عدد الإطارات في الفيديو المُنشأ. يجب أن يبقى 149 لـ WanDancer (الافتراضي: 149). |
| `clip_vision_output` | CLIP_VISION_OUTPUT | لا | | تضمينات رؤية CLIP للإطار الأول. |
| `clip_vision_output_ref` | CLIP_VISION_OUTPUT | لا | | تضمينات رؤية CLIP للصورة المرجعية. |
| `start_image` | IMAGE | لا | | الصورة (الصور) الأولية المراد تشفيرها. يمكن أن تكون أي عدد من الإطارات، حتى الطول المحدد `length`. |
| `mask` | MASK | لا | | قناع تكييف الصورة للصورة (الصور) الأولية. المناطق البيضاء تُحتفظ بها، والمناطق السوداء تُنشأ. يُستخدم للتوليد المحلي. |
| `audio_encoder_output` | AUDIO_ENCODER_OUTPUT | لا | | المخرجات من مشفر الصوت، والتي توفر ميزات الصوت ومعدل الإطارات (fps) ومقياس الحقن (inject scale) للتوليد المشروط بالصوت. |

**ملاحظة حول قيود المعاملات:**
- مدخلات `start_image` و `mask` اختيارية ولكن يمكن استخدامها معًا. عند توفير `start_image`، يتم تشفيرها ودمجها مع الكامن (latent). إذا تم توفير `mask` أيضًا، فإنه يتحكم في أجزاء الصورة الأولية التي يتم الاحتفاظ بها (الأبيض) وتلك التي يتم إعادة توليدها (الأسود). إذا لم يتم توفير `mask`، يتم استخدام منطقة الصورة الأولية بأكملها كدليل تكييف.
- مدخلات `clip_vision_output` و `clip_vision_output_ref` اختيارية ويمكن استخدامها معًا لتوفير سياق بصري للإطار الأول والصورة المرجعية.
- مدخل `audio_encoder_output` اختياري ويوفر ميزات الصوت للتوليد المشروط بالصوت.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|------------|--------------|--------|
| `positive` | CONDITIONING | التكييف الإيجابي مع أي بيانات إضافية مرفقة (الكامن المدمج، رؤية CLIP، الصوت). |
| `negative` | CONDITIONING | التكييف السلبي مع أي بيانات إضافية مرفقة (الكامن المدمج، رؤية CLIP، الصوت). |
| `latent` | LATENT | موتر كامن فارغ بأبعاد تتطابق مع طول الفيديو وارتفاعه وعرضه المحدد. |
