# ترجمة وثيقة CLIPTextEncodeSD3

عقدة CLIPTextEncodeSD3 تقوم بمعالجة مدخلات النص لنماذج Stable Diffusion 3 عن طريق ترميز مطالبات نصية متعددة باستخدام نماذج CLIP مختلفة. تتعامل العقدة مع ثلاثة مدخلات نصية منفصلة (clip_g و clip_l و t5xxl) وتوفر خيارات لإدارة حشو النص الفارغ. تضمن العقدة محاذاة صحيحة للرموز بين مدخلات النص المختلفة وتعيد بيانات التكييف المناسبة لخطوط إنتاج SD3.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|----------|---------------|--------|---------|---------|
| `clip` | CLIP | نعم | - | نموذج CLIP المستخدم لترميز النص |
| `clip_l` | STRING | نعم | - | مدخل النص لنموذج CLIP المحلي. يدعم النص متعدد الأسطر والمطالبات الديناميكية. |
| `clip_g` | STRING | نعم | - | مدخل النص لنموذج CLIP العام. يدعم النص متعدد الأسطر والمطالبات الديناميكية. |
| `t5xxl` | STRING | نعم | - | مدخل النص لنموذج T5-XXL. يدعم النص متعدد الأسطر والمطالبات الديناميكية. |
| `empty_padding` | COMBO | نعم | `"none"`<br>`"empty_prompt"` | يتحكم في كيفية معالجة مدخلات النص الفارغة. عند التعيين على "none"، ستؤدي مدخلات النص الفارغة لـ `clip_g` أو `clip_l` أو `t5xxl` إلى قوائم رموز فارغة بدلاً من الحشو. هذه معلمة متقدمة (القيمة الافتراضية: "none"). |

**قيود المعاملات:**

- عندما يتم تعيين `empty_padding` على "none"، فإن مدخلات النص الفارغة لـ `clip_g` أو `clip_l` أو `t5xxl` ستؤدي إلى قوائم رموز فارغة بدلاً من الحشو
- تقوم العقدة تلقائياً بموازنة أطوال الرموز بين مدخلات `clip_l` و `clip_g` عن طريق حشو المدخل الأقصر برموز فارغة عندما تختلف الأطوال
- جميع مدخلات النص تدعم المطالبات الديناميكية وإدخال النص متعدد الأسطر

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|---------------|---------|
| `CONDITIONING` | CONDITIONING | بيانات تكييف النص المشفرة الجاهزة للاستخدام في خطوط إنتاج SD3 |
