Вот перевод документации на русский язык:

Узел CLIPTextEncodeSD3 обрабатывает текстовые вводы для моделей Stable Diffusion 3 путем кодирования нескольких текстовых подсказок с использованием различных моделей CLIP. Он обрабатывает три отдельных текстовых ввода (`clip_g`, `clip_l` и `t5xxl`) и предоставляет опции для управления заполнением пустого текста. Узел обеспечивает правильное выравнивание токенов между различными текстовыми вводами и возвращает данные conditioning, подходящие для конвейеров генерации SD3.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `clip` | CLIP | Да | - | Модель CLIP, используемая для кодирования текста |
| `clip_l` | STRING | Да | - | Текстовый ввод для локальной модели CLIP. Поддерживает многострочный текст и динамические подсказки. |
| `clip_g` | STRING | Да | - | Текстовый ввод для глобальной модели CLIP. Поддерживает многострочный текст и динамические подсказки. |
| `t5xxl` | STRING | Да | - | Текстовый ввод для модели T5-XXL. Поддерживает многострочный текст и динамические подсказки. |
| `empty_padding` | COMBO | Да | `"none"`<br>`"empty_prompt"` | Управляет обработкой пустых текстовых вводов. При значении "none" пустые текстовые вводы для `clip_g`, `clip_l` или `t5xxl` приведут к пустым спискам токенов вместо заполнения. Это расширенный параметр (по умолчанию: "none"). |

**Ограничения параметров:**

- Когда `empty_padding` установлен в "none", пустые текстовые вводы для `clip_g`, `clip_l` или `t5xxl` приведут к пустым спискам токенов вместо заполнения
- Узел автоматически выравнивает длину токенов между вводами `clip_l` и `clip_g`, дополняя более короткий пустыми токенами при различии в длине
- Все текстовые вводы поддерживают динамические подсказки и многострочный ввод текста

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | Закодированные данные conditioning, готовые к использованию в конвейерах генерации SD3 |
