تتيح لك عقدة تحرير الصور Bria FIBO تعديل صورة موجودة باستخدام تعليمات نصية. ترسل العقدة الصورة والموجه الخاص بك إلى واجهة برمجة تطبيقات Bria، التي تستخدم نموذج FIBO لإنشاء نسخة محررة جديدة من الصورة بناءً على طلبك. يمكنك أيضًا توفير قناع لتقييد التعديلات على منطقة معينة.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `النموذج` | COMBO | نعم | `"FIBO"` | إصدار النموذج المستخدم لتحرير الصور. |
| `الصورة` | IMAGE | نعم | - | الصورة المدخلة التي ترغب في تحريرها. |
| `التوجيه` | STRING | لا | - | التعليمات النصية التي تصف كيفية تحرير الصورة (الافتراضي: فارغ). |
| `توجيه سلبي` | STRING | لا | - | نص يصف ما لا تريد ظهوره في الصورة المحررة (الافتراضي: فارغ). |
| `توجيه منظم` | STRING | لا | - | سلسلة نصية تحتوي على موجه التحرير المنظم بتنسيق JSON. استخدم هذا بدلاً من الموجه المعتاد للتحكم الدقيق والبرمجي (الافتراضي: فارغ). |
| `البذرة` | INT | نعم | 1 إلى 2147483647 | رقم يُستخدم لتهيئة التوليد العشوائي، مما يضمن نتائج قابلة للتكرار (الافتراضي: 1). |

| `الخطوات` | INT | نعم | 20 إلى 50 | عدد خطوات إزالة التشويش التي سينفذها النموذج (الافتراضي: 50). |

| `القناع` | MASK | لا | - | صورة قناع اختيارية. إذا تم توفيرها، سيتم تطبيق التعديلات فقط على المناطق المقنعة من الصورة. |

**قيود مهمة:**

* يجب عليك توفير واحد على الأقل من مدخلات `prompt` أو `structured_prompt`. لا يمكن أن يكون كلاهما فارغين.
* مطلوب صورة `image` واحدة بالضبط.
* عندما يتم تعيين معامل `moderation` إلى `"true"`، تصبح ثلاثة مدخلات منطقية إضافية متاحة: `prompt_content_moderation` (الافتراضي: false)، `visual_input_moderation` (الافتراضي: false)، و `visual_output_moderation` (الافتراضي: true).
