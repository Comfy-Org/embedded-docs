Вот перевод документации на русский язык:

Узел LoraSave извлекает и сохраняет файлы LoRA (Low-Rank Adaptation) из разностей моделей. Он может обрабатывать разности диффузионных моделей, разности текстовых энкодеров или обе одновременно, преобразуя их в формат LoRA с указанным рангом и типом. Полученный файл LoRA сохраняется в выходной каталог для последующего использования.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `filename_prefix` | STRING | Да | - | Префикс для имени выходного файла (по умолчанию: "loras/ComfyUI_extracted_lora") |
| `rank` | INT | Да | 1-4096 | Значение ранга для LoRA, определяющее размер и сложность (по умолчанию: 8) |
| `lora_type` | COMBO | Да | `"standard"`<br>`"locon"`<br>`"loha"`<br>`"lokr"`<br>`"dylora"` | Тип создаваемой LoRA (по умолчанию: "standard") |
| `bias_diff` | BOOLEAN | Да | - | Включать ли разности смещений в расчет LoRA (по умолчанию: True) |
| `model_diff` | MODEL | Нет | - | Выходные данные ModelSubtract для преобразования в LoRA |
| `text_encoder_diff` | CLIP | Нет | - | Выходные данные CLIPSubtract для преобразования в LoRA |

**Примечание:** Для работы узла необходимо указать хотя бы один из параметров `model_diff` или `text_encoder_diff`. Если оба параметра опущены, узел не будет выдавать результат.

## Выходные данные

| Имя выходного параметра | Тип данных | Описание |
|-------------------------|------------|----------|
| - | - | Этот узел сохраняет файл LoRA в выходной каталог, но не возвращает никаких данных через рабочий процесс |
