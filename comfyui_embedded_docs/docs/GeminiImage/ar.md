# ترجمة وثيقة عقد GeminiImage

## نظرة عامة

يقوم عقد GeminiImage بإنشاء استجابات نصية وصورية من نماذج الذكاء الاصطناعي Gemini من Google. يتيح لك تقديم مدخلات متعددة الوسائط تشمل النصوص والصور والملفات لإنشاء مخرجات نصية وصورية متماسكة. يتولى العقد جميع عمليات الاتصال بواجهة API وتحليل الاستجابات باستخدام أحدث نماذج Gemini.

## المدخلات

| المعامل | نوع البيانات | نوع الإدخال | القيمة الافتراضية | النطاق | الوصف |
|---------|--------------|--------------|-------------------|--------|--------|
| `prompt` | STRING | مطلوب | "" | - | النص التحفيزي للتوليد |
| `model` | COMBO | مطلوب | gemini_2_5_flash_image_preview | `gemini_2_5_flash_image_preview`<br>`gemini_2_5_pro_exp_03_25`<br>`gemini_2_0_flash_exp_image_generation` | نموذج Gemini المستخدم لتوليد الاستجابات. |
| `seed` | INT | مطلوب | 42 | 0 إلى 18446744073709551615 | عند تثبيت البذرة على قيمة محددة، يبذل النموذج قصارى جهده لتقديم نفس الاستجابة للطلبات المتكررة. لا يُضمن الإخراج الحتمي. كما أن تغيير النموذج أو إعدادات المعاملات، مثل درجة الحرارة، قد يسبب تغيرات في الاستجابة حتى عند استخدام نفس قيمة البذرة. بشكل افتراضي، تُستخدم قيمة بذرة عشوائية. |
| `images` | IMAGE | اختياري | لا شيء | - | صورة (صور) اختيارية لاستخدامها كسياق للنموذج. لتضمين صور متعددة، يمكنك استخدام عقد تجميع الصور. |
| `files` | GEMINI_INPUT_FILES | اختياري | لا شيء | - | ملف (ملفات) اختيارية لاستخدامها كسياق للنموذج. يقبل المدخلات من عقد ملفات إدخال محتوى Gemini. |

**ملاحظة:** يتضمن العقد معاملات مخفية (`auth_token`, `comfy_api_key`, `unique_id`) يتم التعامل معها تلقائيًا بواسطة النظام ولا تتطلب إدخالاً من المستخدم.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|--------------|--------|
| `IMAGE` | IMAGE | استجابة الصورة المُنشأة من نموذج Gemini |
| `STRING` | STRING | استجابة النص المُنشأة من نموذج Gemini |
