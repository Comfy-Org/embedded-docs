هذه ترجمة الوثيقة التقنية لعقدة **ConditioningSetPropertiesAndCombine** في ComfyUI إلى العربية، مع الالتزام بقواعد الترجمة المحددة:

---

تعدل عقدة **ConditioningSetPropertiesAndCombine** بيانات التكييف من خلال تطبيق خصائص من مدخل تكييف جديد على مدخل تكييف موجود. تقوم بدمج مجموعتي التكييف مع التحكم في قوة التكييف الجديد وتحديد كيفية تطبيق منطقة التكييف.

## المدخلات

| المعامل | نوع البيانات | نوع الإدخال | القيمة الافتراضية | النطاق | الوصف |
|---------|--------------|-------------|-------------------|--------|-------|
| `تهيئة` | CONDITIONING | مطلوب | - | - | بيانات التكييف الأصلية المراد تعديلها |
| `تهيئة جديدة` | CONDITIONING | مطلوب | - | - | بيانات التكييف الجديدة التي توفر الخصائص المراد تطبيقها |
| `القوة` | FLOAT | مطلوب | 1.0 | 0.0 - 10.0 | يتحكم في شدة خصائص التكييف الجديدة |
| `تعيين منطقة التهيئة` | STRING | مطلوب | default | ["default", "mask bounds"] | يحدد كيفية تطبيق منطقة التكييف |
| `القناع` | MASK | اختياري | - | - | قناع اختياري لتحديد مناطق محددة للتكييف |
| `الخطافات` | HOOKS | اختياري | - | - | دوال ربط اختيارية للمعالجة المخصصة |
| `خطوات زمنية` | TIMESTEPS_RANGE | اختياري | - | - | نطاق زمني اختياري للتحكم في وقت تطبيق التكييف |

**ملاحظة:** عند توفير `mask`، يمكن استخدام المعامل `set_cond_area` بقيمة "mask bounds" لتقييد تطبيق التكييف على المناطق المقنعة.
