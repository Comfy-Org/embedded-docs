Вот перевод документации на русский язык:

Узел ConditioningSetPropertiesAndCombine изменяет данные кондиционирования, применяя свойства из нового входа кондиционирования к существующему входу кондиционирования. Он объединяет два набора кондиционирования, контролируя силу нового кондиционирования и определяя способ применения области кондиционирования.

## Входы

| Параметр | Тип данных | Тип входа | По умолчанию | Диапазон | Описание |
|-----------|-----------|------------|---------|-------|-------------|
| `условие` | CONDITIONING | Обязательный | - | - | Исходные данные кондиционирования для изменения |
| `новое_условие` | CONDITIONING | Обязательный | - | - | Новые данные кондиционирования, предоставляющие свойства для применения |
| `сила` | FLOAT | Обязательный | 1.0 | 0.0 - 10.0 | Управляет интенсивностью свойств нового кондиционирования |
| `установить_область_условия` | STRING | Обязательный | default | ["default", "mask bounds"] | Определяет способ применения области кондиционирования |
| `маска` | MASK | Необязательный | - | - | Необязательная маска для определения конкретных областей кондиционирования |
| `хуки` | HOOKS | Необязательный | - | - | Необязательные функции-перехватчики для пользовательской обработки |
| `временные_шаги` | TIMESTEPS_RANGE | Необязательный | - | - | Необязательный диапазон временных шагов для управления моментом применения кондиционирования |

**Примечание:** Если указана `mask`, параметр `set_cond_area` может использовать значение "mask bounds", чтобы ограничить применение кондиционирования областями маски.

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | Объединенные данные кондиционирования с измененными свойствами |
