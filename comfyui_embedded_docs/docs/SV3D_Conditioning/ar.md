هذه ترجمة الوثيقة التقنية لعقدة SV3D_Conditioning من ComfyUI إلى العربية، مع الالتزام بقواعد الترجمة المحددة:

## نظرة عامة

تقوم عقدة `SV3D_Conditioning` بإعداد بيانات التكييف لتوليد فيديو ثلاثي الأبعاد باستخدام نموذج SV3D. تأخذ العقدة صورة أولية وتعالجها عبر مشفرات CLIP البصري وVAE لإنشاء تكييف إيجابي وسلبي، بالإضافة إلى تمثيل كامن. تقوم العقدة بتوليد تسلسلات ارتفاع وسمت الكاميرا لتوليد فيديو متعدد الإطارات بناءً على عدد إطارات الفيديو المحدد.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `clip_vision` | CLIP_VISION | نعم | - | نموذج CLIP البصري المستخدم لتشفير الصورة المدخلة |
| `init_image` | IMAGE | نعم | - | الصورة الأولية التي تعمل كنقطة انطلاق لتوليد الفيديو ثلاثي الأبعاد |
| `vae` | VAE | نعم | - | نموذج VAE المستخدم لتشفير الصورة إلى الفضاء الكامن |
| `width` | INT | لا | 16 إلى MAX_RESOLUTION | عرض الإخراج لإطارات الفيديو المُولَّدة (الافتراضي: 576، يجب أن يكون قابلاً للقسمة على 8) |
| `height` | INT | لا | 16 إلى MAX_RESOLUTION | ارتفاع الإخراج لإطارات الفيديو المُولَّدة (الافتراضي: 576، يجب أن يكون قابلاً للقسمة على 8) |
| `video_frames` | INT | لا | 1 إلى 4096 | عدد الإطارات المراد توليدها لتسلسل الفيديو (الافتراضي: 21) |
| `elevation` | FLOAT | لا | -90.0 إلى 90.0 | زاوية ارتفاع الكاميرا بالدرجات للعرض ثلاثي الأبعاد (الافتراضي: 0.0) |

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|-----------|-------------|
| `positive` | CONDITIONING | بيانات التكييف الإيجابي التي تحتوي على تضمينات الصورة ومعاملات الكاميرا للتوليد |
| `negative` | CONDITIONING | بيانات التكييف السلبي مع تضمينات مصفرة للتوليد التبايني |
| `latent` | LATENT | موتر كامن فارغ بأبعاد تتطابق مع عدد إطارات الفيديو والدقة المحددين |
