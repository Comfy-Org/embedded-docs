# SeedVR2TemporalChunk

Этот узел разделяет видео-латент SeedVR2 на меньшие временные фрагменты, которые могут обрабатываться по одному в рамках доступной видеопамяти. Он автоматически вычисляет оптимальный размер фрагмента на основе вашей памяти GPU или позволяет указать размер фрагмента вручную, и выводит фрагменты в порядке последовательности для обработки.

## Входы

| Параметр | Описание | Тип данных | Обязательный | Диапазон |
|----------|----------|------------|--------------|----------|
| `latent` | Латенты SeedVR2, закодированные VAE, которые необходимо разделить. | LATENT | Да | - |
| `temporal_overlap` | Кадры латентов, общие между соседними фрагментами и перекрестно смешиваемые при слиянии; 0 означает отсутствие перекрытия (по умолчанию: 0). | INT | Нет | от 0 до 16384 |
| `chunking_mode` | Ручной режим использует frames_per_chunk точно; автоматический режим предсказывает наибольший фрагмент, помещающийся в свободную видеопамять. | COMBO | Да | "auto"<br>"manual" |

Когда для `chunking_mode` установлено значение "manual", становится доступен дополнительный параметр:

| Параметр | Описание | Тип данных | Обязательный | Диапазон |
|----------|----------|------------|--------------|----------|
| `frames_per_chunk` | Пиксельные кадры на временной фрагмент. Должно быть значением вида 4n+1 (1, 5, 9, 13, 17, 21, ...) (по умолчанию: 21). | INT | Да | от 1 до 16384 |

Примечание: Параметр `frames_per_chunk` отображается только когда для `chunking_mode` установлено значение "manual". Значение должно удовлетворять формуле `(frames_per_chunk - 1) % 4 == 0`, то есть должно быть одним из: 1, 5, 9, 13, 17, 21 и т.д.

## Выходы

| Имя выхода | Описание | Тип данных |
|------------|----------|------------|
| `latents` | Временные фрагменты в порядке последовательности. | LATENT |
| `temporal_overlap` | Эффективное перекрытие кадров латентов между соседними фрагментами, для узла Merge SeedVR2 Latents. | INT |
