Вот перевод документации на русский язык в соответствии с вашими правилами:

Узел GeminiImage2Node создаёт или редактирует изображения с помощью модели Google Vertex AI Gemini. Он отправляет текстовый запрос и, при необходимости, эталонные изображения или файлы в API и возвращает сгенерированное изображение и/или текстовое описание.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `prompt` | STRING | Да | Н/Д | Текстовый запрос, описывающий изображение для генерации или изменения. Включает любые ограничения, стили или детали, которым должна следовать модель. |
| `model` | COMBO | Да | `"gemini-3-pro-image-preview"`<br>`"Nano Banana 2 (Gemini 3.1 Flash Image)"` | Конкретная модель Gemini для генерации. Опция "Nano Banana 2" внутренне сопоставляется с моделью `gemini-3.1-flash-image-preview`. |
| `seed` | INT | Да | от 0 до 18446744073709551615 | При фиксации на определённом значении модель прилагает максимальные усилия для предоставления одинакового ответа на повторные запросы. Детерминированный вывод не гарантируется. Изменение модели или других настроек может вызвать вариации даже при одинаковом seed. По умолчанию: 42. |
| `aspect_ratio` | COMBO | Да | `"auto"`<br>`"1:1"`<br>`"2:3"`<br>`"3:2"`<br>`"3:4"`<br>`"4:3"`<br>`"4:5"`<br>`"5:4"`<br>`"9:16"`<br>`"16:9"`<br>`"21:9"` | Желаемое соотношение сторон для выходного изображения. Если установлено значение 'auto', оно соответствует соотношению сторон вашего входного изображения; если изображение не предоставлено, обычно генерируется квадрат 16:9. По умолчанию: "auto". |
| `resolution` | COMBO | Да | `"1K"`<br>`"2K"`<br>`"4K"` | Целевое разрешение вывода. Для 2K/4K используется встроенный апскейлер Gemini. |
| `response_modalities` | COMBO | Да | `"IMAGE+TEXT"`<br>`"IMAGE"` | Выберите 'IMAGE' для вывода только изображения, или 'IMAGE+TEXT', чтобы вернуть как сгенерированное изображение, так и текстовый ответ. |
| `images` | IMAGE | Нет | Н/Д | Необязательное эталонное изображение(я). Чтобы включить несколько изображений, используйте узел Batch Images (до 14). |
| `files` | CUSTOM | Нет | Н/Д | Необязательный файл(ы) для использования в качестве контекста для модели. Принимает входные данные от узла Gemini Generate Content Input Files. |
| `system_prompt` | STRING | Нет | Н/Д | Базовые инструкции, определяющие поведение ИИ. По умолчанию: предопределённый системный запрос для генерации изображений. |

**Ограничения:**

* Вход `images` поддерживает максимум 14 изображений. Если предоставлено больше, будет вызвана ошибка.
* Вход `files` должен быть подключён к узлу, который выводит тип данных `GEMINI_INPUT_FILES`.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `image` | IMAGE | Изображение, сгенерированное или отредактированное моделью Gemini. |
| `string` | STRING | Текстовый ответ от модели. Этот вывод будет пустым, если `response_modalities` установлено в "IMAGE". |
