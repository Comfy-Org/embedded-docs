Вот перевод документации для узла `LatentCutToBatch`:

Узел LatentCutToBatch разделяет латентное представление вдоль выбранного измерения на несколько срезов и объединяет их в новый пакет. Это позволяет независимо обрабатывать различные части латентного сэмпла.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `samples` | LATENT | Да | - | Латентное представление, которое будет разделено и объединено в пакет. |
| `dim` | COMBO | Да | `"t"`<br>`"x"`<br>`"y"` | Измерение, вдоль которого будут разрезаны латентные сэмплы. `"t"` относится к временному измерению, `"x"` — к ширине, а `"y"` — к высоте. |
| `slice_size` | INT | Да | от 1 до 16384 | Размер каждого среза, вырезаемого из указанного измерения. Если размер измерения не делится на это значение без остатка, остаток отбрасывается. (по умолчанию: 1) |

## Выходы

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `samples` | LATENT | Результирующий латентный пакет, содержащий разрезанные и объединённые сэмплы. |
