# Узел LumaImageNode

Генерирует изображения синхронно на основе текстового запроса и соотношения сторон. Этот узел создает изображения с использованием текстовых описаний и позволяет управлять размерами и стилем изображения через различные референсные входы, включая изображения персонажей и стилей.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `prompt` | STRING | Да | - | Запрос для генерации изображения (по умолчанию: пустая строка). Должен содержать не менее 3 символов. |
| `model` | COMBO | Да | `photon-flash-1`<br>`photon-1`<br>`photon` | Выбор модели для генерации изображения. Разные модели имеют разную стоимость. |
| `aspect_ratio` | COMBO | Да | `16:9`<br>`1:1`<br>`4:3`<br>`3:2`<br>`21:9`<br>`9:16`<br>`3:4`<br>`2:3`<br>`9:21` | Соотношение сторон генерируемого изображения (по умолчанию: `16:9`) |
| `seed` | INT | Да | от 0 до 18446744073709551615 | Сид для определения необходимости повторного запуска узла; фактические результаты недетерминированы независимо от сида (по умолчанию: 0) |
| `style_image_weight` | FLOAT | Нет | от 0.0 до 1.0 | Вес изображения стиля. Игнорируется, если не указано `style_image` (по умолчанию: 1.0) |
| `image_luma_ref` | LUMA_REF | Нет | - | Подключение узла Luma Reference для влияния на генерацию с помощью входных изображений; может учитываться до 4 изображений. |
| `style_image` | IMAGE | Нет | - | Референсное изображение стиля; будет использовано только 1 изображение. |
| `character_image` | IMAGE | Нет | - | Референсные изображения персонажа; может быть пакетом из нескольких, до 4 изображений. |

**Ограничения параметров:**

- `prompt` должен содержать не менее 3 символов после удаления пробелов.
- Параметр `image_luma_ref` может принимать до 4 референсных изображений.
- Параметр `character_image` может принимать до 4 референсных изображений персонажа.
- Параметр `style_image` принимает только 1 референсное изображение стиля.
- Параметр `style_image_weight` используется только при наличии `style_image`.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `output` | IMAGE | Сгенерированное изображение на основе входных параметров. |
