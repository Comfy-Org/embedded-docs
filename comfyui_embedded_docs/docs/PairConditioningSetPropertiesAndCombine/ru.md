Вот перевод документации на русский язык:

Узел **PairConditioningSetPropertiesAndCombine** изменяет и объединяет пары кондиционирования, применяя новые данные кондиционирования к существующим положительным и отрицательным входам кондиционирования. Он позволяет регулировать силу применяемого кондиционирования и управлять тем, как задается область кондиционирования. Этот узел особенно полезен для продвинутых рабочих процессов манипуляции кондиционированием, где требуется смешивать несколько источников кондиционирования вместе.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `positive` | CONDITIONING | Да | - | Исходный положительный вход кондиционирования |
| `negative` | CONDITIONING | Да | - | Исходный отрицательный вход кондиционирования |
| `positive_NEW` | CONDITIONING | Да | - | Новое положительное кондиционирование для применения |
| `negative_NEW` | CONDITIONING | Да | - | Новое отрицательное кондиционирование для применения |
| `strength` | FLOAT | Да | 0.0 до 10.0 | Коэффициент силы для применения нового кондиционирования (по умолчанию: 1.0) |
| `set_cond_area` | STRING | Да | "default"<br>"mask bounds" | Управляет тем, как применяется область кондиционирования (по умолчанию: "default") |
| `mask` | MASK | Нет | - | Необязательная маска для ограничения области применения кондиционирования |
| `hooks` | HOOKS | Нет | - | Необязательная группа хуков для расширенного управления |
| `timesteps` | TIMESTEPS_RANGE | Нет | - | Необязательная спецификация диапазона временных шагов |

## Выходы

| Имя выхода | Тип данных | Описание |

| `negative` | CONDITIONING | Объединенный отрицательный выход кондиционирования |
