Вот перевод документации на русский язык:

Узел **PairConditioningSetPropertiesAndCombine** изменяет и объединяет пары кондиционирования, применяя новые данные кондиционирования к существующим положительным и отрицательным входам кондиционирования. Он позволяет регулировать силу применяемого кондиционирования и управлять тем, как задается область кондиционирования. Этот узел особенно полезен для продвинутых рабочих процессов манипуляции кондиционированием, где требуется смешивать несколько источников кондиционирования вместе.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `положительный` | CONDITIONING | Да | - | Исходный положительный вход кондиционирования |
| `отрицательный` | CONDITIONING | Да | - | Исходный отрицательный вход кондиционирования |
| `положительный_НОВЫЙ` | CONDITIONING | Да | - | Новое положительное кондиционирование для применения |
| `отрицательный_НОВЫЙ` | CONDITIONING | Да | - | Новое отрицательное кондиционирование для применения |
| `сила` | FLOAT | Да | 0.0 до 10.0 | Коэффициент силы для применения нового кондиционирования (по умолчанию: 1.0) |
| `установить_область_условия` | STRING | Да | "default"<br>"mask bounds" | Управляет тем, как применяется область кондиционирования (по умолчанию: "default") |
| `маска` | MASK | Нет | - | Необязательная маска для ограничения области применения кондиционирования |
| `хуки` | HOOKS | Нет | - | Необязательная группа хуков для расширенного управления |
| `временные_шаги` | TIMESTEPS_RANGE | Нет | - | Необязательная спецификация диапазона временных шагов |

## Выходы

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `отрицательный` | CONDITIONING | Объединенный положительный выход кондиционирования |
| `отрицательный` | CONDITIONING | Объединенный отрицательный выход кондиционирования |
