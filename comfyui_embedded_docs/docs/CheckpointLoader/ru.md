Узел CheckpointLoader загружает предварительно обученную контрольную точку модели вместе с её конфигурационным файлом. Он принимает конфигурационный файл и файл контрольной точки в качестве входных данных и возвращает загруженные компоненты модели, включая основную модель, модель CLIP и модель VAE для использования в рабочем процессе.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `config_name` | STRING | Да | Доступные конфигурационные файлы | Конфигурационный файл, определяющий архитектуру и настройки модели |
| `ckpt_name` | STRING | Да | Доступные файлы контрольных точек | Файл контрольной точки, содержащий обученные веса и параметры модели |

**Примечание:** Для этого узла требуется выбрать как конфигурационный файл, так и файл контрольной точки. Конфигурационный файл должен соответствовать архитектуре загружаемого файла контрольной точки.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `MODEL` | MODEL | Загруженный основной компонент модели, готовый к инференсу |
| `CLIP` | CLIP | Загруженный компонент модели CLIP для кодирования текста |
| `VAE` | VAE | Загруженный компонент модели VAE для кодирования и декодирования изображений |

**Важное примечание:** Этот узел помечен как устаревший и может быть удалён в будущих версиях. Для новых рабочих процессов рекомендуется использовать альтернативные узлы загрузки.
