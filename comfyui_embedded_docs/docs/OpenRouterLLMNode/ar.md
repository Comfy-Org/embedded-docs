## نظرة عامة

ترسل عقدة OpenRouter LLM مطالبة نصية إلى مجموعة منتقاة من نماذج اللغة الشائعة المتاحة عبر خدمة OpenRouter، وتُعيد الاستجابة النصية المُولَّدة. تدعم العقدة نماذج من مزودين مثل xAI وDeepSeek وQwen وMistral وZ.AI (GLM) وMoonshot (Kimi) وPerplexity Sonar، ويمكنها اختياريًا تضمين الصور أو مقاطع الفيديو في الطلب.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `prompt` | STRING | نعم | غير متاح | النص المُدخل إلى النموذج. |
| `model` | STRING | نعم | تتوفر خيارات متعددة (انظر الملاحظة أدناه) | نموذج OpenRouter المُستخدم لتوليد الاستجابة. |
| `seed` | INT | نعم | 0 إلى 2147483647 | البذرة المستخدمة في أخذ العينات. اضبطها على 0 للإلغاء. تعامل معظم النماذج هذه القيمة كإشارة إرشادية فقط. (القيمة الافتراضية: 0) |
| `system_prompt` | STRING | لا | غير متاح | تعليمات أساسية تُحدد سلوك النموذج. (القيمة الافتراضية: "") |

**ملاحظة حول معامل `model`:** يتم بناء خيارات النماذج المتاحة ديناميكيًا وقد تتضمن نماذج ذات قدرات مختلفة. تدعم بعض النماذج ميزات إضافية مثل جهد الاستدلال، أو البحث على الويب، أو إدخال الصور/مقاطع الفيديو. ستتحقق العقدة من أن عدد الصور أو مقاطع الفيديو المُقدمة لا يتجاوز الحد الأقصى المدعوم من قبل النموذج.

**ملاحظة حول معامل `seed`:** يتميز معامل `seed` بسلوك "التحكم بعد التوليد"، مما يعني أنه يمكن ضبطه ليتغير تلقائيًا (مثل التوزيع العشوائي، أو الزيادة، أو الثبات) بعد كل تنفيذ للعقدة، وذلك وفقًا لإعدادات عناصر واجهة المستخدم الخاصة بالمستخدم.

**ملاحظة حول `system_prompt`:** هذا المعامل اختياري ويتم تمييزه كمعامل متقدم في واجهة المستخدم.
