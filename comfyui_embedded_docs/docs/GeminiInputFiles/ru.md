Загружает и форматирует входные файлы для использования с Gemini API. Этот узел позволяет пользователям включать текстовые файлы (.txt) и PDF-файлы (.pdf) в качестве входного контекста для модели Gemini. Файлы преобразуются в формат, требуемый API, и могут быть объединены в цепочку для включения нескольких файлов в один запрос.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `file` | COMBO | Да | Доступно несколько опций | Входные файлы, включаемые в качестве контекста для модели. На данный момент принимаются только текстовые файлы (.txt) и PDF-файлы (.pdf). Размер файлов не должен превышать максимально допустимый лимит. |
| `GEMINI_INPUT_FILES` | GEMINI_INPUT_FILES | Нет | Н/Д | Дополнительный файл(ы) для объединения в пакет с файлом, загруженным через этот узел. Позволяет объединять входные файлы в цепочку, чтобы одно сообщение могло содержать несколько входных файлов. |

**Примечание:** Параметр `file` отображает только текстовые файлы (.txt) и PDF-файлы (.pdf), размер которых не превышает максимально допустимый лимит. Файлы автоматически фильтруются и сортируются по имени.

## Выходные параметры

| Имя выходного параметра | Тип данных | Описание |
|-------------------------|------------|----------|
| `GEMINI_INPUT_FILES` | GEMINI_INPUT_FILES | Отформатированные данные файла, готовые к использованию с узлами Gemini LLM, содержащие загруженное содержимое файла в соответствующем формате API. |
