Вот перевод документации на русский язык:

Узел TextEncodeQwenImageEdit обрабатывает текстовые подсказки и опциональные изображения для генерации данных кондиционирования (conditioning) при создании или редактировании изображений. Он использует модель CLIP для токенизации входных данных и может опционально кодировать эталонные изображения с помощью VAE для создания эталонных латентов (reference latents). При наличии изображения узел автоматически изменяет его размер для обеспечения единообразных размеров обработки.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `clip` | CLIP | Да | - | Модель CLIP, используемая для токенизации текста и изображений |
| `prompt` | STRING | Да | - | Текстовая подсказка для генерации кондиционирования, поддерживает многострочный ввод и динамические подсказки |
| `vae` | VAE | Нет | - | Опциональная модель VAE для кодирования эталонных изображений в латенты |
| `image` | IMAGE | Нет | - | Опциональное входное изображение для эталона или редактирования |

**Примечание:** Если указаны и `image`, и `vae`, узел кодирует изображение в эталонные латенты и присоединяет их к выходным данным кондиционирования. Изображение автоматически изменяется в размере для поддержания единообразного масштаба обработки примерно 1024x1024 пикселя.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | Данные кондиционирования, содержащие текстовые токены и опциональные эталонные латенты для генерации изображения |
