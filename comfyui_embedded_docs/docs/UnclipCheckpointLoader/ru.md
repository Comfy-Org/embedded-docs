Этот узел обнаруживает модели, расположенные в папке `ComfyUI/models/checkpoints`, а также считывает модели из дополнительных путей, указанных в файле extra_model_paths.yaml. Иногда может потребоваться **обновить интерфейс ComfyUI**, чтобы он смог прочитать файлы моделей из соответствующей папки.

Узел unCLIPCheckpointLoader предназначен для загрузки контрольных точек (checkpoints), специально адаптированных для моделей unCLIP. Он упрощает получение и инициализацию моделей, модулей CLIP vision и VAE из указанной контрольной точки, оптимизируя процесс настройки для последующих операций или анализа.

## Входные данные

| Поле         | Тип Comfy        | Описание                                                                                     |
|--------------|------------------|----------------------------------------------------------------------------------------------|
| `название_ckpt`  | `COMBO[STRING]`  | Указывает имя загружаемой контрольной точки, определяя и извлекая нужный файл из заданного каталога, что влияет на инициализацию моделей и конфигураций. |

## Выходные данные

| Поле          | Тип Comfy      | Описание                                                              | Тип Python          |
|---------------|----------------|-----------------------------------------------------------------------|---------------------|
| `model`       | `MODEL`        | Представляет основную модель, загруженную из контрольной точки.       | `torch.nn.Module`   |
| `clip`        | `CLIP`         | Представляет модуль CLIP, загруженный из контрольной точки (если доступен). | `torch.nn.Module`   |
| `vae`         | `VAE`          | Представляет модуль VAE, загруженный из контрольной точки (если доступен).  | `torch.nn.Module`   |
| `clip_vision` | `CLIP_VISION`  | Представляет модуль CLIP vision, загруженный из контрольной точки (если доступен). | `torch.nn.Module`   |
