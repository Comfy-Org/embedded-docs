## Обзор

Редактирование одного изображения с помощью Beeble SwitchX. Этот узел может заменять любые элементы сцены (фон, освещение, костюм), сохраняя пиксели исходного объекта. Укажите референсное изображение и/или текстовый запрос для описания нового вида. Максимальное разрешение составляет примерно 2,77 мегапикселя.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `изображение` | IMAGE | Да | - | Исходное изображение для редактирования. |
| `промпт` | STRING | Да | - | Текстовое описание желаемого нового вида (например, "рыцарь в сияющих доспехах"). |
| `alpha_mode` | COMBO | Да | `"select"`<br>`"fill"`<br>`"custom"` | Способ обработки альфа-канала. "select" использует ключевой кадр для выделения объекта, "fill" заменяет всё изображение без отдельного альфа-канала, "custom" использует маску, предоставленную пользователем. |
| `максимальное разрешение` | COMBO | Да | `"1080p"`<br>`"720p"` | Максимальное разрешение выходного изображения. Более высокое разрешение требует больше кредитов. |
| `seed` | INT | Да | - | Значение зерна для воспроизводимости результатов. |
| `референс-изображение` | IMAGE | Нет | - | Дополнительное референсное изображение для управления стилем или внешним видом новых элементов сцены. |

**Примечание о `alpha_mode`:** Если `alpha_mode` установлен в `"select"`, необходимо также указать `alpha_keyframe` (изображение ключевого кадра для выделения объекта). Если установлен в `"custom"`, необходимо указать `alpha_mask` (маска, созданная пользователем). Если установлен в `"fill"`, ввод альфа-канала не требуется.

## Выходные данные

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `изображение` | IMAGE | Отредактированное изображение с заменёнными элементами сцены. |
| `alpha` | MASK | Альфа-канал, используемый Beeble. Пуст для режима "fill", который не имеет отдельного альфа-канала. |
