## نمای کلی

یک تصویر را با استفاده از Beeble SwitchX ویرایش کنید. این گره می‌تواند هر چیزی را در صحنه (پس‌زمینه، نورپردازی، لباس) تغییر دهد، در حالی که پیکسل‌های سوژه اصلی را حفظ می‌کند. یک تصویر مرجع و/یا توضیح متنی برای توصیف ظاهر جدید ارائه دهید. حداکثر رزولوشن تقریباً ۲.۷۷ مگاپیکسل است.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|-----------|-----------|----------|-------|-------------|
| `تصویر` | IMAGE | بله | - | تصویر مبدأ برای ویرایش. |
| `پرامپت` | STRING | بله | - | توضیح متنی از ظاهر جدید مورد نظر (مثلاً «یک شوالیه در زره درخشان»). |
| `حالت آلفا` | COMBO | بله | `"select"`<br>`"fill"`<br>`"custom"` | نحوه مدیریت مات آلفا. «select» از یک فریم کلیدی برای انتخاب سوژه استفاده می‌کند، «fill» کل تصویر را بدون مات جداگانه جایگزین می‌کند، «custom» از یک ماسک ارائه‌شده توسط کاربر استفاده می‌کند. |
| `حداکثر وضوح` | COMBO | بله | `"1080p"`<br>`"720p"` | حداکثر رزولوشن برای تصویر خروجی. رزولوشن بالاتر هزینه اعتبار بیشتری دارد. |
| `seed` | INT | بله | - | یک مقدار seed برای تکرارپذیری. |
| `تصویر مرجع` | IMAGE | خیر | - | یک تصویر مرجع اختیاری برای هدایت سبک یا ظاهر عناصر جدید صحنه. |

**نکته درباره `alpha_mode`:** وقتی `alpha_mode` روی `"select"` تنظیم شده است، باید یک `alpha_keyframe` (یک تصویر فریم کلیدی برای انتخاب سوژه) نیز ارائه دهید. وقتی روی `"custom"` تنظیم شده است، باید یک `alpha_mask` (یک ماسک ساخته‌شده توسط کاربر) ارائه دهید. وقتی روی `"fill"` تنظیم شده است، نیازی به ورودی آلفا نیست.

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
|-------------|-----------|-------------|
| `تصویر` | IMAGE | تصویر ویرایش‌شده با عناصر صحنه تغییر یافته. |
| `alpha` | MASK | مات آلفای استفاده‌شده توسط Beeble. برای حالت «fill» که مات جداگانه‌ای ندارد، خالی است. |
