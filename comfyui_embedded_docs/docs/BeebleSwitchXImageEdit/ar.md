# نظرة عامة

تحرير صورة واحدة باستخدام Beeble SwitchX. يمكن لهذه العقدة تبديل أي عنصر في المشهد (الخلفية، الإضاءة، الزي) مع الحفاظ على وحدات البكسل الأصلية للكائن الرئيسي. قم بتوفير صورة مرجعية و/أو نص وصفي لتحديد المظهر الجديد. الحد الأقصى للدقة هو حوالي 2.77 ميجابكسل.

# المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|---------|---------------|--------|---------|-------------|
| `صورة` | IMAGE | نعم | - | الصورة المصدر المراد تحريرها. |
| `وصف نصي` | STRING | نعم | - | وصف نصي للمظهر الجديد المطلوب (مثال: "فارس يرتدي درعًا لامعًا"). |
| `وضع ألفا` | COMBO | نعم | `"select"`<br>`"fill"`<br>`"custom"` | كيفية معالجة قناة الشفافية ألفا. "select" تستخدم إطارًا مفتاحيًا لتحديد الكائن الرئيسي، "fill" تستبدل الصورة بأكملها بدون قناة شفافية منفصلة، "custom" تستخدم قناعًا يوفره المستخدم. |
| `أقصى دقة` | COMBO | نعم | `"1080p"`<br>`"720p"` | الحد الأقصى لدقة الصورة الناتجة. الدقة الأعلى تستهلك رصيدًا أكثر. |
| `البذرة` | INT | نعم | - | قيمة البذرة لإمكانية إعادة إنتاج النتائج. |
| `الصورة المرجعية` | IMAGE | لا | - | صورة مرجعية اختيارية لتوجيه نمط أو مظهر عناصر المشهد الجديدة. |

**ملاحظة حول `alpha_mode`:** عند ضبط `alpha_mode` على `"select"`، يجب أيضًا توفير `alpha_keyframe` (صورة إطار مفتاحي تستخدم لتحديد الكائن الرئيسي). عند ضبطه على `"custom"`، يجب توفير `alpha_mask` (قناع من إنشاء المستخدم). عند ضبطه على `"fill"`، لا حاجة لإدخال قناة شفافية.

# المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|---------------|-------------|
| `صورة` | IMAGE | الصورة المحررة مع تبديل عناصر المشهد. |
| `alpha` | MASK | قناة الشفافية ألفا المستخدمة بواسطة Beeble. فارغة في وضع "fill" الذي لا يحتوي على قناة شفافية منفصلة. |
