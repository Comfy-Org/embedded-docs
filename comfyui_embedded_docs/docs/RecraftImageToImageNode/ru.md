Вот перевод документации на русский язык:

Этот узел изменяет существующее изображение на основе текстового запроса и параметра силы. Он использует API Recraft для преобразования входного изображения в соответствии с заданным описанием, сохраняя при этом некоторое сходство с исходным изображением в зависимости от настройки силы.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `изображение` | IMAGE | Да | - | Входное изображение для модификации |
| `запрос` | STRING | Да | - | Запрос для генерации изображения (по умолчанию: "", макс. длина: 1000 символов) |
| `n` | INT | Да | 1-6 | Количество генерируемых изображений (по умолчанию: 1) |
| `степень изменения` | FLOAT | Да | 0.0-1.0 | Определяет отличие от исходного изображения, должно находиться в диапазоне [0, 1], где 0 означает почти полную идентичность, а 1 — минимальное сходство (по умолчанию: 0.5) |
| `seed` | INT | Да | 0-18446744073709551615 | Сид для определения необходимости повторного запуска узла; фактические результаты недетерминированы независимо от сида (по умолчанию: 0) |
| `recraft_style` | STYLEV3 | Нет | - | Необязательный выбор стиля для генерации изображения. Если не указан, по умолчанию используется `realistic_image` |
| `негативный запрос` | STRING | Нет | - | Необязательное текстовое описание нежелательных элементов на изображении (по умолчанию: "") |
| `recraft_controls` | CONTROLS | Нет | - | Необязательные дополнительные элементы управления генерацией через узел Recraft Controls |

**Примечание:** Параметр `seed` только инициирует повторное выполнение узла, но не гарантирует детерминированных результатов. Параметр силы округляется до 2 знаков после запятой внутри системы. Запрос проверяется и не должен превышать 1000 символов. Если `recraft_style` не указан, узел по умолчанию использует стиль `realistic_image`.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `изображение` | IMAGE | Сгенерированное изображение(я) на основе входного изображения и запроса |
