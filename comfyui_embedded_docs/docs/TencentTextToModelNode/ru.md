Этот узел использует API Hunyuan3D Pro от Tencent для генерации 3D-модели на основе текстового описания. Он отправляет запрос на создание задачи генерации, опрашивает результат и загружает итоговые файлы модели в форматах GLB и OBJ.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `model` | COMBO | Да | `"3.0"`<br>`"3.1"` | Версия модели Hunyuan3D для использования. Опция LowPoly недоступна для модели `3.1`. |
| `prompt` | STRING | Да | - | Текстовое описание 3D-модели для генерации. Поддерживается до 1024 символов. |
| `face_count` | INT | Да | 3000 - 1500000 | Целевое количество полигонов для генерируемой 3D-модели. По умолчанию: 500000. |
| `generate_type` | DYNAMICCOMBO | Да | `"Normal"`<br>`"LowPoly"`<br>`"Geometry"` | Тип генерируемой 3D-модели. Доступные опции и связанные с ними параметры:<br>- **Normal**: Создает стандартную модель. Включает параметр `pbr` (по умолчанию: `False`).<br>- **LowPoly**: Создает низкополигональную модель. Включает параметры `polygon_type` (`"triangle"` или `"quadrilateral"`) и `pbr` (по умолчанию: `False`).<br>- **Geometry**: Создает модель только с геометрией. |
| `seed` | INT | Нет | 0 - 2147483647 | Значение зерна для генерации. Результаты недетерминированы независимо от зерна. Установка нового зерна определяет, должен ли узел перезапуститься. По умолчанию: 0. |

**Примечание:** Параметр `generate_type` является динамическим. Выбор `"LowPoly"` откроет дополнительные поля для `polygon_type` и `pbr`. Выбор `"Normal"` откроет поле для `pbr`. Выбор `"Geometry"` не откроет дополнительных полей.

**Ограничение:** Тип генерации `"LowPoly"` нельзя использовать с моделью `"3.1"`.

## Выходные данные

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `model_file` | STRING | Устаревший выход для обратной совместимости. |
| `GLB` | FILE3DGLB | Сгенерированная 3D-модель в формате GLB. |
| `OBJ` | FILE3DOBJ | Сгенерированная 3D-модель в формате OBJ. |
| `texture_image` | IMAGE | Текстурное изображение, извлеченное из сгенерированного файла OBJ (если доступно). |
