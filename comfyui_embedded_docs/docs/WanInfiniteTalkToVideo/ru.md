Вот перевод документации на русский язык в соответствии с вашими правилами:

Узел WanInfiniteTalkToVideo генерирует видеопоследовательности из аудиовхода. Он использует модель диффузии видео, обусловленную аудиохарактеристиками, извлеченными из одного или двух дикторов, для создания латентного представления видео с говорящей головой. Узел может генерировать новую последовательность или расширять существующую, используя предыдущие кадры для контекста движения.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `mode` | COMBO | Да | `"single_speaker"`<br>`"two_speakers"` | Режим аудиовхода. `"single_speaker"` использует один аудиовход. `"two_speakers"` включает входы для второго диктора и соответствующие маски. |
| `model` | MODEL | Да | - | Базовая модель диффузии видео. |
| `model_patch` | MODELPATCH | Да | - | Патч модели, содержащий слои проекции аудио. |
| `positive` | CONDITIONING | Да | - | Положительное кондиционирование для направления генерации. |
| `negative` | CONDITIONING | Да | - | Отрицательное кондиционирование для направления генерации. |
| `vae` | VAE | Да | - | VAE, используемый для кодирования изображений в латентное пространство и обратно. |
| `width` | INT | Нет | 16 - MAX_RESOLUTION | Ширина выходного видео в пикселях. Должна делиться на 16. (по умолчанию: 832) |
| `height` | INT | Нет | 16 - MAX_RESOLUTION | Высота выходного видео в пикселях. Должна делиться на 16. (по умолчанию: 480) |
| `length` | INT | Нет | 1 - MAX_RESOLUTION | Количество кадров для генерации. (по умолчанию: 81) |
| `clip_vision_output` | CLIPVISIONOUTPUT | Нет | - | Необязательный выход CLIP vision для дополнительного кондиционирования. |
| `start_image` | IMAGE | Нет | - | Необязательное начальное изображение для инициализации видеопоследовательности. |
| `audio_encoder_output_1` | AUDIOENCODEROUTPUT | Да | - | Основной выход аудиокодера, содержащий характеристики для первого диктора. |
| `motion_frame_count` | INT | Нет | 1 - 33 | Количество предыдущих кадров, используемых в качестве контекста движения при расширении последовательности. (по умолчанию: 9) |
| `audio_scale` | FLOAT | Нет | -10.0 - 10.0 | Коэффициент масштабирования, применяемый к аудиокондиционированию. (по умолчанию: 1.0) |
| `previous_frames` | IMAGE | Нет | - | Необязательные предыдущие кадры видео для расширения. |
| `audio_encoder_output_2` | AUDIOENCODEROUTPUT | Нет | - | Второй выход аудиокодера. Требуется, когда `mode` установлен в `"two_speakers"`. |
| `mask_1` | MASK | Нет | - | Маска для первого диктора, требуется при использовании двух аудиовходов. |
| `mask_2` | MASK | Нет | - | Маска для второго диктора, требуется при использовании двух аудиовходов. |

**Ограничения параметров:**

* Когда `mode` установлен в `"two_speakers"`, параметры `audio_encoder_output_2`, `mask_1` и `mask_2` становятся обязательными.
* Если предоставлен `audio_encoder_output_2`, также должны быть предоставлены `mask_1` и `mask_2`.
* Если предоставлены `mask_1` и `mask_2`, также должен быть предоставлен `audio_encoder_output_2`.
* Если предоставлен `previous_frames`, он должен содержать как минимум столько же кадров, сколько указано в `motion_frame_count`.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `model` | MODEL | Патченная модель с примененным аудиокондиционированием. |
| `positive` | CONDITIONING | Положительное кондиционирование, потенциально измененное с добавлением дополнительного контекста (например, начальное изображение, CLIP vision). |
| `negative` | CONDITIONING | Отрицательное кондиционирование, потенциально измененное с добавлением дополнительного контекста. |
| `latent` | LATENT | Сгенерированная видеопоследовательность в латентном пространстве. |
| `trim_image` | INT | Количество кадров от начала контекста движения, которые следует обрезать при расширении последовательности. |
