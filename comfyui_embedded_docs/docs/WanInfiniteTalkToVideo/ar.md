# ترجمة وثيقة عقدة WanInfiniteTalkToVideo

تقوم عقدة WanInfiniteTalkToVideo بتوليد تسلسلات فيديو من المدخلات الصوتية. تستخدم نموذج انتشار فيديو، معتمدًا على السمات الصوتية المستخرجة من متحدث واحد أو اثنين، لإنتاج تمثيل كامن لفيديو وجه متحدث. يمكن للعقدة توليد تسلسل جديد أو تمديد تسلسل موجود باستخدام الإطارات السابقة لسياق الحركة.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|---------|---------------|--------|---------|-------|
| `mode` | COMBO | نعم | `"single_speaker"`<br>`"two_speakers"` | وضع الإدخال الصوتي. يستخدم `"single_speaker"` مدخلاً صوتيًا واحدًا. يتيح `"two_speakers"` إدخالات لمتحدث ثانٍ والأقنعة المقابلة. |
| `model` | MODEL | نعم | - | نموذج انتشار الفيديو الأساسي. |
| `model_patch` | MODELPATCH | نعم | - | تصحيح النموذج الذي يحتوي على طبقات الإسقاط الصوتي. |
| `positive` | CONDITIONING | نعم | - | التكييف الإيجابي لتوجيه التوليد. |
| `negative` | CONDITIONING | نعم | - | التكييف السلبي لتوجيه التوليد. |
| `vae` | VAE | نعم | - | التشفير التلقائي المتغير المستخدم لتشفير الصور من وإلى الفضاء الكامن. |
| `width` | INT | لا | 16 - MAX_RESOLUTION | عرض الفيديو الناتج بالبكسل. يجب أن يكون قابلاً للقسمة على 16. (الافتراضي: 832) |
| `height` | INT | لا | 16 - MAX_RESOLUTION | ارتفاع الفيديو الناتج بالبكسل. يجب أن يكون قابلاً للقسمة على 16. (الافتراضي: 480) |
| `length` | INT | لا | 1 - MAX_RESOLUTION | عدد الإطارات المراد توليدها. (الافتراضي: 81) |
| `clip_vision_output` | CLIPVISIONOUTPUT | لا | - | مخرجات رؤية CLIP اختيارية لتكييف إضافي. |
| `start_image` | IMAGE | لا | - | صورة بداية اختيارية لتهيئة تسلسل الفيديو. |
| `audio_encoder_output_1` | AUDIOENCODEROUTPUT | نعم | - | مخرجات المشفر الصوتي الأساسي التي تحتوي على سمات المتحدث الأول. |
| `motion_frame_count` | INT | لا | 1 - 33 | عدد الإطارات السابقة المستخدمة كسياق حركة عند تمديد تسلسل. (الافتراضي: 9) |
| `audio_scale` | FLOAT | لا | -10.0 - 10.0 | عامل قياس يُطبق على التكييف الصوتي. (الافتراضي: 1.0) |
| `previous_frames` | IMAGE | لا | - | إطارات فيديو سابقة اختيارية للتمديد منها. |
| `audio_encoder_output_2` | AUDIOENCODEROUTPUT | لا | - | مخرجات المشفر الصوتي الثاني. مطلوبة عندما يكون الوضع مضبوطًا على `"two_speakers"`. |
| `mask_1` | MASK | لا | - | قناع المتحدث الأول، مطلوب في حالة استخدام مدخلين صوتيين. |
| `mask_2` | MASK | لا | - | قناع المتحدث الثاني، مطلوب في حالة استخدام مدخلين صوتيين. |

**قيود المعاملات:**

* عندما يكون الوضع مضبوطًا على `"two_speakers"`، تصبح المعاملات `audio_encoder_output_2` و `mask_1` و `mask_2` إلزامية.
* إذا تم توفير `audio_encoder_output_2`، يجب أيضًا توفير كل من `mask_1` و `mask_2`.
* إذا تم توفير `mask_1` و `mask_2`، يجب أيضًا توفير `audio_encoder_output_2`.
* إذا تم توفير `previous_frames`، يجب أن يحتوي على عدد إطارات لا يقل عن العدد المحدد في `motion_frame_count`.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|---------------|---------|
| `model` | MODEL | النموذج المُصحّح مع تطبيق التكييف الصوتي. |
| `positive` | CONDITIONING | التكييف الإيجابي، وربما تم تعديله بسياق إضافي (مثل صورة البداية، رؤية CLIP). |
| `negative` | CONDITIONING | التكييف السلبي، وربما تم تعديله بسياق إضافي. |
| `latent` | LATENT | تسلسل الفيديو المُولّد في الفضاء الكامن. |
| `trim_image` | INT | عدد الإطارات من بداية سياق الحركة التي يجب قصها عند تمديد تسلسل. |
