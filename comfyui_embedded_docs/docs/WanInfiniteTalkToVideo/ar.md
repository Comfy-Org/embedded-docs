# ترجمة وثيقة عقدة WanInfiniteTalkToVideo

تقوم عقدة WanInfiniteTalkToVideo بتوليد تسلسلات فيديو من المدخلات الصوتية. تستخدم نموذج انتشار فيديو، معتمدًا على السمات الصوتية المستخرجة من متحدث واحد أو اثنين، لإنتاج تمثيل كامن لفيديو وجه متحدث. يمكن للعقدة توليد تسلسل جديد أو تمديد تسلسل موجود باستخدام الإطارات السابقة لسياق الحركة.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|---------|---------------|--------|---------|-------|
| `الوضع` | COMBO | نعم | `"single_speaker"`<br>`"two_speakers"` | وضع الإدخال الصوتي. يستخدم `"single_speaker"` مدخلاً صوتيًا واحدًا. يتيح `"two_speakers"` إدخالات لمتحدث ثانٍ والأقنعة المقابلة. |
| `النموذج` | MODEL | نعم | - | نموذج انتشار الفيديو الأساسي. |
| `تصحيح النموذج` | MODELPATCH | نعم | - | تصحيح النموذج الذي يحتوي على طبقات الإسقاط الصوتي. |
| `إيجابي` | CONDITIONING | نعم | - | التكييف الإيجابي لتوجيه التوليد. |
| `سلبي` | CONDITIONING | نعم | - | التكييف السلبي لتوجيه التوليد. |
| `vae` | VAE | نعم | - | التشفير التلقائي المتغير المستخدم لتشفير الصور من وإلى الفضاء الكامن. |
| `العرض` | INT | لا | 16 - MAX_RESOLUTION | عرض الفيديو الناتج بالبكسل. يجب أن يكون قابلاً للقسمة على 16. (الافتراضي: 832) |
| `الارتفاع` | INT | لا | 16 - MAX_RESOLUTION | ارتفاع الفيديو الناتج بالبكسل. يجب أن يكون قابلاً للقسمة على 16. (الافتراضي: 480) |
| `الطول` | INT | لا | 1 - MAX_RESOLUTION | عدد الإطارات المراد توليدها. (الافتراضي: 81) |
| `مخرجات clip للرؤية` | CLIPVISIONOUTPUT | لا | - | مخرجات رؤية CLIP اختيارية لتكييف إضافي. |
| `صورة البداية` | IMAGE | لا | - | صورة بداية اختيارية لتهيئة تسلسل الفيديو. |
| `مخرجات ترميز الصوت ١` | AUDIOENCODEROUTPUT | نعم | - | مخرجات المشفر الصوتي الأساسي التي تحتوي على سمات المتحدث الأول. |
| `عدد إطارات الحركة` | INT | لا | 1 - 33 | عدد الإطارات السابقة المستخدمة كسياق حركة عند تمديد تسلسل. (الافتراضي: 9) |
| `مقياس الصوت` | FLOAT | لا | -10.0 - 10.0 | عامل قياس يُطبق على التكييف الصوتي. (الافتراضي: 1.0) |
| `الإطارات السابقة` | IMAGE | لا | - | إطارات فيديو سابقة اختيارية للتمديد منها. |
| `audio_encoder_output_2` | AUDIOENCODEROUTPUT | لا | - | مخرجات المشفر الصوتي الثاني. مطلوبة عندما يكون الوضع مضبوطًا على `"two_speakers"`. |
| `mask_1` | MASK | لا | - | قناع المتحدث الأول، مطلوب في حالة استخدام مدخلين صوتيين. |
| `mask_2` | MASK | لا | - | قناع المتحدث الثاني، مطلوب في حالة استخدام مدخلين صوتيين. |

**قيود المعاملات:**

* عندما يكون الوضع مضبوطًا على `"two_speakers"`، تصبح المعاملات `audio_encoder_output_2` و `mask_1` و `mask_2` إلزامية.
* إذا تم توفير `audio_encoder_output_2`، يجب أيضًا توفير كل من `mask_1` و `mask_2`.
* إذا تم توفير `mask_1` و `mask_2`، يجب أيضًا توفير `audio_encoder_output_2`.
* إذا تم توفير `previous_frames`، يجب أن يحتوي على عدد إطارات لا يقل عن العدد المحدد في `motion_frame_count`.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|---------------|---------|
| `إيجابي` | MODEL | النموذج المُصحّح مع تطبيق التكييف الصوتي. |
| `سلبي` | CONDITIONING | التكييف الإيجابي، وربما تم تعديله بسياق إضافي (مثل صورة البداية، رؤية CLIP). |
| `الفضاء الكامن` | CONDITIONING | التكييف السلبي، وربما تم تعديله بسياق إضافي. |
| `قص الصورة` | LATENT | تسلسل الفيديو المُولّد في الفضاء الكامن. |
| `trim_image` | INT | عدد الإطارات من بداية سياق الحركة التي يجب قصها عند تمديد تسلسل. |
