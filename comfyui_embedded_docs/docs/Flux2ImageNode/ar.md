نظرة عامة
---

توليد الصور باستخدام نموذج Flux.2 [pro] أو Flux.2 [max] من خلال نص وصفي (برومبت) وصور مرجعية اختيارية. ترسل هذه العقدة طلبك إلى واجهة برمجة تطبيقات BFL، وتستقصي النتيجة، وتُعيد الصورة المُنشأة كمُوتِّر (tensor).

المدخلات
---

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
| :--- | :--- | :--- | :--- | :--- |
| `prompt` | STRING | نعم | غير متاح | النص الوصفي لتوليد الصورة أو تحريرها (القيمة الافتراضية: سلسلة فارغة). |
| `model` | COMBO | نعم | `"Flux.2 [pro]"`<br>`"Flux.2 [max]"` | إصدار نموذج Flux.2 المراد استخدامه. يؤدي اختيار نموذج إلى فتح معاملات إضافية للعرض والارتفاع والصور المرجعية الاختيارية. |
| `seed` | INT | نعم | 0 إلى 18446744073709551615 | البذرة العشوائية المستخدمة لإنشاء الضوضاء. يمكن ضبطها لتكون عشوائية بعد كل عملية توليد (القيمة الافتراضية: 0). |

**المعاملات الإضافية (يتم فتحها باختيار `model`):**

عند تحديد نموذج، تصبح المعاملات التالية متاحة:

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
| :--- | :--- | :--- | :--- | :--- |
| `model.width` | INT | نعم | 256 إلى 1440 | عرض الصورة المُنشأة بالبكسل. |
| `model.height` | INT | نعم | 256 إلى 1440 | ارتفاع الصورة المُنشأة بالبكسل. |
| `model.images` | IMAGE | لا | 0 إلى 8 صور | صور مرجعية اختيارية لتوجيه عملية التوليد. الحد الأقصى المدعوم هو 8 صور. |

**القيود:**
- الحد الأقصى لعدد الصور المرجعية هو 8. إذا تم توفير أكثر من 8 صور، فسيتم إظهار خطأ.
- تؤثر قيمتا `model.width` و `model.height` على تكلفة التوليد (راجع منطق شارة السعر في الكود المصدري).
