### نمای کلی

گره بزرگ‌نمایی دقیق تصویر (نسخه دقیق V2) با کنترل دقیق بر وضوح، دانه‌بندی و افزایش جزئیات، تصاویر را با وفاداری بالا بزرگ‌نمایی می‌کند. این فرآیند از طریق یک API خارجی انجام می‌شود و حداکثر رزولوشن خروجی ۱۰۰۶۰×۱۰۰۶۰ پیکسل را پشتیبانی می‌کند. این گره سبک‌های پردازشی مختلفی ارائه می‌دهد و در صورتی که ابعاد خروجی درخواستی از حداکثر اندازه مجاز فراتر رود، می‌تواند به‌طور خودکار ورودی را کاهش دهد.

## ورودی‌ها

| پارامتر | نوع داده | ضروری | محدوده | توضیحات |
|---------|----------|-------|--------|---------|
| `تصویر` | IMAGE | بله | - | تصویر ورودی برای بزرگ‌نمایی. دقیقاً یک تصویر مورد نیاز است. حداقل ابعاد ۱۶۰×۱۶۰ پیکسل است. نسبت ابعاد باید بین ۱:۳ و ۳:۱ باشد. |
| `ضریب بزرگ‌نمایی` | STRING | بله | `"2x"`<br>`"4x"`<br>`"8x"`<br>`"16x"` | ضریب بزرگ‌نمایی مورد نظر. |
| `سبک پردازش` | STRING | بله | `"sublime"`<br>`"photo"`<br>`"photo_denoiser"` | سبک پردازش. "sublime" برای استفاده عمومی، "photo" بهینه‌شده برای عکس‌ها، و "photo_denoiser" برای عکس‌های نویزدار است. |
| `وضوح` | INT | خیر | ۰ تا ۱۰۰ | شدت وضوح‌بخشی تصویر را برای افزایش تعریف لبه‌ها و شفافیت کنترل می‌کند. مقادیر بالاتر نتیجه وضوح‌تری ایجاد می‌کنند. پیش‌فرض: ۷. |
| `افزایش هوشمند دانه‌بندی` | INT | خیر | ۰ تا ۱۰۰ | دانه‌بندی یا بهبود بافت هوشمند را اضافه می‌کند تا از بیش از حد صاف یا مصنوعی به نظر رسیدن تصویر بزرگ‌نمایی‌شده جلوگیری کند. پیش‌فرض: ۷. |
| `جزئیات فوق‌العاده` | INT | خیر | ۰ تا ۱۰۰ | میزان جزئیات ریز، بافت‌ها و ریزجزئیات اضافه‌شده در فرآیند بزرگ‌نمایی را کنترل می‌کند. پیش‌فرض: ۳۰. |
| `کاهش خودکار ابعاد` | BOOLEAN | خیر | - | در صورت فعال بودن، اگر ابعاد خروجی محاسبه‌شده از حداکثر رزولوشن مجاز ۱۰۰۶۰×۱۰۰۶۰ پیکسل فراتر رود، گره به‌طور خودکار تصویر ورودی را کاهش می‌دهد. این کار از بروز خطا جلوگیری می‌کند اما ممکن است کیفیت را تحت تأثیر قرار دهد. پیش‌فرض: False. |

**توجه:** اگر `auto_downscale` غیرفعال باشد و اندازه خروجی درخواستی (ابعاد ورودی × `scale_factor`) از ۱۰۰۶۰×۱۰۰۶۰ پیکسل فراتر رود، گره یک خطا ایجاد می‌کند.
