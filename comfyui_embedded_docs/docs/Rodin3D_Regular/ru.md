Вот перевод документации на русский язык:

Узел Rodin 3D Regular генерирует 3D-активы с помощью API Rodin. Он принимает входные изображения и обрабатывает их через сервис Rodin для создания 3D-моделей. Узел управляет всем рабочим процессом: от создания задачи до загрузки готовых файлов 3D-моделей.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `Images` | IMAGE | Да | - | Входные изображения, используемые для генерации 3D-модели. Можно указать несколько изображений. |
| `Seed` | INT | Да | - | Значение зерна случайной последовательности для воспроизводимых результатов. |
| `Material_Type` | STRING | Да | - | Тип материала, применяемого к 3D-модели. |
| `Polygon_count` | STRING | Да | - | Целевое количество полигонов для создаваемой 3D-модели. Этот параметр определяет уровень качества и сложность сетки. |

## Выходные данные

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `3D Model Path` | STRING | Путь к файлу сгенерированной 3D-модели (сохранён для обратной совместимости). |
| `GLB` | FILE3DGLB | Сгенерированная 3D-модель в формате GLB. |
