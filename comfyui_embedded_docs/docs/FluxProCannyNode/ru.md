Вот перевод документации на русский язык:

Генерирует изображение с использованием управляющего изображения (canny). Этот узел принимает управляющее изображение и создает новое изображение на основе предоставленного запроса, следуя структуре границ, обнаруженной в управляющем изображении.

## Входные данные

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `control_image` | IMAGE | Да | - | Входное изображение, используемое для управления обнаружением границ методом Canny |
| `prompt` | STRING | Нет | - | Запрос для генерации изображения (по умолчанию: пустая строка) |
| `prompt_upsampling` | BOOLEAN | Нет | - | Выполнять ли повышение разрешения запроса. Если активно, автоматически изменяет запрос для более творческой генерации, но результаты недетерминированы (одинаковое зерно не даст точно такого же результата). (по умолчанию: False) |
| `canny_low_threshold` | FLOAT | Нет | 0.01 - 0.99 | Нижний порог для обнаружения границ методом Canny; игнорируется, если `skip_preprocessing` равно True (по умолчанию: 0.1) |
| `canny_high_threshold` | FLOAT | Нет | 0.01 - 0.99 | Верхний порог для обнаружения границ методом Canny; игнорируется, если `skip_preprocessing` равно True (по умолчанию: 0.4) |
| `skip_preprocessing` | BOOLEAN | Нет | - | Пропускать ли предварительную обработку; установите True, если `control_image` уже является Canny-изображением, False — если это исходное изображение. (по умолчанию: False) |
| `guidance` | FLOAT | Нет | 1 - 100 | Сила управления для процесса генерации изображения (по умолчанию: 30) |
| `steps` | INT | Нет | 15 - 50 | Количество шагов для процесса генерации изображения (по умолчанию: 50) |
| `seed` | INT | Нет | 0 - 18446744073709551615 | Случайное зерно, используемое для создания шума. (по умолчанию: 0) |

**Примечание:** Когда `skip_preprocessing` установлено в True, параметры `canny_low_threshold` и `canny_high_threshold` игнорируются, так как предполагается, что управляющее изображение уже обработано как Canny-изображение границ. В этом случае `control_image` используется напрямую как предварительно обработанное изображение.

## Выходные данные

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `output_image` | IMAGE | Сгенерированное изображение на основе управляющего изображения и запроса |
