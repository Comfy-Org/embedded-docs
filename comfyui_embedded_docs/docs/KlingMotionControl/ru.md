Вот перевод документации на русский язык:

Узел Kling Motion Control генерирует видео, применяя движения, мимику и перемещения камеры из эталонного видео к персонажу, заданному эталонным изображением и текстовым запросом. Он позволяет управлять тем, берется ли финальная ориентация персонажа из эталонного видео или из эталонного изображения.

## Входные данные

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `промпт` | STRING | Да | Н/Д | Текстовое описание желаемого видео. Максимальная длина — 2500 символов. |
| `референсное_изображение` | IMAGE | Да | Н/Д | Изображение персонажа для анимации. Минимальные размеры — 340x340 пикселей. Соотношение сторон должно быть от 1:2,5 до 2,5:1. |
| `референсное_видео` | VIDEO | Да | Н/Д | Эталонное видео с движениями, используемое для управления движениями и мимикой персонажа. Минимальные размеры — 340x340 пикселей, максимальные — 3850x3850 пикселей. Ограничения по длительности зависят от настройки `ориентация_персонажа`. |
| `сохранить_оригинальный_звук` | BOOLEAN | Нет | Н/Д | Определяет, сохраняется ли оригинальный звук из эталонного видео в выходном файле. По умолчанию — `True`. |
| `ориентация_персонажа` | COMBO | Нет | `"video"`<br>`"image"` | Управляет источником ориентации/направления персонажа. `"video"`: движения, мимика, перемещения камеры и ориентация следуют эталонному видео (остальные детали — из запроса). `"image"`: движения и мимика по-прежнему следуют эталонному видео, но ориентация персонажа соответствует эталонному изображению (камера и остальные детали — из запроса). |
| `режим` | COMBO | Нет | `"pro"`<br>`"std"` | Используемый режим генерации. |
| `модель` | COMBO | Нет | `"kling-v3"`<br>`"kling-v2-6"` | Версия модели Kling для использования. По умолчанию — `"kling-v2-6"`. |

**Ограничения:**

* Длительность `reference_video` должна быть от 3 до 30 секунд, когда `character_orientation` установлено в `"video"`.
* Длительность `reference_video` должна быть от 3 до 10 секунд, когда `character_orientation` установлено в `"image"`.

## Выходные данные

| Имя выходных данных | Тип данных | Описание |
|---------------------|------------|----------|
| `output` | VIDEO | Сгенерированное видео с персонажем, выполняющим движения из эталонного видео. |
