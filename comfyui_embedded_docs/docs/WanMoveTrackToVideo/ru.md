Вот перевод документации узла WanMoveTrackToVideo на русский язык:

Узел WanMoveTrackToVideo подготавливает данные для кондиционирования и латентного пространства для генерации видео, опционально используя информацию об отслеживании движения. Он кодирует начальную последовательность изображений в латентное представление и может объединять данные о положении из треков объектов для управления движением в генерируемом видео. Узел выводит модифицированные положительное и отрицательное кондиционирования, а также пустой латентный тензор, готовый для использования в видеомодели.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `positive` | CONDITIONING | Да | - | Входное положительное кондиционирование, которое будет модифицировано. |
| `negative` | CONDITIONING | Да | - | Входное отрицательное кондиционирование, которое будет модифицировано. |
| `vae` | VAE | Да | - | Модель VAE, используемая для кодирования начального изображения в латентное пространство. |
| `треки` | TRACKS | Нет | - | Опциональные данные отслеживания движения, содержащие пути объектов. |
| `интенсивность` | FLOAT | Нет | 0.0 - 100.0 | Сила воздействия треков на кондиционирование. (по умолчанию: 1.0) |
| `ширина` | INT | Нет | 16 - MAX_RESOLUTION | Ширина выходного видео. Должна быть кратна 16. (по умолчанию: 832) |
| `высота` | INT | Нет | 16 - MAX_RESOLUTION | Высота выходного видео. Должна быть кратна 16. (по умолчанию: 480) |
| `длина` | INT | Нет | 1 - MAX_RESOLUTION | Количество кадров в видеопоследовательности. (по умолчанию: 81) |
| `размер_пакета` | INT | Нет | 1 - 4096 | Размер пакета для латентного выхода. (по умолчанию: 1) |
| `стартовое_изображение` | IMAGE | Да | - | Начальное изображение или последовательность изображений для кодирования. |
| `clip_vision_output` | CLIPVISIONOUTPUT | Нет | - | Опциональный выход модели CLIP vision для добавления к кондиционированию. |

**Примечание:** Параметр `strength` действует только при наличии `tracks`. Если `tracks` не предоставлены или `strength` равен 0.0, кондиционирование треками не применяется. `start_image` используется для создания латентного изображения и маски для кондиционирования; если он не предоставлен, узел только пропускает кондиционирование и выводит пустой латент.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `positive` | CONDITIONING | Модифицированное положительное кондиционирование, потенциально содержащее `concat_latent_image`, `concat_mask` и `clip_vision_output`. |
| `negative` | CONDITIONING | Модифицированное отрицательное кондиционирование, потенциально содержащее `concat_latent_image`, `concat_mask` и `clip_vision_output`. |
| `latent` | LATENT | Пустой латентный тензор с размерами, заданными параметрами `размер_пакета`, `длина`, `высота` и `ширина`. |
