Вот перевод документации узла ComfyUI Wan22FunControlToVideo на русский язык:

Узел Wan22FunControlToVideo подготавливает представления conditioning и скрытого пространства (latent) для генерации видео с использованием архитектуры видео-модели Wan. Он обрабатывает положительные и отрицательные входные данные conditioning вместе с опциональными референсными изображениями и управляющими видео для создания необходимых представлений скрытого пространства для синтеза видео. Узел выполняет пространственное масштабирование и обработку временных измерений для генерации соответствующих данных conditioning для видео-моделей.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `positive` | CONDITIONING | Да | - | Положительный вход conditioning для направления генерации видео |
| `negative` | CONDITIONING | Да | - | Отрицательный вход conditioning для направления генерации видео |
| `vae` | VAE | Да | - | Модель VAE, используемая для кодирования изображений в скрытое пространство |
| `width` | INT | Да | от 16 до MAX_RESOLUTION | Ширина выходного видео в пикселях (по умолчанию: 832, шаг: 16) |
| `height` | INT | Да | от 16 до MAX_RESOLUTION | Высота выходного видео в пикселях (по умолчанию: 480, шаг: 16) |
| `length` | INT | Да | от 1 до MAX_RESOLUTION | Количество кадров в видеопоследовательности (по умолчанию: 81, шаг: 4) |
| `batch_size` | INT | Да | от 1 до 4096 | Количество видеопоследовательностей для генерации (по умолчанию: 1) |
| `ref_image` | IMAGE | Нет | - | Опциональное референсное изображение для предоставления визуального ориентира |
| `control_video` | IMAGE | Нет | - | Опциональное управляющее видео для направления процесса генерации |

**Примечание:** Параметр `length` обрабатывается блоками по 4 кадра, и узел автоматически выполняет временное масштабирование для скрытого пространства. Когда предоставлено `ref_image`, оно влияет на conditioning через референсные скрытые представления. Когда предоставлено `control_video`, оно напрямую влияет на конкатенированное скрытое представление, используемое в conditioning. Параметр `start_image` не представлен как входной в схеме этого узла, но на него есть ссылка в логике выполнения.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `positive` | CONDITIONING | Модифицированное положительное conditioning с данными скрытого пространства, специфичными для видео, включая конкатенированное скрытое представление, маску и опциональные референсные скрытые представления |
| `negative` | CONDITIONING | Модифицированное отрицательное conditioning с данными скрытого пространства, специфичными для видео, включая конкатенированное скрытое представление, маску и опциональные референсные скрытые представления |
| `latent` | LATENT | Пустой тензор скрытого пространства с соответствующими размерами для генерации видео на основе размера пакета, каналов скрытого пространства и пространственного/временного масштабирования |
