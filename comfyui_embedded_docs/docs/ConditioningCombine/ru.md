Этот узел объединяет два входа conditioning в один выход, эффективно объединяя их информацию. Два условия комбинируются с помощью конкатенации списков.

## Входы

| Имя параметра        | Тип данных          | Описание |
|----------------------|---------------------|----------|
| `conditioning_1`     | `CONDITIONING`      | Первый вход conditioning для объединения. Он имеет равную важность с `conditioning_2` в процессе комбинирования. |
| `conditioning_2`     | `CONDITIONING`      | Второй вход conditioning для объединения. Он имеет равную важность с `conditioning_1` в процессе комбинирования. |

## Выходы

| Имя параметра        | Тип данных          | Описание |
|----------------------|---------------------|----------|
| `conditioning`       | `CONDITIONING`      | Результат объединения `conditioning_1` и `conditioning_2`, содержащий объединённую информацию. |

## Сценарии использования

Сравните две группы ниже: левая сторона использует узел ConditioningCombine, в то время как правая сторона показывает обычный выход.

![Сравнение](./asset/compare.jpg)

В этом примере два условия, используемые в `Conditioning Combine`, имеют одинаковую важность. Поэтому вы можете использовать разные текстовые кодировки для стиля изображения, характеристик объекта и т.д., что позволяет более полно выводить признаки промпта. Второй промпт использует объединённый полный промпт, но семантическое понимание может кодировать совершенно разные условия.

Используя этот узел, вы можете достичь:

- Базового объединения текста: Подключите выходы двух узлов `CLIP Text Encode` к двум входным портам `Conditioning Combine`
- Комбинирования сложных промптов: Объедините позитивные и негативные промпты или отдельно кодируйте основные описания и описания стиля перед объединением
- Цепочки комбинирования условий: Несколько узлов `Conditioning Combine` могут быть использованы последовательно для достижения постепенного объединения нескольких условий
