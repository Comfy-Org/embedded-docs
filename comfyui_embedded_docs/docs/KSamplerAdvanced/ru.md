Узел KSamplerAdvanced предназначен для улучшения процесса семплирования за счет расширенных конфигураций и методов. Он предлагает более сложные опции для генерации образцов из модели, расширяя функциональность базового KSampler.

## Входные данные

| Параметр | Тип данных | Описание |
|----------------------|-------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `model` | MODEL | Указывает модель, из которой будут генерироваться образцы, играя ключевую роль в процессе семплирования. |
| `add_noise` | COMBO[STRING] | Определяет, следует ли добавлять шум в процесс семплирования, влияя на разнообразие и качество генерируемых образцов. |
| `noise_seed` | INT | Задает зерно для генерации шума, обеспечивая воспроизводимость в процессе семплирования. |
| `steps` | INT | Определяет количество шагов, выполняемых в процессе семплирования, влияя на детализацию и качество выходных данных. |
| `cfg` | FLOAT | Управляет коэффициентом кондиционирования, влияя на направление и пространство процесса семплирования. |
| `sampler_name` | COMBO[STRING] | Выбирает конкретный семплер для использования, позволяя настраивать технику семплирования. |
| `scheduler` | COMBO[STRING] | Выбирает планировщик для управления процессом семплирования, влияя на прогрессию и качество образцов. |
| `positive` | CONDITIONING | Задает положительное кондиционирование для направления семплирования к желаемым атрибутам. |
| `negative` | CONDITIONING | Задает отрицательное кондиционирование для отвода семплирования от определенных атрибутов. |
| `latent_image` | LATENT | Предоставляет начальное латентное изображение для использования в процессе семплирования, служащее отправной точкой. |
| `start_at_step` | INT | Определяет начальный шаг процесса семплирования, позволяя контролировать его прогрессию. |
| `end_at_step` | INT | Устанавливает конечный шаг процесса семплирования, определяя его область действия. |
| `return_with_leftover_noise` | COMBO[STRING] | Указывает, следует ли возвращать образец с остаточным шумом, что влияет на внешний вид конечного результата. |

## Выходные данные

| Параметр | Тип данных | Описание |
|-------------|-------------|------------------------------------------------------------------------------------------------------------------------------|
| `latent` | LATENT | Выходные данные представляют собой латентное изображение, сгенерированное из модели, отражающее примененные конфигурации и методы. |
