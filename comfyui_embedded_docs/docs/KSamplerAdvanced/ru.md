Узел KSamplerAdvanced предназначен для улучшения процесса семплирования за счет расширенных конфигураций и методов. Он предлагает более сложные опции для генерации образцов из модели, расширяя функциональность базового KSampler.

## Входные данные

| Параметр | Тип данных | Описание |
|----------------------|-------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `модель` | MODEL | Указывает модель, из которой будут генерироваться образцы, играя ключевую роль в процессе семплирования. |
| `добавить_шум` | COMBO[STRING] | Определяет, следует ли добавлять шум в процесс семплирования, влияя на разнообразие и качество генерируемых образцов. |
| `сид_шума` | INT | Задает зерно для генерации шума, обеспечивая воспроизводимость в процессе семплирования. |
| `шаги` | INT | Определяет количество шагов, выполняемых в процессе семплирования, влияя на детализацию и качество выходных данных. |
| `cfg` | FLOAT | Управляет коэффициентом кондиционирования, влияя на направление и пространство процесса семплирования. |
| `название_сэмплера` | COMBO[STRING] | Выбирает конкретный семплер для использования, позволяя настраивать технику семплирования. |
| `scheduler` | COMBO[STRING] | Выбирает планировщик для управления процессом семплирования, влияя на прогрессию и качество образцов. |
| `положительный` | CONDITIONING | Задает положительное кондиционирование для направления семплирования к желаемым атрибутам. |
| `отрицательный` | CONDITIONING | Задает отрицательное кондиционирование для отвода семплирования от определенных атрибутов. |
| `латентное_изображение` | LATENT | Предоставляет начальное латентное изображение для использования в процессе семплирования, служащее отправной точкой. |
| `начать_с_шага` | INT | Определяет начальный шаг процесса семплирования, позволяя контролировать его прогрессию. |
| `закончить_на_шаге` | INT | Устанавливает конечный шаг процесса семплирования, определяя его область действия. |
| `вернуться_с_оставшимся_шумом` | COMBO[STRING] | Указывает, следует ли возвращать образец с остаточным шумом, что влияет на внешний вид конечного результата. |

## Выходные данные

| Параметр | Тип данных | Описание |
|-------------|-------------|------------------------------------------------------------------------------------------------------------------------------|
| `latent` | LATENT | Выходные данные представляют собой латентное изображение, сгенерированное из модели, отражающее примененные конфигурации и методы. |
