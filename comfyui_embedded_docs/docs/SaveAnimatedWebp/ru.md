Этот узел предназначен для сохранения последовательности изображений в виде анимированного файла WEBP. Он выполняет объединение отдельных кадров в целостную анимацию, применяет указанные метаданные и оптимизирует вывод на основе настроек качества и сжатия.

## Входы

| Поле              | Тип данных | Описание                                                                         |
|-------------------|-------------|-------------------------------------------------------------------------------------|
| `изображения`          | `IMAGE`     | Список изображений, сохраняемых как кадры анимированного WEBP. Этот параметр необходим для определения визуального содержимого анимации. |
| `префикс_названия_файла` | `STRING`    | Задает базовое имя выходного файла, к которому будет добавлен счетчик и расширение '.webp'. Этот параметр важен для идентификации и организации сохраняемых файлов. |
| `fps`             | `FLOAT`     | Частота кадров в секунду для анимации, влияющая на скорость воспроизведения. |
| `без_потерь`        | `BOOLEAN`   | Логическое значение, указывающее, следует ли использовать сжатие без потерь, что влияет на размер файла и качество анимации. |
| `качество`         | `INT`       | Значение от 0 до 100, задающее уровень качества сжатия: более высокие значения обеспечивают лучшее качество изображения, но увеличивают размер файла. |
| `метод`          | COMBO[STRING] | Определяет метод сжатия, который может влиять на скорость кодирования и размер файла. |

## Выходы

| Поле | Тип данных | Описание                                                                       |
|-------|-------------|-----------------------------------------------------------------------------------|
| `ui`  | N/A         | Предоставляет компонент пользовательского интерфейса, отображающий сохраненные анимированные изображения WEBP вместе с их метаданными, а также указывающий, включена ли анимация. |
