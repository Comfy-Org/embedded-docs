# ترجمة وثيقة عقدة MeshyMultiImageToModelNode

تستخدم هذه العقدة واجهة برمجة تطبيقات Meshy لإنشاء نموذج ثلاثي الأبعاد من صور إدخال متعددة. تقوم برفع الصور المقدمة، وإرسال مهمة معالجة، وإرجاع ملفات النموذج ثلاثي الأبعاد الناتجة (GLB و FBX) بالإضافة إلى معرف المهمة للرجوع إليه.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
| :--- | :--- | :--- | :--- | :--- |
| `model` | COMBO | نعم | `"latest"` | يحدد إصدار نموذج الذكاء الاصطناعي المراد استخدامه. |
| `images` | IMAGE | نعم | من 2 إلى 4 صور | مجموعة من الصور المستخدمة لإنشاء النموذج ثلاثي الأبعاد. يجب توفير ما بين 2 و 4 صور. |
| `should_remesh` | COMBO | نعم | `"true"`<br>`"false"` | يحدد ما إذا كان يجب معالجة الشبكة المُنشأة. عند تعيينه على `"false"`، تُرجع العقدة شبكة مثلثية غير معالجة. |
| `topology` | COMBO | لا | `"triangle"`<br>`"quad"` | نوع المضلع المستهدف لإخراج إعادة التقسيم. هذا المعامل متاح ومطلوب فقط عندما يتم تعيين `should_remesh` على `"true"`. |
| `target_polycount` | INT | لا | 100 إلى 300000 | العدد المستهدف من المضلعات لنموذج إعادة التقسيم (الافتراضي: 300000). هذا المعامل متاح فقط عندما يتم تعيين `should_remesh` على `"true"`. |
| `symmetry_mode` | COMBO | نعم | `"auto"`<br>`"on"`<br>`"off"` | يتحكم في تطبيق التماثل على النموذج المُنشأ. |
| `should_texture` | COMBO | نعم | `"true"`<br>`"false"` | يحدد ما إذا كان سيتم إنشاء القوام. تعيينه على `"false"` يتجاوز مرحلة القوام ويعيد شبكة بدون قوام. |
| `enable_pbr` | BOOLEAN | لا | True / False | عندما يكون `should_texture` على `"true"`، يُنشئ هذا الخيار خرائط PBR (معدني، خشونة، عادي) بالإضافة إلى اللون الأساسي (الافتراضي: False). |
| `texture_prompt` | STRING | لا | - | موجه نصي لتوجيه عملية إنشاء القوام (بحد أقصى 600 حرف). لا يمكن استخدامه في نفس وقت استخدام `texture_image`. هذا المعامل متاح فقط عندما يتم تعيين `should_texture` على `"true"`. |
| `texture_image` | IMAGE | لا | - | صورة لتوجيه عملية إنشاء القوام. يمكن استخدام واحد فقط من `texture_image` أو `texture_prompt` في نفس الوقت. هذا المعامل متاح فقط عندما يتم تعيين `should_texture` على `"true"`. |
| `pose_mode` | COMBO | نعم | `""` (فارغ)<br>`"A-pose"`<br>`"T-pose"` | يحدد وضعية النموذج المُنشأ. |
| `seed` | INT | نعم | 0 إلى 2147483647 | قيمة بذرة لعملية الإنشاء (الافتراضي: 0). النتائج غير حتمية بغض النظر عن البذرة، لكن تغيير البذرة يمكن أن يؤدي إلى إعادة تشغيل العقدة. |

**قيود المعاملات:**

* يجب توفير ما بين 2 و 4 صور لمدخل `images`.
* معاملا `topology` و `target_polycount` نشطان فقط عندما يتم تعيين `should_remesh` على `"true"`.
* معاملات `enable_pbr` و `texture_prompt` و `texture_image` نشطة فقط عندما يتم تعيين `should_texture` على `"true"`.
* لا يمكن استخدام `texture_prompt` و `texture_image` في نفس الوقت؛ فهما متنافيان.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
| :--- | :--- | :--- |
| `model_file` | STRING | اسم ملف نموذج GLB المُنشأ. يتم توفير هذا المخرج للتوافق مع الإصدارات السابقة. |
| `meshy_task_id` | MESHY_TASK_ID | المعرف الفريد لمهمة واجهة برمجة تطبيقات Meshy. |
| `GLB` | FILE3DGLB | النموذج ثلاثي الأبعاد المُنشأ بتنسيق GLB. |
| `FBX` | FILE3DFBX | النموذج ثلاثي الأبعاد المُنشأ بتنسيق FBX. |
