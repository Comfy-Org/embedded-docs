این گره از API Meshy برای تولید یک مدل سه‌بعدی از چند تصویر ورودی استفاده می‌کند. تصاویر ارائه‌شده را بارگذاری می‌کند، یک وظیفه پردازشی ارسال می‌کند و فایل‌های مدل سه‌بعدی حاصل (GLB و FBX) را به همراه شناسه وظیفه برای مرجع بازمی‌گرداند.

## ورودی‌ها

| پارامتر | نوع داده | ضروری | محدوده | توضیحات |
| :--- | :--- | :--- | :--- | :--- |
| `model` | COMBO | بله | `"latest"` | نسخه مدل هوش مصنوعی مورد استفاده را مشخص می‌کند. |
| `images` | IMAGE | بله | ۲ تا ۴ تصویر | مجموعه‌ای از تصاویر که برای تولید مدل سه‌بعدی استفاده می‌شود. باید بین ۲ تا ۴ تصویر ارائه دهید. |
| `should_remesh` | COMBO | بله | `"true"`<br>`"false"` | تعیین می‌کند که آیا مش تولیدشده باید پردازش شود. وقتی روی `"false"` تنظیم شود، گره یک مش مثلثی پردازش‌نشده بازمی‌گرداند. |
| `topology` | COMBO | خیر | `"triangle"`<br>`"quad"` | نوع چندضلعی هدف برای خروجی مش‌بندی‌شده. این پارامتر فقط زمانی در دسترس و ضروری است که `should_remesh` روی `"true"` تنظیم شده باشد. |
| `target_polycount` | INT | خیر | ۱۰۰ تا ۳۰۰۰۰۰ | تعداد چندضلعی‌های هدف برای مدل مش‌بندی‌شده (پیش‌فرض: ۳۰۰۰۰۰). این پارامتر فقط زمانی در دسترس است که `should_remesh` روی `"true"` تنظیم شده باشد. |
| `symmetry_mode` | COMBO | بله | `"auto"`<br>`"on"`<br>`"off"` | کنترل می‌کند که آیا تقارن روی مدل تولیدشده اعمال شود. |
| `should_texture` | COMBO | بله | `"true"`<br>`"false"` | تعیین می‌کند که آیا بافت‌ها تولید شوند. تنظیم آن روی `"false"` مرحله بافت‌دهی را رد کرده و یک مش بدون بافت بازمی‌گرداند. |
| `enable_pbr` | BOOLEAN | خیر | True / False | وقتی `should_texture` روی `"true"` باشد، این گزینه نقشه‌های PBR (فلزی، زبری، نرمال) را علاوه بر رنگ پایه تولید می‌کند (پیش‌فرض: False). |
| `texture_prompt` | STRING | خیر | - | یک راهنمای متنی برای هدایت فرآیند بافت‌دهی (حداکثر ۶۰۰ کاراکتر). نمی‌توان همزمان با `texture_image` استفاده شود. این پارامتر فقط زمانی در دسترس است که `should_texture` روی `"true"` تنظیم شده باشد. |
| `texture_image` | IMAGE | خیر | - | یک تصویر برای هدایت فرآیند بافت‌دهی. فقط یکی از `texture_image` یا `texture_prompt` می‌تواند همزمان استفاده شود. این پارامتر فقط زمانی در دسترس است که `should_texture` روی `"true"` تنظیم شده باشد. |
| `pose_mode` | COMBO | بله | `""` (خالی)<br>`"A-pose"`<br>`"T-pose"` | حالت ایستادن را برای مدل تولیدشده مشخص می‌کند. |
| `seed` | INT | بله | ۰ تا ۲۱۴۷۴۸۳۶۴۷ | یک مقدار دانه برای فرآیند تولید (پیش‌فرض: ۰). نتایج صرف‌نظر از دانه، غیرقطعی هستند، اما تغییر دانه می‌تواند باعث اجرای مجدد گره شود. |

**محدودیت‌های پارامتر:**

* باید بین ۲ تا ۴ تصویر برای ورودی `images` ارائه دهید.
* پارامترهای `topology` و `target_polycount` فقط زمانی فعال هستند که `should_remesh` روی `"true"` تنظیم شده باشد.
* پارامترهای `enable_pbr`، `texture_prompt` و `texture_image` فقط زمانی فعال هستند که `should_texture` روی `"true"` تنظیم شده باشد.
* نمی‌توانید همزمان از `texture_prompt` و `texture_image` استفاده کنید؛ این دو متقابلاً انحصاری هستند.

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `model_file` | STRING | نام فایل مدل GLB تولیدشده. این خروجی برای سازگاری با نسخه‌های قبلی ارائه شده است. |
| `meshy_task_id` | MESHY_TASK_ID | شناسه یکتای وظیفه API Meshy. |
| `GLB` | FILE3DGLB | مدل سه‌بعدی تولیدشده در قالب GLB. |
| `FBX` | FILE3DFBX | مدل سه‌بعدی تولیدشده در قالب FBX. |
