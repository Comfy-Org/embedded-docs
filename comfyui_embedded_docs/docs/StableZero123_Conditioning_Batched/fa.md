گره `StableZero123_Conditioning_Batched` یک تصویر ورودی را پردازش کرده و داده‌های شرطی‌سازی (conditioning) برای تولید مدل سه‌بعدی ایجاد می‌کند. این گره با استفاده از مدل‌های CLIP vision و VAE، تصویر را کدگذاری کرده و سپس بر اساس زوایای ارتفاع (elevation) و آزیموت (azimuth)، جاسازی‌های دوربین (camera embeddings) را تولید می‌کند تا شرطی‌سازی مثبت و منفی به همراه نمایش‌های نهفته (latent representations) برای پردازش دسته‌ای (batch processing) ایجاد کند.

## ورودی‌ها

| پارامتر | نوع داده | اجباری | محدوده | توضیحات |
|-----------|-----------|----------|-------|-------------|
| `clip_vision` | CLIP_VISION | بله | - | مدل بینایی CLIP که برای کدگذاری تصویر ورودی استفاده می‌شود |
| `تصویر اولیه` | IMAGE | بله | - | تصویر ورودی اولیه که باید پردازش و کدگذاری شود |
| `vae` | VAE | بله | - | مدل VAE که برای کدگذاری پیکسل‌های تصویر به فضای نهفته (latent space) استفاده می‌شود |
| `عرض` | INT | خیر | 16 تا MAX_RESOLUTION | عرض خروجی برای تصویر پردازش‌شده (پیش‌فرض: ۲۵۶، باید بر ۸ بخش‌پذیر باشد) |
| `ارتفاع` | INT | خیر | 16 تا MAX_RESOLUTION | ارتفاع خروجی برای تصویر پردازش‌شده (پیش‌فرض: ۲۵۶، باید بر ۸ بخش‌پذیر باشد) |
| `اندازه دسته` | INT | خیر | 1 تا 4096 | تعداد نمونه‌های شرطی‌سازی که در هر دسته تولید می‌شود (پیش‌فرض: ۱) |
| `زاویه ارتفاع` | FLOAT | خیر | 180.0- تا 180.0 | زاویه ارتفاع اولیه دوربین بر حسب درجه (پیش‌فرض: 0.0) |
| `زاویه سمت` | FLOAT | خیر | 180.0- تا 180.0 | زاویه آزیموت اولیه دوربین بر حسب درجه (پیش‌فرض: 0.0) |
| `افزایش دسته‌ای ارتفاع` | FLOAT | خیر | 180.0- تا 180.0 | مقدار افزایش ارتفاع برای هر آیتم دسته (پیش‌فرض: 0.0) |
| `افزایش دسته‌ای سمت` | FLOAT | خیر | 180.0- تا 180.0 | مقدار افزایش آزیموت برای هر آیتم دسته (پیش‌فرض: 0.0) |

**نکته:** پارامترهای `width` و `height` باید بر ۸ بخش‌پذیر باشند، زیرا گره به‌صورت داخلی این ابعاد را برای تولید فضای نهفته بر ۸ تقسیم می‌کند.
