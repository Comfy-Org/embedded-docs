Вот перевод документации на русский язык:

Узел ExtendIntermediateSigmas берёт существующую последовательность значений сигма и вставляет между ними дополнительные промежуточные значения сигма. Он позволяет указать, сколько дополнительных шагов добавить, метод интерполяции для распределения значений, а также необязательные начальную и конечную границы сигма, чтобы контролировать, в какой части последовательности сигма происходит расширение.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `sigmas` | SIGMAS | Да | - | Исходная последовательность сигма для расширения промежуточными значениями |
| `steps` | INT | Да | от 1 до 100 | Количество промежуточных шагов для вставки между существующими сигма (по умолчанию: 2) |
| `start_at_sigma` | FLOAT | Да | от -1.0 до 20000.0 | Верхняя граница сигма для расширения — расширять только сигма ниже этого значения (по умолчанию: -1.0, что означает бесконечность) |
| `end_at_sigma` | FLOAT | Да | от 0.0 до 20000.0 | Нижняя граница сигма для расширения — расширять только сигма выше этого значения (по умолчанию: 12.0) |
| `spacing` | COMBO | Да | `"linear"`<br>`"cosine"`<br>`"sine"` | Метод интерполяции для распределения промежуточных значений сигма (по умолчанию: "linear") |

**Примечание:** Узел вставляет промежуточные сигма только между существующими парами сигма, где текущая сигма меньше или равна `start_at_sigma` и больше или равна `end_at_sigma`. Когда `start_at_sigma` установлено в -1.0, это значение трактуется как бесконечность, то есть применяется только нижняя граница `end_at_sigma`.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `sigmas` | SIGMAS | Расширенная последовательность сигма с добавленными дополнительными промежуточными значениями |
