Вот перевод документации на русский язык:

Узел Rodin 3D Detail генерирует детализированные 3D-активы с использованием API Rodin. Он принимает входные изображения и обрабатывает их через сервис Rodin для создания высококачественных 3D-моделей с детальной геометрией и материалами. Узел управляет всем рабочим процессом: от создания задачи до загрузки готового файла 3D-модели.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `Изображения` | IMAGE | Да | - | Входные изображения, используемые для генерации 3D-модели. Можно указать несколько изображений. |
| `Сид` | INT | Да | - | Значение случайного зерна для воспроизводимых результатов |
| `Тип_материала` | STRING | Да | - | Тип материала, применяемого к 3D-модели |
| `Количество_полигонов` | STRING | Да | - | Целевое количество полигонов для создаваемой 3D-модели. Определяет уровень детализации сетки. |

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `3D Model Path` | STRING | Путь к файлу сгенерированной 3D-модели (только для обратной совместимости) |
| `GLB` | FILE3DGLB | Сгенерированная 3D-модель в формате GLB |
