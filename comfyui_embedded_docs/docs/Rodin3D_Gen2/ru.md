Вот перевод документации на русский язык:

Узел Rodin3D_Gen2 генерирует 3D-активы с использованием API Rodin. Он принимает входные изображения и преобразует их в 3D-модели с различными типами материалов и количеством полигонов. Узел автоматически обрабатывает весь процесс генерации, включая создание задачи, опрос статуса и загрузку файлов.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `Images` | IMAGE | Да | - | Входные изображения для генерации 3D-модели |
| `Seed` | INT | Нет | 0-65535 | Случайное начальное значение для генерации (по умолчанию: 0) |
| `Material_Type` | COMBO | Нет | "PBR"<br>"Shaded" | Тип материала, применяемого к 3D-модели (по умолчанию: "PBR") |
| `Polygon_count` | COMBO | Нет | "4K-Quad"<br>"8K-Quad"<br>"18K-Quad"<br>"50K-Quad"<br>"2K-Triangle"<br>"20K-Triangle"<br>"150K-Triangle"<br>"500K-Triangle" | Целевое количество полигонов для создаваемой 3D-модели (по умолчанию: "500K-Triangle") |
| `TAPose` | BOOLEAN | Нет | - | Применять ли обработку TAPose (по умолчанию: False) |

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `3D Model Path` | STRING | Путь к файлу сгенерированной 3D-модели (для обратной совместимости) |
| `GLB` | FILE3DGLB | Сгенерированная 3D-модель в формате GLB |
