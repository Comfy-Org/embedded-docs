### گره WanSCAILToVideo

گره WanSCAILToVideo، آماده‌سازی کاندیشنینگ (شرط‌گذاری) و یک فضای نهفته خالی برای تولید ویدیو انجام می‌دهد. این گره ورودی‌های اختیاری مانند تصاویر مرجع، ویدیوهای حالت (pose) و خروجی‌های CLIP vision را پردازش کرده و آن‌ها را در کاندیشنینگ مثبت و منفی برای یک مدل ویدیو جاسازی می‌کند. خروجی گره شامل کاندیشنینگ اصلاح‌شده و یک تنسور نهفته خالی با ابعاد ویدیوی مشخص‌شده است.

## ورودی‌ها

| پارامتر | نوع داده | ضروری | محدوده | توضیحات |
|-----------|-----------|----------|-------|-------------|
| `positive` | CONDITIONING | بله | - | ورودی کاندیشنینگ مثبت. |
| `negative` | CONDITIONING | بله | - | ورودی کاندیشنینگ منفی. |
| `vae` | VAE | بله | - | مدل VAE مورد استفاده برای رمزگذاری تصاویر و فریم‌های ویدیو. |
| `width` | INT | بله | 32 تا MAX_RESOLUTION | عرض ویدیوی خروجی بر حسب پیکسل (پیش‌فرض: 512). باید بر ۸ بخش‌پذیر باشد. |
| `height` | INT | بله | 32 تا MAX_RESOLUTION | ارتفاع ویدیوی خروجی بر حسب پیکسل (پیش‌فرض: 896). باید بر ۸ بخش‌پذیر باشد. |
| `length` | INT | بله | 1 تا MAX_RESOLUTION | تعداد فریم‌های ویدیو (پیش‌فرض: 81). باید بر ۴ بخش‌پذیر باشد. |
| `batch_size` | INT | بله | 1 تا 4096 | تعداد ویدیوهایی که در یک بسته تولید می‌شوند (پیش‌فرض: 1). |
| `clip_vision_output` | CLIP_VISION_OUTPUT | خیر | - | خروجی اختیاری CLIP vision برای کاندیشنینگ. |
| `reference_image` | IMAGE | خیر | - | تصویر مرجع اختیاری برای کاندیشنینگ. |
| `pose_video` | IMAGE | خیر | - | ویدیوی مورد استفاده برای کاندیشنینگ حالت (pose). به نصف وضوح ویدیوی اصلی کاهش می‌یابد. |
| `pose_strength` | FLOAT | بله | 0.0 تا 10.0 | قدرت نهفته حالت (پیش‌فرض: 1.0). |
| `pose_start` | FLOAT | بله | 0.0 تا 1.0 | مرحله شروع استفاده از کاندیشنینگ حالت (پیش‌فرض: 0.0). |
| `pose_end` | FLOAT | بله | 0.0 تا 1.0 | مرحله پایان استفاده از کاندیشنینگ حالت (پیش‌فرض: 1.0). |

**نکته:** ورودی `pose_video` فقط برای اولین `length` فریم پردازش می‌شود. ورودی `reference_image` فقط برای اولین تصویر در بسته پردازش می‌شود. هنگامی که `reference_image` ارائه می‌شود، یک نهانه پر از صفر با همان اندازه برای کاندیشنینگ منفی استفاده می‌شود. هنگامی که `clip_vision_output` ارائه می‌شود، به هر دو کاندیشنینگ مثبت و منفی اعمال می‌شود.

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
|-------------|-----------|-------------|
| `positive` | CONDITIONING | کاندیشنینگ مثبت اصلاح‌شده که احتمالاً حاوی نهانه‌های تصویر مرجع جاسازی‌شده، خروجی CLIP vision یا نهانه‌های ویدیوی حالت است. |
| `negative` | CONDITIONING | کاندیشنینگ منفی اصلاح‌شده که احتمالاً حاوی نهانه‌های تصویر مرجع جاسازی‌شده، خروجی CLIP vision یا نهانه‌های ویدیوی حالت است. |
| `latent` | LATENT | یک تنسور نهفته خالی با شکل `[batch_size, 16, ((length - 1) // 4) + 1, height // 8, width // 8]`. |
