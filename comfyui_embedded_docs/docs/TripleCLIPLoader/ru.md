Вот перевод документации на русский язык:

Узел TripleCLIPLoader загружает три различных текстовых энкодера одновременно и объединяет их в единую модель CLIP. Это полезно для продвинутых сценариев кодирования текста, где требуется несколько текстовых энкодеров, например, в рабочих процессах SD3, которые требуют совместной работы моделей clip-l, clip-g и t5.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `clip_name1` | STRING | Да | Доступно несколько вариантов | Первая модель текстового энкодера для загрузки из доступных текстовых энкодеров |
| `clip_name2` | STRING | Да | Доступно несколько вариантов | Вторая модель текстового энкодера для загрузки из доступных текстовых энкодеров |
| `clip_name3` | STRING | Да | Доступно несколько вариантов | Третья модель текстового энкодера для загрузки из доступных текстовых энкодеров |

**Примечание:** Все три параметра текстового энкодера должны быть выбраны из доступных моделей текстовых энкодеров в вашей системе. Узел загрузит все три модели и объединит их в единую модель CLIP для обработки.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `CLIP` | CLIP | Объединенная модель CLIP, содержащая все три загруженных текстовых энкодера |
