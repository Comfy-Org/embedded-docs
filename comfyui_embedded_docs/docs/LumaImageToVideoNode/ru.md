Вот перевод документации на русский язык:

Генерирует видео синхронно на основе текстового запроса и опциональных начального/конечного изображений. Этот узел использует API Luma для создания видео, позволяя вам определить содержимое видео через запрос и опционально указать первый и/или последний кадр для управления структурой видео.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `prompt` | STRING | Да | - | Запрос для генерации видео (по умолчанию: "") |
| `model` | COMBO | Да | Доступно несколько вариантов | Выбирает модель генерации видео из доступных моделей Luma |
| `resolution` | COMBO | Да | `"540p"`<br>`"720p"`<br>`"1080p"`<br>`"4k"` | Разрешение выходного видео (по умолчанию: "540p"). Этот параметр игнорируется при использовании модели `ray-1-6`. |
| `duration` | COMBO | Да | `"5s"`<br>`"9s"` | Длительность генерируемого видео. Этот параметр игнорируется при использовании модели `ray-1-6`. |
| `loop` | BOOLEAN | Да | - | Должно ли генерируемое видео зацикливаться (по умолчанию: False) |
| `seed` | INT | Да | от 0 до 18446744073709551615 | Сид для определения необходимости повторного запуска узла; фактические результаты недетерминированы независимо от сида. (по умолчанию: 0) |
| `first_image` | IMAGE | Нет | - | Первый кадр генерируемого видео. (опционально) |
| `last_image` | IMAGE | Нет | - | Последний кадр генерируемого видео. (опционально) |
| `luma_concepts` | CUSTOM | Нет | - | Опциональные концепции камеры для управления движением камеры через узел Luma Concepts. (опционально) |

**Примечание:** Необходимо указать хотя бы одно из изображений `first_image` или `last_image`. Узел вызовет исключение, если оба отсутствуют. Параметры `resolution` и `duration` игнорируются, когда для `model` установлено значение `ray-1-6`.

## Выходные данные

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `output` | VIDEO | Сгенерированный видеофайл |
