Генерирует изображения синхронно на основе запроса и разрешения. Этот узел подключается к API Recraft для создания изображений по текстовым описаниям с заданными размерами и необязательными параметрами стиля и управления.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `prompt` | STRING | Да | - | Запрос для генерации изображения. (по умолчанию: "") |
| `size` | COMBO | Да | "1024x1024"<br>"1152x896"<br>"896x1152"<br>"1216x832"<br>"832x1216"<br>"1344x768"<br>"768x1344"<br>"1536x640"<br>"640x1536" | Размер генерируемого изображения. (по умолчанию: "1024x1024") |
| `n` | INT | Да | 1-6 | Количество изображений для генерации. (по умолчанию: 1) |
| `seed` | INT | Да | 0-18446744073709551615 | Сид для определения необходимости повторного запуска узла; фактические результаты недетерминированы независимо от сида. (по умолчанию: 0) |
| `recraft_style` | RECRAFT_STYLE | Нет | Доступно несколько вариантов | Необязательный выбор стиля для генерации изображения. Если не указан, по умолчанию используется стиль "realistic_image". |
| `negative_prompt` | STRING | Нет | - | Необязательное текстовое описание нежелательных элементов на изображении. (по умолчанию: "") |
| `recraft_controls` | RECRAFT_CONTROLS | Нет | Доступно несколько вариантов | Необязательные дополнительные параметры управления генерацией через узел Recraft Controls. |

**Примечание:** Параметр `seed` управляет только моментом повторного запуска узла, но не делает генерацию изображения детерминированной. Фактические выходные изображения будут различаться даже при одинаковом значении сида.

**Примечание:** Длина параметра `prompt` должна составлять от 1 до 1000 символов.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `IMAGE` | IMAGE | Сгенерированное изображение(я) в виде пакетного тензора. При генерации нескольких изображений (n > 1) они объединяются по размерности пакета. |
