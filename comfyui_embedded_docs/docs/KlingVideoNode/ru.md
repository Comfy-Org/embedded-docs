Вот перевод документации на русский язык:

Этот узел генерирует видео с использованием модели Kling V3. Он поддерживает два основных режима: текст-в-видео, где видео создаётся из текстового описания, и изображение-в-видео, где анимируется существующее изображение. Также предлагаются расширенные функции, такие как создание мультисегментных видео с разными подсказками для каждой части (раскадровки) и опциональная генерация сопутствующего аудио.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `multi_shot` | COMBO | Да | `"disabled"`<br>`"1 storyboard"`<br>`"2 storyboards"`<br>`"3 storyboards"`<br>`"4 storyboards"`<br>`"5 storyboards"`<br>`"6 storyboards"` | Управляет созданием одного видео или серии сегментов с индивидуальными подсказками и длительностью. Если значение не "disabled", появляются дополнительные входы для подсказки и длительности каждой раскадровки. |
| `generate_audio` | BOOLEAN | Да | `True` / `False` | При включении узел генерирует аудио для видео. По умолчанию `True`. |
| `model` | COMBO | Да | `"kling-v3"` | Модель и связанные с ней настройки. Выбор этой опции открывает подпараметры `resolution` и `aspect_ratio`. |
| `model.resolution` | COMBO | Да | `"4k"`<br>`"1080p"`<br>`"720p"` | Разрешение генерируемого видео. Эта настройка доступна, когда `model` установлена на "kling-v3". |
| `model.aspect_ratio` | COMBO | Да | `"16:9"`<br>`"9:16"`<br>`"1:1"` | Соотношение сторон генерируемого видео. Эта настройка игнорируется, если для `start_frame` указано изображение (режим изображение-в-видео). Доступна, когда `model` установлена на "kling-v3". |
| `seed` | INT | Да | от 0 до 2147483647 | Значение зерна для генерации. Изменение этого значения приведёт к повторному запуску узла, но результаты не являются детерминированными. По умолчанию `0`. |
| `start_frame` | IMAGE | Нет | - | Необязательное начальное изображение. При подключении узел переключается из режима текст-в-видео в режим изображение-в-видео, анимируя предоставленное изображение. |

**Входные параметры для режима `multi_shot`:**

* Когда `multi_shot` установлен в **"disabled"**, появляются следующие входы:
  * `prompt` (STRING): Основное текстовое описание видео. Обязательно. Должно содержать от 1 до 2500 символов.
  * `negative_prompt` (STRING): Текст, описывающий то, что не должно появляться в видео. Необязательно.
  * `duration` (INT): Длительность видео в секундах. Должна быть от 3 до 15. По умолчанию `5`.
* Когда `multi_shot` установлен в один из вариантов раскадровки (например, `"3 storyboards"`), появляются входы для каждого сегмента раскадровки (например, `storyboard_1_prompt`, `storyboard_1_duration`). Каждая подсказка должна содержать от 1 до 512 символов. **Общая сумма длительностей всех раскадровок** должна быть от 3 до 15 секунд.

**Ограничения:**

* Узел работает в режиме **текст-в-видео**, когда `start_frame` не подключён. В этом режиме используется настройка `model.aspect_ratio`.
* Узел работает в режиме **изображение-в-видео**, когда `start_frame` подключён. Настройка `model.aspect_ratio` игнорируется. Входное изображение должно быть не менее 300x300 пикселей и иметь соотношение сторон от 1:2.5 до 2.5:1.
* В режиме раскадровки (`multi_shot` не "disabled") основные входы `prompt` и `negative_prompt` скрыты и не используются.

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `video` | VIDEO | Сгенерированный видеофайл. |
