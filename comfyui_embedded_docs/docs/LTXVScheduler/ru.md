Узел LTXVScheduler генерирует значения сигм для пользовательских процессов сэмплирования. Он вычисляет параметры расписания шума на основе количества токенов во входном латентном представлении и применяет сигмоидное преобразование для создания расписания сэмплирования. Узел может опционально растягивать результирующие сигмы для соответствия заданному конечному значению.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `шаги` | INT | Да | 1-10000 | Количество шагов сэмплирования (по умолчанию: 20) |
| `максимальное_смещение` | FLOAT | Да | 0.0-100.0 | Максимальное значение сдвига для расчета сигм (по умолчанию: 2.05) |
| `базовое_смещение` | FLOAT | Да | 0.0-100.0 | Базовое значение сдвига для расчета сигм (по умолчанию: 0.95) |
| `растяжение` | BOOLEAN | Да | True/False | Растянуть сигмы в диапазон [terminal, 1] (по умолчанию: True) |
| `терминал` | FLOAT | Да | 0.0-0.99 | Конечное значение сигм после растяжения (по умолчанию: 0.1) |
| `скрытый` | LATENT | Нет | - | Опциональное латентное представление, используемое для расчета количества токенов для корректировки сигм |

**Примечание:** Параметр `latent` является опциональным. Если он не указан, узел использует значение количества токенов по умолчанию — 4096.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `sigmas` | SIGMAS | Сгенерированные значения сигм для процесса сэмплирования |
