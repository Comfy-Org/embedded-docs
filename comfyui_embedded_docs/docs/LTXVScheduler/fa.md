گره LTXVScheduler مقادیر سیگما را برای فرآیندهای نمونه‌گیری سفارشی تولید می‌کند. این گره پارامترهای زمان‌بندی نویز را بر اساس تعداد توکن‌های موجود در ورودی نهفته محاسبه کرده و با اعمال تبدیل سیگموئید، زمان‌بندی نمونه‌گیری را ایجاد می‌کند. این گره می‌تواند به‌صورت اختیاری سیگماهای حاصل را برای مطابقت با یک مقدار پایانی مشخص، کشش دهد.

## ورودی‌ها

| پارامتر | نوع داده | اجباری | محدوده | توضیحات |
|---------|----------|--------|--------|---------|
| `steps` | INT | بله | 1-10000 | تعداد مراحل نمونه‌گیری (پیش‌فرض: 20) |
| `max_shift` | FLOAT | بله | 0.0-100.0 | حداکثر مقدار شیفت برای محاسبه سیگما (پیش‌فرض: 2.05) |
| `base_shift` | FLOAT | بله | 0.0-100.0 | مقدار شیفت پایه برای محاسبه سیگما (پیش‌فرض: 0.95) |
| `stretch` | BOOLEAN | بله | True/False | سیگماها را در بازه [terminal, 1] کشش می‌دهد (پیش‌فرض: True) |
| `terminal` | FLOAT | بله | 0.0-0.99 | مقدار پایانی سیگماها پس از کشش (پیش‌فرض: 0.1) |
| `latent` | LATENT | خیر | - | ورودی نهفته اختیاری برای محاسبه تعداد توکن جهت تنظیم سیگما |

**نکته:** پارامتر `latent` اختیاری است. در صورت عدم ارائه، گره از تعداد توکن پیش‌فرض 4096 برای محاسبات استفاده می‌کند.
