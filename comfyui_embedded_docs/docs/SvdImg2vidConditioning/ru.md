Этот узел предназначен для генерации данных кондиционирования для задач создания видео, специально адаптированных для использования с моделями SVD_img2vid. Он принимает различные входные данные, включая начальные изображения, параметры видео и модель VAE, для создания данных кондиционирования, которые можно использовать для управления генерацией видеокадров.

## Входные данные

| Параметр             | Comfy dtype        | Описание |
|----------------------|--------------------|----------|
| `clip_vision`         | `CLIP_VISION`      | Представляет модель CLIP vision, используемую для кодирования визуальных признаков из начального изображения, играя ключевую роль в понимании содержания и контекста изображения для генерации видео. |
| `init_image`          | `IMAGE`            | Начальное изображение, на основе которого будет сгенерировано видео, служащее отправной точкой для процесса генерации видео. |
| `vae`                 | `VAE`              | Вариационный автоэнкодер (VAE), используемый для кодирования начального изображения в скрытое пространство, что способствует генерации связных и непрерывных видеокадров. |
| `width`               | `INT`              | Желаемая ширина генерируемых видеокадров, позволяющая настроить разрешение видео. |
| `height`              | `INT`              | Желаемая высота видеокадров, обеспечивающая контроль над соотношением сторон и разрешением видео. |
| `video_frames`        | `INT`              | Указывает количество кадров, которые будут сгенерированы для видео, определяя его продолжительность. |
| `motion_bucket_id`    | `INT`              | Идентификатор для категоризации типа движения, применяемого при генерации видео, помогающий в создании динамичных и увлекательных видео. |
| `fps`                 | `INT`              | Частота кадров в секунду (fps) для видео, влияющая на плавность и реалистичность сгенерированного видео. |
| `augmentation_level`  | `FLOAT`            | Параметр, управляющий уровнем аугментации, применяемой к начальному изображению, влияющий на разнообразие и вариативность генерируемых видеокадров. |

## Выходные данные

| Параметр     | Comfy dtype        | Описание |
|--------------|--------------------|----------|
| `positive`   | `CONDITIONING`     | Положительные данные кондиционирования, состоящие из закодированных признаков и параметров для направления процесса генерации видео в желаемом направлении. |
| `negative`   | `CONDITIONING`     | Отрицательные данные кондиционирования, обеспечивающие контраст с положительным кондиционированием, которые можно использовать для избежания определенных паттернов или признаков в сгенерированном видео. |
| `latent`     | `LATENT`           | Скрытые представления, сгенерированные для каждого кадра видео, служащие основополагающим компонентом для процесса генерации видео. |
