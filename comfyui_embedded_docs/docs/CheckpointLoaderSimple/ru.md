## Обзор

Загружает файл контрольной точки (checkpoint) диффузионной модели и разбивает его на три основных компонента: основную модель для шумоподавления латентного представления, текстовый энкодер CLIP и энкодер/декодер изображений VAE. Этот узел автоматически обнаруживает все файлы моделей в папке `ComfyUI/models/checkpoints` и любых дополнительных путях, настроенных в вашем файле `extra_model_paths.yaml`.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `название_точки_проверки` | STRING | Да | Все файлы моделей в папке checkpoints | Имя загружаемой контрольной точки (модели). Выберите имя файла контрольной точки, которое определяет модель ИИ, используемую для последующей генерации изображений. |

**Примечание:** Если во время работы ComfyUI были добавлены новые файлы моделей, необходимо обновить браузер (Ctrl+R), чтобы увидеть новые файлы в выпадающем списке.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `MODEL` | MODEL | Модель, используемая для шумоподавления латентного представления. Это основная диффузионная модель, используемая для генерации изображений. |
| `CLIP` | CLIP | Модель CLIP, используемая для кодирования текстовых подсказок, преобразующая текстовые описания в информацию, понятную ИИ. |
| `VAE` | VAE | Модель VAE, используемая для кодирования и декодирования изображений в латентное пространство и обратно. |
