Вот перевод документации на русский язык:

Генерирует переходное видео между двумя входными изображениями с помощью API PixVerse. Вы предоставляете начальное и конечное изображения, а узел создает плавное видео, которое переходит от одного к другому, руководствуясь вашим текстовым запросом и выбранными настройками.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `first_frame` | IMAGE | Да | - | Начальное изображение для перехода в видео |
| `last_frame` | IMAGE | Да | - | Конечное изображение для перехода в видео |
| `prompt` | STRING | Да | - | Запрос для генерации видео (по умолчанию: пустая строка) |
| `quality` | COMBO | Да | `"360p"`<br>`"540p"`<br>`"720p"`<br>`"1080p"` | Настройка качества видео (по умолчанию: `"540p"`) |
| `duration_seconds` | COMBO | Да | `5`<br>`8` | Продолжительность видео в секундах |
| `motion_mode` | COMBO | Да | `"normal"`<br>`"fast"` | Стиль движения для перехода (по умолчанию: `"normal"`) |
| `seed` | INT | Да | от 0 до 2147483647 | Сид для генерации видео (по умолчанию: 0) |
| `negative_prompt` | STRING | Нет | - | Необязательное текстовое описание нежелательных элементов на изображении (по умолчанию: пустая строка) |

**Примечание об ограничениях параметров:** При использовании качества 1080p режим движения автоматически устанавливается в `"normal"`, а длительность ограничивается 5 секундами. Для любой длительности, отличной от 5 секунд, режим движения также автоматически устанавливается в `"normal"`.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `output` | VIDEO | Сгенерированное переходное видео |
