Этот узел применяет заданную анимацию к 3D-модели персонажа, которая уже была оснащена скелетом с помощью сервиса Meshy. Он принимает идентификатор задачи из предыдущей операции риггинга и идентификатор действия для выбора нужной анимации из библиотеки. Затем узел обрабатывает запрос и возвращает анимированную модель в форматах GLB и FBX.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `rig_task_id` | STRING | Да | Н/Д | Уникальный идентификатор задачи из ранее завершенной операции риггинга персонажа в Meshy. |
| `action_id` | INT | Да | от 0 до 696 | Идентификационный номер анимационного действия для применения. Посетите <https://docs.meshy.ai/en/api/animation-library> для получения списка доступных значений. (по умолчанию: 0) |

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `model_file` | STRING | Строковый идентификатор анимированной модели. Этот выход предоставляется только для обратной совместимости. |
| `GLB` | FILE3DGLB | Файл анимированной 3D-модели в формате GLB. |
| `FBX` | FILE3DFBX | Файл анимированной 3D-модели в формате FBX. |
