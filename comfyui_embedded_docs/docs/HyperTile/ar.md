هذه ترجمة الوثيقة التقنية لعقدة HyperTile في ComfyUI إلى العربية، مع الالتزام بقواعد الترجمة المحددة:

عقدة HyperTile تطبق تقنية التقسيم المربعي على آلية الانتباه في نماذج الانتشار لتحسين استخدام الذاكرة أثناء توليد الصور. تقوم بتقسيم الفضاء الكامن إلى مربعات أصغر ومعالجتها بشكل منفصل، ثم إعادة تجميع النتائج. يتيح ذلك العمل بأحجام صور أكبر دون نفاد الذاكرة.

## المدخلات

| المعامل | نوع البيانات | إجباري | النطاق | الوصف |
|---------|---------------|--------|---------|-------------|
| `model` | MODEL | نعم | - | نموذج الانتشار الذي سيتم تطبيق تحسين HyperTile عليه |
| `tile_size` | INT | لا | 1 - 2048 | الحجم المستهدف للمربع للمعالجة (الافتراضي: 256). يتم تقريب الحجم الفعلي للمربع إلى أسفل ليكون من مضاعفات 8، بحد أدنى 32. |
| `swap_size` | INT | لا | 1 - 128 | يتحكم في كيفية إعادة ترتيب المربعات أثناء المعالجة لتحسين الكفاءة (الافتراضي: 2) |
| `max_depth` | INT | لا | 0 - 10 | أقصى مستوى عمق (مقياس الدقة) لتطبيق التقسيم المربعي. القيمة 0 تطبق التقسيم فقط عند أعلى دقة (الافتراضي: 0) |
| `scale_depth` | BOOLEAN | لا | True / False | عند التفعيل، يتم قياس حجم المربع بشكل متناسب في مستويات العمق الأعمق. يمكن أن يساعد ذلك في الحفاظ على الجودة عند الدقة المنخفضة (الافتراضي: False) |
