Узел Switch выбирает между двумя возможными входными данными на основе булева условия. Он передаёт на выход вход `on_true`, когда переключатель `switch` включён, и вход `on_false`, когда переключатель `switch` выключен. Это позволяет создавать условную логику и выбирать разные пути данных в вашем рабочем процессе.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `switch` | BOOLEAN | Да | | Булево условие, определяющее, какой входной параметр будет передан. Когда включено (true), выбирается вход `on_true`. Когда выключено (false), выбирается вход `on_false`. |
| `on_false` | MATCH_TYPE | Нет | | Данные, передаваемые на выход, когда переключатель `switch` выключен (false). Этот вход требуется только при значении `switch` false. |
| `on_true` | MATCH_TYPE | Нет | | Данные, передаваемые на выход, когда переключатель `switch` включён (true). Этот вход требуется только при значении `switch` true. |

**Примечание о требованиях к входам:** Входы `on_false` и `on_true` являются условно обязательными. Узел будет запрашивать вход `on_true` только когда `switch` равен true, и вход `on_false` только когда `switch` равен false. Оба входа должны быть одного типа данных.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `output` | MATCH_TYPE | Выбранные данные. Это будет значение из входа `on_true`, если `switch` равен true, или значение из входа `on_false`, если `switch` равен false. |
