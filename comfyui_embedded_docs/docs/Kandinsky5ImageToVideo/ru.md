Узел Kandinsky5ImageToVideo подготавливает данные conditioning и латентного пространства для генерации видео с использованием модели Kandinsky. Он создает пустой тензор латентного представления видео и, при необходимости, может кодировать начальное изображение для управления начальными кадрами генерируемого видео, соответствующим образом изменяя положительное и отрицательное conditioning.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `позитивный` | CONDITIONING | Да | Н/Д | Положительные промпты conditioning для управления генерацией видео. |
| `негативный` | CONDITIONING | Да | Н/Д | Отрицательные промпты conditioning для исключения определенных концепций из генерации видео. |
| `vae` | VAE | Да | Н/Д | Модель VAE, используемая для кодирования опционального начального изображения в латентное пространство. |
| `ширина` | INT | Нет | от 16 до 8192 (шаг 16) | Ширина выходного видео в пикселях (по умолчанию: 768). |
| `высота` | INT | Нет | от 16 до 8192 (шаг 16) | Высота выходного видео в пикселях (по умолчанию: 512). |
| `длина` | INT | Нет | от 1 до 8192 (шаг 4) | Количество кадров в видео (по умолчанию: 121). |
| `размер_пакета` | INT | Нет | от 1 до 4096 | Количество видеопоследовательностей, генерируемых одновременно (по умолчанию: 1). |
| `стартовое_изображение` | IMAGE | Нет | Н/Д | Опциональное начальное изображение. Если указано, оно кодируется и используется для замены зашумленного начала латентного представления выходных данных модели. |

**Примечание:** При указании `start_image` оно автоматически масштабируется до заданных значений `width` и `height` с помощью билинейной интерполяции. Для кодирования используются первые `length` кадров пакета изображений. Закодированное латентное представление затем внедряется как в положительное, так и в отрицательное conditioning для управления начальным видом видео.

## Выходные параметры

| Имя выходного параметра | Тип данных | Описание |
|-------------|-----------|-------------|
| `позитивный` | CONDITIONING | Измененное положительное conditioning, потенциально обновленное данными закодированного начального изображения. |
| `негативный` | CONDITIONING | Измененное отрицательное conditioning, потенциально обновленное данными закодированного начального изображения. |
| `latent` | LATENT | Пустой тензор латентного представления видео, заполненный нулями и имеющий размерность, соответствующую заданным параметрам. |
| `cond_latent` | LATENT | Чистое закодированное латентное представление предоставленных начальных изображений. Используется внутренне для замены зашумленного начала латентного представления генерируемого видео. |
