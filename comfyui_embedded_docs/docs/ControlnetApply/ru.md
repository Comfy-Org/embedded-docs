Использование controlNet требует предварительной обработки входных изображений. Поскольку стандартные узлы ComfyUI не содержат препроцессоров и моделей controlNet, сначала установите препроцессоры ControlNet [скачать препроцессоры здесь](https://github.com/Fannovel16/comfy_controlnet_preprocessors) и соответствующие модели controlNet.

## Входные параметры

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `positive` | `CONDITIONING` | Положительные данные conditioning, поступают из CLIP Text Encoder или других источников conditioning |
| `negative` | `CONDITIONING` | Отрицательные данные conditioning, поступают из CLIP Text Encoder или других источников conditioning |
| `control_net` | `CONTROL_NET` | Применяемая модель controlNet, обычно поступает из узла ControlNet Loader |
| `изображение` | `IMAGE` | Изображение для применения controlNet, должно быть обработано препроцессором |
| `vae` | `VAE` | Входные данные модели VAE |
| `сила` | `FLOAT` | Управляет силой корректировок сети, диапазон значений 0~10. Рекомендуемые значения находятся в пределах 0.5~1.5. Меньшие значения дают модели больше свободы, большие значения накладывают более строгие ограничения. Слишком высокие значения могут привести к появлению странных изображений. Вы можете тестировать и корректировать это значение для точной настройки влияния управляющей сети. |
| `start_percent` | `FLOAT` | Значение 0.000~1.000, определяет момент начала применения controlNet в процентах. Например, 0.2 означает, что управление ControlNet начнет влиять на генерацию изображения на 20% процесса диффузии |
| `end_percent` | `FLOAT` | Значение 0.000~1.000, определяет момент прекращения применения controlNet в процентах. Например, 0.8 означает, что управление ControlNet перестанет влиять на генерацию изображения на 80% процесса диффузии |

### Выходные параметры

| Параметр | Тип данных | Описание |
| --- | --- | --- |
| `positive` | `CONDITIONING` | Положительные данные conditioning, обработанные ControlNet, могут быть переданы на следующий узел ControlNet или K Sampler |
| `negative` | `CONDITIONING` | Отрицательные данные conditioning, обработанные ControlNet, могут быть переданы на следующий узел ControlNet или K Sampler |
