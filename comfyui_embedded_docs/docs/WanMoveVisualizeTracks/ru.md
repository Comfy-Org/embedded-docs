Вот перевод документации для узла WanMoveVisualizeTracks:

Узел WanMoveVisualizeTracks накладывает данные отслеживания движения на последовательность изображений или видеокадров. Он создает визуальные представления отслеживаемых точек, включая их траектории движения и текущие положения, делая данные о движении видимыми и более удобными для анализа.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `images` | IMAGE | Да | - | Последовательность входных изображений или видеокадров, на которых необходимо визуализировать треки. |
| `tracks` | TRACKS | Нет | - | Данные отслеживания движения, содержащие траектории точек и информацию о видимости. Если не указаны, входные изображения передаются без изменений. |
| `line_resolution` | INT | Да | 1 - 1024 | Количество предыдущих кадров, используемых для отрисовки линии следа для каждого трека (по умолчанию: 24). |
| `circle_size` | INT | Да | 1 - 128 | Размер круга, отображаемого в текущем положении каждого трека (по умолчанию: 12). |
| `opacity` | FLOAT | Да | 0.0 - 1.0 | Непрозрачность наложенных треков (по умолчанию: 0.75). |
| `line_width` | INT | Да | 1 - 128 | Ширина линий, используемых для отрисовки траекторий треков (по умолчанию: 16). |

**Примечание:** Если количество входных изображений не совпадает с количеством кадров в предоставленных данных `tracks`, последовательность изображений будет повторена для соответствия длине треков.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `IMAGE` | IMAGE | Последовательность изображений с визуализированными данными отслеживания движения в виде наложений. Если `tracks` не были предоставлены, возвращаются исходные входные изображения. |
