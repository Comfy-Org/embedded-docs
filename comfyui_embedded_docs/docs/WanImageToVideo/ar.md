# ترجمة وثيقة عقدة WanImageToVideo

تقوم عقدة WanImageToVideo بإعداد تمثيلات التكييف (conditioning) والكامن (latent) لمهام توليد الفيديو. تنشئ هذه العقدة مساحة كامنة فارغة لتوليد الفيديو، ويمكنها دمج الصور الأولية ومخرجات رؤية CLIP بشكل اختياري لتوجيه عملية توليد الفيديو. تقوم العقدة بتعديل مدخلات التكييف الإيجابية والسالبة بناءً على الصورة وبيانات الرؤية المقدمة.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|---------|---------------|--------|---------|-------|
| `إيجابي` | CONDITIONING | نعم | - | مدخل التكييف الإيجابي لتوجيه عملية التوليد |
| `سلبي` | CONDITIONING | نعم | - | مدخل التكييف السلبي لتوجيه عملية التوليد |
| `vae` | VAE | نعم | - | نموذج VAE لتشفير الصور إلى الفضاء الكامن |
| `العرض` | INT | نعم | 16 إلى MAX_RESOLUTION | عرض الفيديو الناتج (الافتراضي: 832، الخطوة: 16) |
| `الارتفاع` | INT | نعم | 16 إلى MAX_RESOLUTION | ارتفاع الفيديو الناتج (الافتراضي: 480، الخطوة: 16) |
| `الطول` | INT | نعم | 1 إلى MAX_RESOLUTION | عدد الإطارات في الفيديو (الافتراضي: 81، الخطوة: 4) |
| `حجم الدفعة` | INT | نعم | 1 إلى 4096 | عدد مقاطع الفيديو المراد توليدها في الدفعة الواحدة (الافتراضي: 1) |
| `ناتج رؤية الكليب` | CLIP_VISION_OUTPUT | لا | - | مخرجات رؤية CLIP اختيارية لتكييف إضافي |
| `صورة البداية` | IMAGE | لا | - | صورة أولية اختيارية لتهيئة توليد الفيديو |

**ملاحظة:** عند توفير `start_image`، تقوم العقدة بتشفير تسلسل الصور وتطبيق الإخفاء (masking) على مدخلات التكييف. معامل `clip_vision_output`، عند توفيره، يضيف تكييفًا قائمًا على الرؤية لكل من المدخلات الإيجابية والسالبة.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|---------------|---------|
| `سلبي` | CONDITIONING | تكييف إيجابي معدّل مع دمج بيانات الصورة والرؤية |
| `مضمر` | CONDITIONING | تكييف سلبي معدّل مع دمج بيانات الصورة والرؤية |
| `latent` | LATENT | موتر الفضاء الكامن الفارغ الجاهز لتوليد الفيديو |
