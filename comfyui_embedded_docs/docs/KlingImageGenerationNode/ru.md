Вот перевод документации на русский язык:

Узел генерации изображений Kling создаёт изображения на основе текстовых подсказок с возможностью использования референсного изображения для направления. Он создаёт одно или несколько изображений на основе вашего текстового описания и настроек референса, затем возвращает сгенерированные изображения в качестве выходных данных.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `prompt` | STRING | Да | - | Положительный текстовый промпт |
| `negative_prompt` | STRING | Да | - | Отрицательный текстовый промпт |
| `image_type` | COMBO | Да | `"subject_reference"`<br>`"style_reference"` | Выбор типа референсного изображения (расширенный). Обязателен при указании референсного изображения. |
| `image_fidelity` | FLOAT | Да | 0.0 - 1.0 | Интенсивность референса для загруженных пользователем изображений (по умолчанию: 0.5, расширенный) |
| `human_fidelity` | FLOAT | Да | 0.0 - 1.0 | Степень сходства с референсным объектом (по умолчанию: 0.45, расширенный) |
| `model_name` | COMBO | Да | `"kling-v3"`<br>`"kling-v2"`<br>`"kling-v1-5"` | Выбор модели для генерации изображений (по умолчанию: "kling-v3") |
| `aspect_ratio` | COMBO | Да | `"16:9"`<br>`"9:16"`<br>`"1:1"`<br>`"4:3"`<br>`"3:4"`<br>`"21:9"`<br>`"9:21"` | Соотношение сторон для генерируемых изображений (по умолчанию: "16:9") |
| `n` | INT | Да | 1 - 9 | Количество генерируемых изображений (по умолчанию: 1) |
| `image` | IMAGE | Нет | - | Необязательное референсное изображение |
| `seed` | INT | Нет | 0 - 2147483647 | Сид управляет тем, должен ли узел перезапускаться; результаты недетерминированы независимо от сида (по умолчанию: 0) |

**Ограничения параметров:**

- Параметр `image` является необязательным. При указании референсного изображения параметр `image_type` должен быть установлен в значение `"subject_reference"` или `"style_reference"`.
- Если референсное изображение не указано, параметры `image_type`, `image_fidelity` и `human_fidelity` не используются.
- Промпт и отрицательный промпт имеют максимальную длину `MAX_PROMPT_LENGTH_IMAGE_GEN` символов.
- Параметр `seed` является необязательным и не гарантирует детерминированных результатов.

## Выходные данные

| Имя выходного параметра | Тип данных | Описание |
|-------------|-----------|-------------|
| `output` | IMAGE | Сгенерированное изображение(я) на основе входных параметров |
