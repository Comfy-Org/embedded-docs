Вот перевод документации на русский язык:

Узел Meshy: Image to Model использует API Meshy для генерации 3D-модели из одного входного изображения. Он загружает ваше изображение, отправляет задачу на обработку и возвращает сгенерированные файлы 3D-модели (GLB и FBX), а также идентификатор задачи для справки.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `model` | COMBO | Да | `"latest"` | Указывает версию модели ИИ, используемую для генерации. |
| `image` | IMAGE | Да | - | Входное изображение для преобразования в 3D-модель. |
| `should_remesh` | DYNAMIC COMBO | Да | `"true"`<br>`"false"` | Определяет, следует ли обрабатывать сгенерированную сетку. При значении `"false"` узел возвращает необработанную треугольную сетку. |
| `topology` | COMBO | Нет* | `"triangle"`<br>`"quad"` | Целевая топология полигонов для ремешинговой модели. Этот параметр доступен только когда `should_remesh` установлен в `"true"`. |
| `target_polycount` | INT | Нет* | 100 - 300000 | Целевое количество полигонов для ремешинговой модели. Этот параметр доступен только когда `should_remesh` установлен в `"true"`. Значение по умолчанию: 300000. |
| `symmetry_mode` | COMBO | Да | `"auto"`<br>`"on"`<br>`"off"` | Управляет симметрией, применяемой к сгенерированной 3D-модели. |
| `should_texture` | DYNAMIC COMBO | Да | `"true"`<br>`"false"` | Определяет, создаются ли текстуры для модели. Установка значения `"false"` пропускает этап текстурирования и возвращает сетку без текстур. |
| `enable_pbr` | BOOLEAN | Нет* | - | Когда `should_texture` имеет значение `"true"`, эта опция генерирует PBR-карты (металличность, шероховатость, нормали) в дополнение к базовому цвету. Значение по умолчанию: `False`. |
| `texture_prompt` | STRING | Нет* | - | Текстовый запрос для управления процессом текстурирования (максимум 600 символов). Этот параметр доступен только когда `should_texture` установлен в `"true"`. Не может использоваться одновременно с `texture_image`. |
| `texture_image` | IMAGE | Нет* | - | Изображение для управления процессом текстурирования. Этот параметр доступен только когда `should_texture` установлен в `"true"`. Не может использоваться одновременно с `texture_prompt`. |
| `pose_mode` | COMBO | Да | `""` (пусто)<br>`"A-pose"`<br>`"T-pose"` | Указывает режим позы для сгенерированной модели. Это расширенный параметр. |
| `seed` | INT | Да | 0 - 2147483647 | Значение зерна для процесса генерации. Результаты не являются детерминированными независимо от значения зерна. Значение по умолчанию: 0. |

**Примечание об ограничениях параметров:**

* Параметры `topology` и `target_polycount` доступны только когда `should_remesh` установлен в `"true"`.
* Параметры `enable_pbr`, `texture_prompt` и `texture_image` доступны только когда `should_texture` установлен в `"true"`.
* Нельзя использовать `texture_prompt` и `texture_image` одновременно. Если оба параметра предоставлены при `should_texture` равном `"true"`, узел выдаст ошибку.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `model_file` | STRING | Имя файла сгенерированной GLB-модели. (Сохраняется для обратной совместимости). |
| `meshy_task_id` | MESHY_TASK_ID | Уникальный идентификатор задачи API Meshy, который можно использовать для справки или устранения неполадок. |
| `GLB` | FILE3DGLB | Сгенерированная 3D-модель в формате файла GLB. |
| `FBX` | FILE3DFBX | Сгенерированная 3D-модель в формате файла FBX. |
