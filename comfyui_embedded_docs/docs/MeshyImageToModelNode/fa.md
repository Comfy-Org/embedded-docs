### گره Meshy: تصویر به مدل

گره Meshy: Image to Model از API Meshy برای تولید یک مدل سه‌بعدی از یک تصویر ورودی استفاده می‌کند. این گره تصویر شما را آپلود کرده، یک وظیفه پردازش ارسال می‌کند و فایل‌های مدل سه‌بعدی تولید شده (GLB و FBX) را به همراه شناسه وظیفه برای مرجع بازمی‌گرداند.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|-----------|-----------|----------|-------|-------------|
| `model` | COMBO | بله | `"latest"` | نسخه مدل هوش مصنوعی مورد استفاده برای تولید را مشخص می‌کند. |
| `image` | IMAGE | بله | - | تصویر ورودی برای تبدیل به مدل سه‌بعدی. |
| `should_remesh` | DYNAMIC COMBO | بله | `"true"`<br>`"false"` | تعیین می‌کند که آیا مش تولید شده باید پردازش شود. وقتی روی `"false"` تنظیم شود، گره یک مش مثلثی پردازش‌نشده بازمی‌گرداند. |
| `topology` | COMBO | خیر* | `"triangle"`<br>`"quad"` | توپولوژی چندضلعی هدف برای مدل بازمش‌بندی‌شده. این ورودی فقط زمانی در دسترس است که `should_remesh` روی `"true"` تنظیم شده باشد. |
| `target_polycount` | INT | خیر* | 100 - 300000 | تعداد چندضلعی‌های هدف برای مدل بازمش‌بندی‌شده. این ورودی فقط زمانی در دسترس است که `should_remesh` روی `"true"` تنظیم شده باشد. مقدار پیش‌فرض 300000 است. |
| `symmetry_mode` | COMBO | بله | `"auto"`<br>`"on"`<br>`"off"` | تقارن اعمال‌شده به مدل سه‌بعدی تولید شده را کنترل می‌کند. |
| `should_texture` | DYNAMIC COMBO | بله | `"true"`<br>`"false"` | تعیین می‌کند که آیا بافت برای مدل تولید شود یا خیر. تنظیم آن روی `"false"` مرحله بافت‌دهی را نادیده گرفته و یک مش بدون بافت بازمی‌گرداند. |
| `enable_pbr` | BOOLEAN | خیر* | - | وقتی `should_texture` روی `"true"` است، این گزینه نقشه‌های PBR (فلزی، زبری، نرمال) را علاوه بر رنگ پایه تولید می‌کند. مقدار پیش‌فرض `False` است. |
| `texture_prompt` | STRING | خیر* | - | یک پرامپت متنی برای هدایت فرآیند بافت‌دهی (حداکثر 600 کاراکتر). این ورودی فقط زمانی در دسترس است که `should_texture` روی `"true"` تنظیم شده باشد. نمی‌توان همزمان با `texture_image` استفاده کرد. |
| `texture_image` | IMAGE | خیر* | - | یک تصویر برای هدایت فرآیند بافت‌دهی. این ورودی فقط زمانی در دسترس است که `should_texture` روی `"true"` تنظیم شده باشد. نمی‌توان همزمان با `texture_prompt` استفاده کرد. |
| `pose_mode` | COMBO | بله | `""` (خالی)<br>`"A-pose"`<br>`"T-pose"` | حالت ژست برای مدل تولید شده را مشخص می‌کند. این یک پارامتر پیشرفته است. |
| `seed` | INT | بله | 0 - 2147483647 | یک مقدار دانه (seed) برای فرآیند تولید. نتایج صرف‌نظر از مقدار دانه، غیرقطعی هستند. مقدار پیش‌فرض 0 است. |

**توجه در مورد محدودیت‌های پارامتر:**

* ورودی‌های `topology` و `target_polycount` فقط زمانی در دسترس هستند که `should_remesh` روی `"true"` تنظیم شده باشد.
* ورودی‌های `enable_pbr`، `texture_prompt` و `texture_image` فقط زمانی در دسترس هستند که `should_texture` روی `"true"` تنظیم شده باشد.
* نمی‌توانید همزمان از `texture_prompt` و `texture_image` استفاده کنید. اگر هر دو هنگام `"true"` بودن `should_texture` ارائه شوند، گره یک خطا ایجاد می‌کند.

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
|-------------|-----------|-------------|
| `model_file` | STRING | نام فایل مدل GLB تولید شده. (برای سازگاری با نسخه‌های قبلی حفظ شده است). |
| `meshy_task_id` | MESHY_TASK_ID | شناسه یکتای وظیفه API Meshy که می‌تواند برای مرجع یا عیب‌یابی استفاده شود. |
| `GLB` | FILE3DGLB | مدل سه‌بعدی تولید شده در قالب فایل GLB. |
| `FBX` | FILE3DFBX | مدل سه‌بعدی تولید شده در قالب فایل FBX. |
