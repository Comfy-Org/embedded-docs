هذا هو النص المترجم إلى العربية وفقًا للقواعد المحددة:

عقدة "Meshy: Image to Model" تستخدم واجهة برمجة تطبيقات Meshy لإنشاء نموذج ثلاثي الأبعاد من صورة إدخال واحدة. تقوم بتحميل صورتك، وتقديم مهمة معالجة، ثم تُرجع ملفات النموذج ثلاثي الأبعاد المُنشأة (GLB و FBX) بالإضافة إلى معرف المهمة للرجوع إليه.

## المدخلات

| المعامل | نوع البيانات | إجباري | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `model` | COMBO | نعم | `"latest"` | يُحدد إصدار نموذج الذكاء الاصطناعي المستخدم في الإنشاء. |
| `image` | IMAGE | نعم | - | الصورة المدخلة المراد تحويلها إلى نموذج ثلاثي الأبعاد. |
| `should_remesh` | DYNAMIC COMBO | نعم | `"true"`<br>`"false"` | يُحدد ما إذا كان يجب معالجة الشبكة المُنشأة. عند تعيينه إلى `"false"`، تُعيد العقدة شبكة مثلثية غير معالجة. |
| `topology` | COMBO | لا* | `"triangle"`<br>`"quad"` | طوبولوجيا المضلعات المستهدفة للنموذج المُعاد تشكيله. هذا المُدخل متاح فقط عندما تكون قيمة `should_remesh` مساوية لـ `"true"`. |
| `target_polycount` | INT | لا* | 100 - 300000 | العدد المستهدف للمضلعات للنموذج المُعاد تشكيله. هذا المُدخل متاح فقط عندما تكون قيمة `should_remesh` مساوية لـ `"true"`. القيمة الافتراضية هي 300000. |
| `symmetry_mode` | COMBO | نعم | `"auto"`<br>`"on"`<br>`"off"` | يتحكم في التماثل المُطبق على النموذج ثلاثي الأبعاد المُنشأ. |
| `should_texture` | DYNAMIC COMBO | نعم | `"true"`<br>`"false"` | يُحدد ما إذا كان سيتم إنشاء القوام للنموذج. تعيينه إلى `"false"` يتخطى مرحلة القوام ويُعيد شبكة بدون قوام. |
| `enable_pbr` | BOOLEAN | لا* | - | عندما تكون قيمة `should_texture` مساوية لـ `"true"`، يُنشئ هذا الخيار خرائط PBR (معدني، خشونة، طبيعي) بالإضافة إلى اللون الأساسي. القيمة الافتراضية هي `False`. |
| `texture_prompt` | STRING | لا* | - | موجه نصي لتوجيه عملية وضع القوام (بحد أقصى 600 حرف). هذا المُدخل متاح فقط عندما تكون قيمة `should_texture` مساوية لـ `"true"`. لا يمكن استخدامه في نفس وقت استخدام `texture_image`. |
| `texture_image` | IMAGE | لا* | - | صورة لتوجيه عملية وضع القوام. هذا المُدخل متاح فقط عندما تكون قيمة `should_texture` مساوية لـ `"true"`. لا يمكن استخدامه في نفس وقت استخدام `texture_prompt`. |
| `pose_mode` | COMBO | نعم | `""` (فارغ)<br>`"A-pose"`<br>`"T-pose"` | يُحدد وضعية النموذج المُنشأ. هذه معلمة متقدمة. |
| `seed` | INT | نعم | 0 - 2147483647 | قيمة بذرة لعملية الإنشاء. النتائج غير حتمية بغض النظر عن قيمة البذرة. القيمة الافتراضية هي 0. |

**ملاحظة حول قيود المعاملات:**

* مُدخلات `topology` و `target_polycount` متاحة فقط عندما تكون قيمة `should_remesh` مساوية لـ `"true"`.
* مُدخلات `enable_pbr` و `texture_prompt` و `texture_image` متاحة فقط عندما تكون قيمة `should_texture` مساوية لـ `"true"`.
* لا يمكنك استخدام `texture_prompt` و `texture_image` في نفس الوقت. إذا تم توفير كليهما عندما تكون قيمة `should_texture` مساوية لـ `"true"`، فستُصدر العقدة خطأً.

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|-----------|-------------|
| `meshy_task_id` | STRING | اسم ملف نموذج GLB المُنشأ. (مُحتفظ به للتوافق مع الإصدارات السابقة). |
| `GLB` | MESHY_TASK_ID | المُعرف الفريد لمهمة واجهة برمجة تطبيقات Meshy، والذي يمكن استخدامه للرجوع إليه أو استكشاف الأخطاء وإصلاحها. |
| `FBX` | FILE3DGLB | النموذج ثلاثي الأبعاد المُنشأ بتنسيق ملف GLB. |
| `FBX` | FILE3DFBX | النموذج ثلاثي الأبعاد المُنشأ بتنسيق ملف FBX. |
