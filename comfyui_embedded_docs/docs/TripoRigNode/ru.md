Вот перевод документации на русский язык:

Узел TripoRigNode генерирует риггированную 3D-модель на основе идентификатора задачи исходной модели. Он отправляет запрос к API Tripo для создания анимированного рига в формате GLB с использованием спецификации Tripo, а затем опрашивает API до завершения задачи генерации рига.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `original_model_task_id` | MODEL_TASK_ID | Да | - | Идентификатор задачи исходной 3D-модели, для которой требуется создать риг |
| `auth_token` | AUTH_TOKEN_COMFY_ORG | Нет | - | Токен аутентификации для доступа к API Comfy.org |
| `comfy_api_key` | API_KEY_COMFY_ORG | Нет | - | Ключ API для аутентификации в сервисе Comfy.org |
| `unique_id` | UNIQUE_ID | Нет | - | Уникальный идентификатор для отслеживания операции |

## Выходные параметры

| Имя выходного параметра | Тип данных | Описание |
|-------------|-----------|-------------|
| `model_file` | STRING | Сгенерированный файл риггированной 3D-модели (сохранён для обратной совместимости) |
| `rig task_id` | RIG_TASK_ID | Идентификатор задачи для отслеживания процесса генерации рига |
| `GLB` | FILE3DGLB | Сгенерированная риггированная 3D-модель в формате GLB |
