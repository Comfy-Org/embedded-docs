### مرور کلی

گره TripoRigNode یک مدل سه‌بعدی ریگ‌شده را از شناسه وظیفه مدل اصلی تولید می‌کند. این گره با ارسال درخواست به API Tripo، یک ریگ انیمیشنی در قالب GLB بر اساس مشخصات Tripo ایجاد کرده و سپس تا تکمیل وظیفه تولید ریگ، API را نظرسنجی می‌کند.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|-----------|-----------|----------|-------|-------------|
| `شناسه وظیفه مدل اصلی` | MODEL_TASK_ID | بله | - | شناسه وظیفه مدل سه‌بعدی اصلی که باید ریگ شود |
| `auth_token` | AUTH_TOKEN_COMFY_ORG | خیر | - | توکن احراز هویت برای دسترسی به API Comfy.org |
| `comfy_api_key` | API_KEY_COMFY_ORG | خیر | - | کلید API برای احراز هویت سرویس Comfy.org |
| `unique_id` | UNIQUE_ID | خیر | - | شناسه یکتا برای ردیابی عملیات |

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
|-------------|-----------|-------------|
| `model_file` | STRING | فایل مدل سه‌بعدی ریگ‌شده تولیدشده (برای سازگاری با نسخه‌های قبلی نگهداری می‌شود) |
| `rig task_id` | RIG_TASK_ID | شناسه وظیفه برای ردیابی فرآیند تولید ریگ |
| `GLB` | FILE3DGLB | مدل سه‌بعدی ریگ‌شده تولیدشده در قالب GLB |
