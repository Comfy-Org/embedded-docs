## نظرة عامة

تقوم عقدة TripoRigNode بإنشاء نموذج ثلاثي الأبعاد مُهيكل (مزوّد بهيكل عظمي) استنادًا إلى معرف مهمة النموذج الأصلي. ترسل العقدة طلبًا إلى واجهة Tripo API لإنشاء هيكل عظمي متحرك بتنسيق GLB باستخدام مواصفات Tripo، ثم تستقصي حالة API بشكل دوري حتى اكتمال مهمة إنشاء الهيكل العظمي.

## المدخلات

| المعامل | نوع البيانات | إجباري | النطاق | الوصف |
|----------|---------------|--------|---------|-------------|
| `original_model_task_id` | MODEL_TASK_ID | نعم | - | معرف مهمة النموذج ثلاثي الأبعاد الأصلي المراد هيكلته |
| `auth_token` | AUTH_TOKEN_COMFY_ORG | لا | - | رمز المصادقة للوصول إلى واجهة Comfy.org API |
| `comfy_api_key` | API_KEY_COMFY_ORG | لا | - | مفتاح API لمصادقة خدمة Comfy.org |
| `unique_id` | UNIQUE_ID | لا | - | معرّف فريد لتتبع العملية |

## المخرجات

| اسم المخرج | نوع البيانات | الوصف |
|-------------|---------------|-------------|
| `model_file` | STRING | ملف النموذج ثلاثي الأبعاد المُهيكل المُنشأ (مُحتفظ به للتوافق مع الإصدارات السابقة) |
| `rig task_id` | RIG_TASK_ID | معرف المهمة لتتبع عملية إنشاء الهيكل العظمي |
| `GLB` | FILE3DGLB | النموذج ثلاثي الأبعاد المُهيكل المُنشأ بتنسيق GLB |
