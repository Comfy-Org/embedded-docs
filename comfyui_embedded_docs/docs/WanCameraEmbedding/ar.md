### نظرة عامة

تقوم عقدة WanCameraEmbedding بتوليد تضمينات مسار الكاميرا باستخدام تضمينات بلوكر بناءً على معاملات حركة الكاميرا. تنشئ العقدة سلسلة من أوضاع الكاميرا التي تحاكي حركات الكاميرا المختلفة، وتحولها إلى موترات تضمين مناسبة لخطوط أنابيب توليد الفيديو.

### المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `وضعية_الكاميرا` | COMBO | نعم | "ثابت"<br>"تحريك لأعلى"<br>"تحريك لأسفل"<br>"تحريك لليسار"<br>"تحريك لليمين"<br>"تكبير"<br>"تصغير"<br>"عكس عقارب الساعة"<br>"مع عقارب الساعة" | نوع حركة الكاميرا المراد محاكاتها (الافتراضي: "ثابت") |
| `العرض` | INT | نعم | 16 إلى MAX_RESOLUTION | عرض المخرجات بالبكسل (الافتراضي: 832، الخطوة: 16) |
| `الارتفاع` | INT | نعم | 16 إلى MAX_RESOLUTION | ارتفاع المخرجات بالبكسل (الافتراضي: 480، الخطوة: 16) |
| `الطول` | INT | نعم | 1 إلى MAX_RESOLUTION | طول سلسلة مسار الكاميرا (الافتراضي: 81، الخطوة: 4) |
| `السرعة` | FLOAT | لا | 0.0 إلى 10.0 | سرعة حركة الكاميرا (الافتراضي: 1.0، الخطوة: 0.1) |
| `fx` | FLOAT | لا | 0.0 إلى 1.0 | معامل البعد البؤري x (الافتراضي: 0.5، الخطوة: 0.000000001) |
| `fy` | FLOAT | لا | 0.0 إلى 1.0 | معامل البعد البؤري y (الافتراضي: 0.5، الخطوة: 0.000000001) |
| `cx` | FLOAT | لا | 0.0 إلى 1.0 | إحداثي x للنقطة الرئيسية (الافتراضي: 0.5، الخطوة: 0.01) |
| `cy` | FLOAT | لا | 0.0 إلى 1.0 | إحداثي y للنقطة الرئيسية (الافتراضي: 0.5، الخطوة: 0.01) |
