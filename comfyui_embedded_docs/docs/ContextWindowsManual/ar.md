بالتأكيد، إليك الترجمة العربية للوثيقة التقنية الخاصة بعقدة "Context Windows (Manual)" في ComfyUI، مع الالتزام التام بقواعد الترجمة المحددة:

تتيح لك عقدة "النوافذ السياقية (يدوي)" (Context Windows (Manual)) تكوين النوافذ السياقية يدويًا للنماذج أثناء عملية أخذ العينات (sampling). تقوم هذه العقدة بإنشاء مقاطع سياقية متداخلة بطول وتداخل وأنماط جدولة محددة لمعالجة البيانات في أجزاء يمكن التحكم فيها مع الحفاظ على الاستمرارية بين المقاطع. توفر هذه العقدة خيارات متقدمة للتحكم في كيفية تطبيق النوافذ السياقية، بما في ذلك خلط الضوضاء (noise shuffling)، والاحتفاظ بالشروط (conditioning retention)، وإصلاح النوافذ السببية (causal window fixes).

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|-----------|-----------|----------|-------|-------------|
| `model` | MODEL | نعم | - | النموذج الذي سيتم تطبيق النوافذ السياقية عليه أثناء أخذ العينات. |
| `context_length` | INT | لا | 1+ | طول النافذة السياقية (الافتراضي: 16). |
| `context_overlap` | INT | لا | 0+ | مقدار التداخل في النافذة السياقية (الافتراضي: 4). |
| `context_schedule` | COMBO | لا | `STATIC_STANDARD`<br>`UNIFORM_STANDARD`<br>`UNIFORM_LOOPED`<br>`BATCHED` | خطوة (stride) النافذة السياقية. |
| `context_stride` | INT | لا | 1+ | خطوة النافذة السياقية؛ تنطبق فقط على الجداول المنتظمة (uniform schedules) (الافتراضي: 1). |
| `closed_loop` | BOOLEAN | لا | - | ما إذا كان سيتم إغلاق حلقة النافذة السياقية؛ تنطبق فقط على الجداول الحلقية (looped schedules) (الافتراضي: False). |
| `fuse_method` | COMBO | لا | `PYRAMID`<br>`LIST_STATIC` | الطريقة المستخدمة لدمج النوافذ السياقية (الافتراضي: PYRAMID). |
| `dim` | INT | لا | 0-5 | البُعد الذي سيتم تطبيق النوافذ السياقية عليه (الافتراضي: 0). |
| `ضجيج حر` | BOOLEAN | لا | - | ما إذا كان سيتم تطبيق خلط ضوضاء FreeNoise، مما يحسن مزج النوافذ (الافتراضي: False). |
| `cond_retain_index_list` | STRING | لا | - | قائمة بمؤشرات (indices) المساحة الكامنة (latent) التي سيتم الاحتفاظ بها في موترات الشروط (conditioning tensors) لكل نافذة. على سبيل المثال، تعيين هذا إلى '0' سيستخدم الصورة الأولية لكل نافذة (الافتراضي: ""). |
| `split_conds_to_windows` | BOOLEAN | لا | - | ما إذا كان سيتم تقسيم الشروط المتعددة (المنشأة بواسطة ConditionCombine) إلى كل نافذة بناءً على مؤشر المنطقة (region index) (الافتراضي: False). |
| `causal_window_fix` | BOOLEAN | لا | - | ما إذا كان سيتم إضافة إطار إصلاح سببي (causal fix frame) إلى النوافذ السياقية غير المفهرسة بالرقم 0 (الافتراضي: True). |

**قيود المعاملات:**

- يتم استخدام `context_stride` فقط عند تحديد جداول منتظمة (uniform schedules)
- `closed_loop` ينطبق فقط على الجداول الحلقية (looped schedules)
- يجب أن تكون قيمة `dim` بين 0 و 5 شاملة
- يتوقع `cond_retain_index_list` قائمة مفصولة بفواصل من المؤشرات الصحيحة كسلسلة نصية (على سبيل المثال، "0,1,2")
