Вот перевод документации для узла WanMoveConcatTrack:

Узел WanMoveConcatTrack объединяет два набора данных отслеживания движения в единую, более длинную последовательность. Он работает путем соединения путей треков и масок видимости из входных данных по соответствующим размерностям. Если предоставлен только один входной трек, узел просто передает эти данные без изменений.

## Входы

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `tracks_1` | TRACKS | Да | | Первый набор данных отслеживания движения для конкатенации. |
| `tracks_2` | TRACKS | Нет | | Необязательный второй набор данных отслеживания движения. Если не указан, `tracks_1` передается напрямую на выход. |

## Выходы

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `tracks` | TRACKS | Конкатенированные данные отслеживания движения, содержащие объединенные `track_path` и `track_visibility` из входных данных. |
