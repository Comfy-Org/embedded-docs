Вот перевод документации на русский язык:

Узел ImageOnlyCheckpointSave сохраняет файл контрольной точки (checkpoint), содержащий модель, кодировщик CLIP vision и VAE. Он создает файл в формате safetensors с указанным префиксом имени и сохраняет его в выходной директории. Этот узел специально предназначен для совместного сохранения компонентов модели, связанных с изображениями, в одном файле контрольной точки.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `модель` | MODEL | Да | - | Модель, сохраняемая в контрольной точке |
| `clip_vision` | CLIP_VISION | Да | - | Кодировщик CLIP vision, сохраняемый в контрольной точке |
| `vae` | VAE | Да | - | VAE (вариационный автоэнкодер), сохраняемый в контрольной точке |
| `префикс_названия_файла` | STRING | Да | - | Префикс для имени выходного файла (по умолчанию: "checkpoints/ComfyUI") |
| `prompt` | PROMPT | Нет | - | Скрытый параметр для данных рабочего процесса |
| `extra_pnginfo` | EXTRA_PNGINFO | Нет | - | Скрытый параметр для дополнительных метаданных PNG |

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| - | - | Этот узел не возвращает никаких выходных данных |
