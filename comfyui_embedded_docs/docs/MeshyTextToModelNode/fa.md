### گره Meshy: متن به مدل

گره Meshy: Text to Model از API Meshy برای تولید یک مدل سه‌بعدی بر اساس توضیحات متنی استفاده می‌کند. این گره درخواستی همراه با prompt و تنظیمات شما به API ارسال می‌کند، سپس منتظر تکمیل فرآیند تولید می‌ماند و فایل‌های مدل نهایی را دانلود می‌کند.

## ورودی‌ها

| پارامتر | نوع داده | الزامی | محدوده | توضیحات |
|---------|----------|--------|--------|---------|
| `model` | COMBO | بله | `"latest"` | نسخه مدل هوش مصنوعی مورد استفاده را مشخص می‌کند. در حال حاضر فقط نسخه "latest" در دسترس است. |
| `prompt` | STRING | بله | - | توضیحات متنی مدل سه‌بعدی که می‌خواهید تولید کنید. طول آن باید بین ۱ تا ۶۰۰ کاراکتر باشد. |
| `style` | COMBO | بله | `"realistic"`<br>`"sculpture"` | سبک هنری برای مدل سه‌بعدی تولیدشده. |
| `should_remesh` | DYNAMIC COMBO | بله | `"true"`<br>`"false"` | کنترل می‌کند که آیا مش تولیدشده پردازش شود یا خیر. وقتی روی "false" تنظیم شود، گره یک مش مثلثی پردازش‌نشده برمی‌گرداند. انتخاب "true" پارامترهای اضافی برای توپولوژی و تعداد چندضلعی‌ها را نمایش می‌دهد. |
| `topology` | COMBO | خیر* | `"triangle"`<br>`"quad"` | نوع چندضلعی هدف برای مدل remesh شده. این پارامتر فقط زمانی در دسترس و الزامی است که `should_remesh` روی "true" تنظیم شده باشد. |
| `target_polycount` | INT | خیر* | ۱۰۰ - ۳۰۰۰۰۰ | تعداد چندضلعی‌های هدف برای مدل remesh شده. مقدار پیش‌فرض ۳۰۰۰۰۰ است. این پارامتر فقط زمانی در دسترس و الزامی است که `should_remesh` روی "true" تنظیم شده باشد. |
| `symmetry_mode` | COMBO | بله | `"auto"`<br>`"on"`<br>`"off"` | تقارن در مدل تولیدشده را کنترل می‌کند. |
| `pose_mode` | COMBO | بله | `""`<br>`"A-pose"`<br>`"T-pose"` | حالت ایستادن (pose) برای مدل تولیدشده را مشخص می‌کند. رشته خالی به معنای عدم درخواست حالت خاص است. |
| `seed` | INT | بله | ۰ - ۲۱۴۷۴۸۳۶۴۷ | مقدار seed برای تولید. تنظیم این مقدار تعیین می‌کند که آیا گره باید دوباره اجرا شود، اما نتایج صرف‌نظر از مقدار seed غیرقطعی هستند. مقدار پیش‌فرض ۰ است. |

*توجه: پارامترهای `topology` و `target_polycount` به‌صورت شرطی الزامی هستند. آن‌ها فقط زمانی ظاهر می‌شوند و باید تنظیم شوند که پارامتر `should_remesh` روی "true" تنظیم شده باشد.

## خروجی‌ها

| نام خروجی | نوع داده | توضیحات |
|-----------|----------|---------|
| `model_file` | STRING | نام فایل مدل GLB تولیدشده. این خروجی برای سازگاری با نسخه‌های قبلی ارائه شده است. |
| `meshy_task_id` | MESHY_TASK_ID | شناسه یکتای وظیفه API Meshy. |
| `GLB` | FILE3DGLB | فایل مدل سه‌بعدی تولیدشده در قالب GLB. |
| `FBX` | FILE3DFBX | فایل مدل سه‌بعدی تولیدشده در قالب FBX. |
