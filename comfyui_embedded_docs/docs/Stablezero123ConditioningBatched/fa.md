### نمای کلی

این گره برای پردازش اطلاعات شرطی‌سازی (conditioning) به صورت دسته‌ای و متناسب با مدل StableZero123 طراحی شده است. تمرکز اصلی آن بر مدیریت همزمان چندین مجموعه داده شرطی‌سازی با کارایی بالا بوده و گردش کار را برای سناریوهایی که پردازش دسته‌ای حیاتی است، بهینه می‌کند.

### ورودی‌ها

| پارامتر | نوع داده | توضیحات |
|----------------------|--------------|-------------|
| `clip_vision` | `CLIP_VISION` | بردارهای嵌入 (embeddings) بینایی CLIP که زمینه بصری را برای فرآیند شرطی‌سازی فراهم می‌کنند. |
| `init_image` | `IMAGE` | تصویر اولیه که به عنوان نقطه شروع برای فرآیند تولید، شرطی‌سازی روی آن انجام می‌شود. |
| `vae` | `VAE` | خودرمزگذار متغیر (variational autoencoder) مورد استفاده برای رمزگذاری و رمزگشایی تصاویر در فرآیند شرطی‌سازی. |
| `width` | `INT` | عرض تصویر خروجی. |
| `height` | `INT` | ارتفاع تصویر خروجی. |
| `batch_size` | `INT` | تعداد مجموعه‌های شرطی‌سازی که در یک دسته پردازش می‌شوند. |
| `elevation` | `FLOAT` | زاویه ارتفاع برای شرطی‌سازی مدل سه‌بعدی که بر زاویه دید تصویر تولیدی تأثیر می‌گذارد. |
| `azimuth` | `FLOAT` | زاویه سمت (azimuth) برای شرطی‌سازی مدل سه‌بعدی که بر جهت‌گیری تصویر تولیدی تأثیر می‌گذارد. |
| `elevation_batch_increment` | `FLOAT` | تغییر افزایشی زاویه ارتفاع در سراسر دسته، که امکان ایجاد زوایای دید متنوع را فراهم می‌کند. |
| `azimuth_batch_increment` | `FLOAT` | تغییر افزایشی زاویه سمت در سراسر دسته، که امکان ایجاد جهت‌گیری‌های متنوع را فراهم می‌کند. |

### خروجی‌ها

| پارامتر | نوع داده | توضیحات |
|---------------|--------------|-------------|
| `positive` | `CONDITIONING` | خروجی شرطی‌سازی مثبت، که برای تقویت ویژگی‌ها یا جنبه‌های خاص در محتوای تولیدی تنظیم شده است. |
| `negative` | `CONDITIONING` | خروجی شرطی‌سازی منفی، که برای تضعیف ویژگی‌ها یا جنبه‌های خاص در محتوای تولیدی تنظیم شده است. |
| `latent` | `LATENT` | بازنمایی نهفته (latent representation) حاصل از فرآیند شرطی‌سازی، آماده برای مراحل بعدی پردازش یا تولید. |
