گره CropByBBoxes نواحی مستطیلی مشخصی را از یک دسته تصویر ورودی استخراج و تغییر اندازه می‌دهد. این گره از مختصات جعبه‌های مرزی (bounding box) ارائه‌شده برای تعیین ناحیه برش از هر تصویر استفاده می‌کند. نواحی برش‌خورده سپس به ابعاد خروجی مشخص‌شده تغییر اندازه می‌یابند، با گزینه‌هایی برای کشیدن (stretch) برش یا حاشیه‌گذاری (pad) آن به‌منظور حفظ نسبت ابعاد اصلی.

## ورودی‌ها

| پارامتر | نوع داده | اجباری | محدوده | توضیحات |
|---------|----------|--------|--------|---------|
| `image` | IMAGE | بله | - | دسته تصاویر ورودی برای برش. |
| `bboxes` | BOUNDINGBOX | بله | - | لیست جعبه‌های مرزی تعریف‌کننده نواحی برش. این ورودی اجباری است، یعنی باید متصل شود. |
| `output_width` | INT | خیر | 64 - 4096 | عرضی که هر برش به آن تغییر اندازه می‌یابد (پیش‌فرض: 512). |
| `output_height` | INT | خیر | 64 - 4096 | ارتفاعی که هر برش به آن تغییر اندازه می‌یابد (پیش‌فرض: 512). |
| `padding` | INT | خیر | 0 - 1024 | حاشیه اضافی بر حسب پیکسل که به هر طرف جعبه مرزی قبل از برش اضافه می‌شود (پیش‌فرض: 0). |
| `keep_aspect` | COMBO | خیر | `"stretch"`<br>`"pad"` | مشخص می‌کند که آیا برش برای تطبیق با اندازه خروجی کشیده شود، یا با پیکسل‌های سیاه حاشیه‌گذاری شود تا نسبت ابعاد آن حفظ گردد (پیش‌فرض: "stretch"). |

**توجه:** این گره هر بار یک فریم تصویر را پردازش می‌کند. اگر چندین جعبه مرزی برای یک فریم ارائه شود، یک ناحیه برش واحد محاسبه می‌کند که حاصل اجتماع (کوچک‌ترین مستطیل شامل تمام جعبه‌ها) تمام جعبه‌های ارائه‌شده است. اگر یک ناحیه برش محاسبه‌شده نامعتبر باشد (مثلاً عرض یا ارتفاع صفر)، گره یک برش جایگزین از مرکز-بالای تصویر ایجاد می‌کند.
