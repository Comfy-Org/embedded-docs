Вот перевод документации на русский язык:

Узел TextEncodeZImageOmni — это расширенный узел кондиционирования, который кодирует текстовый запрос вместе с дополнительными референсными изображениями в формат кондиционирования, подходящий для моделей генерации изображений. Он может обрабатывать до трех изображений, опционально кодируя их с помощью энкодера изображений и/или VAE для создания референсных латентов, и интегрирует эти визуальные референсы с текстовым запросом, используя определенную структуру шаблона.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `clip` | CLIP | Да | | Модель CLIP, используемая для токенизации и кодирования текстового запроса. |
| `image_encoder` | CLIPVision | Нет | | Дополнительная модель энкодера изображений. Если указана, она будет использоваться для кодирования входных изображений, а полученные эмбеддинги будут добавлены к кондиционированию. |
| `prompt` | STRING | Да | | Текстовый запрос для кодирования. Это поле поддерживает многострочный ввод и динамические запросы. |
| `auto_resize_images` | BOOLEAN | Нет | | Если включено (по умолчанию: True), входные изображения будут автоматически изменять размер на основе их площади в пикселях перед передачей в VAE для кодирования. |
| `vae` | VAE | Нет | | Дополнительная модель VAE. Если указана, она будет использоваться для кодирования входных изображений в латентные представления, которые добавляются к кондиционированию в качестве референсных латентов. |
| `image1` | IMAGE | Нет | | Первое дополнительное референсное изображение. |
| `image2` | IMAGE | Нет | | Второе дополнительное референсное изображение. |
| `image3` | IMAGE | Нет | | Третье дополнительное референсное изображение. |

**Примечание:** Узел может принимать максимум три изображения (`image1`, `image2`, `image3`). Входные параметры `image_encoder` и `vae` используются только в том случае, если предоставлено хотя бы одно изображение. Когда `auto_resize_images` имеет значение True и подключен `vae`, изображения изменяются так, чтобы их общая площадь в пикселях была близка к 1024x1024 перед кодированием.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | Итоговый результат кондиционирования, который содержит закодированный текстовый запрос и может включать закодированные эмбеддинги изображений и/или референсные латенты, если были предоставлены изображения. |
