# نظرة عامة

عقدة TextEncodeZImageOmni هي عقدة تكييف متقدمة تقوم بتشفير موجه نصي مع صور مرجعية اختيارية إلى تنسيق تكييف مناسب لنماذج توليد الصور. يمكنها معالجة ما يصل إلى ثلاث صور، مع تشفيرها اختياريًا باستخدام مشفر رؤية و/أو VAE لإنتاج مراجع كامنة، وتدمج هذه المراجع البصرية مع الموجه النصي باستخدام هيكل قالب محدد.

## المدخلات

| المعامل | نوع البيانات | إلزامي | النطاق | الوصف |
|----------|---------------|--------|---------|--------|
| `clip` | CLIP | نعم | | نموذج CLIP المستخدم لتقطيع وتشفير الموجه النصي. |
| `مُرمّز الصورة` | CLIPVision | لا | | نموذج مشفر رؤية اختياري. إذا تم توفيره، سيتم استخدامه لتشفير الصور المدخلة، وستتم إضافة التضمينات الناتجة إلى التكييف. |
| `التوجيه` | STRING | نعم | | الموجه النصي المراد تشفيره. يدعم هذا الحقل الإدخال متعدد الأسطر والموجهات الديناميكية. |
| `تغيير حجم الصور تلقائياً` | BOOLEAN | لا | | عند التمكين (الافتراضي: صحيح)، سيتم تغيير حجم الصور المدخلة تلقائيًا بناءً على مساحة البكسل الخاصة بها قبل تمريرها إلى VAE للتشفير. |
| `vae` | VAE | لا | | نموذج VAE اختياري. إذا تم توفيره، سيتم استخدامه لتشفير الصور المدخلة إلى تمثيلات كامنة، والتي تضاف إلى التكييف كمراجع كامنة. |
| `الصورة ١` | IMAGE | لا | | الصورة المرجعية الاختيارية الأولى. |
| `الصورة ٢` | IMAGE | لا | | الصورة المرجعية الاختيارية الثانية. |
| `الصورة ٣` | IMAGE | لا | | الصورة المرجعية الاختيارية الثالثة. |

**ملاحظة:** يمكن للعقدة استقبال حد أقصى ثلاث صور (`image1`، `image2`، `image3`). يتم استخدام مدخلات `image_encoder` و `vae` فقط إذا تم توفير صورة واحدة على الأقل. عندما تكون `auto_resize_images` صحيحة ويكون `vae` متصلاً، يتم تغيير حجم الصور بحيث تكون مساحة البكسل الإجمالية قريبة من 1024×1024 قبل التشفير.
