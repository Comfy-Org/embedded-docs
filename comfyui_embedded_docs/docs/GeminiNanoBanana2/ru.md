Вот перевод документации на русский язык:

Узел GeminiNanoBanana2 создаёт или редактирует изображения с помощью модели Google Vertex AI Gemini. Он отправляет текстовый запрос вместе с дополнительными эталонными изображениями или файлами в API и возвращает сгенерированное изображение, а также любой сопроводительный текст.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `prompt` | STRING | Да | Н/Д | Текстовый запрос, описывающий изображение для генерации или изменения. Включите все ограничения, стили или детали, которым должна следовать модель. |
| `model` | COMBO | Да | `"Nano Banana 2 (Gemini 3.1 Flash Image)"` | Конкретная модель Gemini для генерации изображений. |
| `seed` | INT | Да | от 0 до 18446744073709551615 | Когда значение seed фиксировано, модель прилагает максимум усилий для возврата одинакового ответа при повторных запросах. Детерминированный вывод не гарантируется. Кроме того, изменение модели или настроек параметров, таких как температура, может вызвать изменения в ответе даже при использовании одного и того же значения seed. По умолчанию используется случайное значение seed. (по умолчанию: 42) |
| `aspect_ratio` | COMBO | Да | `"auto"`<br>`"1:1"`<br>`"2:3"`<br>`"3:2"`<br>`"3:4"`<br>`"4:3"`<br>`"4:5"`<br>`"5:4"`<br>`"9:16"`<br>`"16:9"`<br>`"21:9"` | Если установлено значение 'auto', соответствует соотношению сторон вашего входного изображения; если изображение не предоставлено, обычно генерируется квадрат 16:9. (по умолчанию: "auto") |
| `resolution` | COMBO | Да | `"1K"`<br>`"2K"`<br>`"4K"` | Целевое разрешение вывода. Для 2K/4K используется встроенный апскейлер Gemini. |
| `response_modalities` | COMBO | Да | `"IMAGE"`<br>`"IMAGE+TEXT"` | Определяет тип контента, который вернёт модель. (расширенные) |
| `thinking_level` | COMBO | Да | `"MINIMAL"`<br>`"HIGH"` | Управляет глубиной процесса рассуждения модели. |
| `images` | IMAGE | Нет | Н/Д | Дополнительные эталонные изображения. Чтобы включить несколько изображений, используйте узел Batch Images (до 14). |
| `files` | CUSTOM | Нет | Н/Д | Дополнительные файлы для использования в качестве контекста для модели. Принимает входные данные от узла Gemini Generate Content Input Files. |
| `system_prompt` | STRING | Нет | Н/Д | Базовые инструкции, определяющие поведение ИИ. (расширенные) |

**Примечание:** Вход `images` поддерживает максимум 14 изображений. Если предоставлено больше, узел выдаст ошибку.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `image` | IMAGE | Основное изображение, сгенерированное или изменённое моделью. |
| `string` | STRING | Любой текстовый контент, возвращённый моделью. |
| `thought_image` | IMAGE | Первое изображение из процесса рассуждения модели. Доступно только при thinking_level HIGH и модальности IMAGE+TEXT. |
