Вот перевод документации на русский язык:

Узел TextEncodeAceStepAudio обрабатывает текстовые входные данные для аудиокондиционирования, объединяя теги и текст песни в токены, а затем кодируя их с регулируемой силой влияния текста песни. Он принимает модель CLIP вместе с текстовыми описаниями и текстом песни, токенизирует их совместно и генерирует данные кондиционирования, подходящие для задач аудиогенерации. Узел позволяет точно настраивать влияние текста песни через параметр силы, который контролирует его воздействие на конечный результат.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|----------|------------|--------------|----------|----------|
| `clip` | CLIP | Да | - | Модель CLIP, используемая для токенизации и кодирования |
| `tags` | STRING | Да | - | Текстовые теги или описания для аудиокондиционирования (поддерживает многострочный ввод и динамические подсказки) |
| `lyrics` | STRING | Да | - | Текст песни для аудиокондиционирования (поддерживает многострочный ввод и динамические подсказки) |
| `lyrics_strength` | FLOAT | Нет | 0.0 - 10.0 | Управляет силой влияния текста песни на выходные данные кондиционирования (по умолчанию: 1.0, шаг: 0.01) |

## Выходные данные

| Имя выхода | Тип данных | Описание |
|------------|------------|----------|
| `conditioning` | CONDITIONING | Закодированные данные кондиционирования, содержащие обработанные текстовые токены с примененной силой влияния текста песни |
