Вот перевод документации на русский язык:

Узел WanMoveTracksFromCoords создаёт траектории движения на основе строки координат в формате JSON. Он преобразует данные координат в тензорный формат, который может использоваться другими узлами обработки видео, и при необходимости может применять маску для управления видимостью траекторий с течением времени.

## Входные параметры

| Параметр | Тип данных | Обязательный | Диапазон | Описание |
|-----------|-----------|----------|-------|-------------|
| `track_coords` | STRING | Нет | N/A | Строка в формате JSON, содержащая данные координат для траекторий. Значение по умолчанию — пустой список (`"[]"`). |
| `track_mask` | MASK | Нет | N/A | Необязательная маска. При её наличии узел использует её для определения видимости каждой траектории в каждом кадре. |

**Примечание:** Входной параметр `track_coords` ожидает определённую структуру JSON. Это должен быть список траекторий, где каждая траектория представляет собой список кадров, а каждый кадр — объект с координатами `x` и `y`. Количество кадров должно быть одинаковым для всех траекторий.

## Выходные параметры

| Имя выхода | Тип данных | Описание |
|-------------|-----------|-------------|
| `tracks` | TRACKS | Сгенерированные данные траекторий, содержащие координаты пути и информацию о видимости для каждой траектории. |
| `track_length` | INT | Общее количество кадров в сгенерированных траекториях. |
